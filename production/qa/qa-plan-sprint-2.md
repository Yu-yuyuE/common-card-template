## QA Execution Protocol

1. **Automated Tests**: All Logic and Integration stories require passing automated tests in the `tests/` directory.
2. **UI Evidence**: All UI stories require manual verification with screenshots saved in `production/qa/evidence/`.
3. **Smoke Check**: Run `/smoke-check sprint` after all stories are complete.
4. **QA Sign-off**: Run `/team-qa sprint` to generate the official sign-off report.

## Critical Path Testing

The following integration tests are essential for the sprint goal:

1. **Combat Loop Integration (2-4)**: Player plays card → Enemy responds → Status effects apply → Resources change → Phase advances
2. **Resource/Status联动 (2-10)**: DOT damage reduces HP → StatusManager triggers → Resource recovery triggers status removal
3. **Phase Transition + Weather (2-3 + 2-9)**: Enemy HP<40% triggers phase change → Weather change triggers status modifiers
