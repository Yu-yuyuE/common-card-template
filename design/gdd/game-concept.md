# Game Concept v3: 三国称雄 (Three Kingdoms Ascendant)

*Updated: 2026-03-27*
*Status: Draft v4（敌方多主将 + 附属兵种层）*

---

## Elevator Pitch

> 一款三国题材Roguelike卡牌游戏。玩家扮演历史武将，在树形战役地图中推进节点，通过“武将卡组决策 + 兵种召唤自动作战 + 地形克制博弈”赢得战斗。战场采用“多主将 + 附属兵种层”模型：武将为主，兵种为附属编制层，将令不占手牌，负面体系可被经营与反制。

---

## Overview

本作是单人PC向中型策略卡牌项目，核心体验为“运筹帷幄之中，决胜千里之外”。

v3关键更新：
1. 兵种定位为“辅助武将作战”，不喧宾夺主。
2. 战场采用“我方1主将 vs 敌方最多3主将”，兵种改为主将附属层。
3. 负面卡体系扩展：既可作为风险，也可作为构筑资源。
4. 地图采用5张树形小地图（每张10-15层，单Boss终点）。
5. 卡牌支持单次升级与双离场机制（战斗移除/地图消耗）。

## Player Fantasy

**“我是主将，兵种听令辅战，借地利与谋略制敌。”**

玩家核心决策顺序：
- 先用武将卡组做主决策；
- 再用召唤兵种稳定战线；
- 关键回合用将令改写兵种行为；
- 结合地形和克制形成战术优势。

## Detailed Rules

### 1) 战斗资源层

- **手牌（主资源）**：武将行动卡 + 兵种召唤卡。
- **费用（行动点）**：每回合刷新。
- **将令槽（独立资源）**：
  - 不占手牌，不消耗抽牌位。
  - 默认每回合+1，将令上限2。
  - 通用将令：集火 / 固守 / 突击。

### 2) 多主将+附属兵种模型（已简化）

采用“主将为目标体，兵种为附属层”模型：
- 我方：**主将位1**（可挂载兵种附属）
- 敌方：**敌将位最多3**（每名敌将可挂载兵种附属）

规则要点：
1. 武将是主输出/主决策核心。
2. 兵种不再作为独立目标单位，而是所属主将的附属编制层。
3. 默认攻击结算顺序：先命中目标主将的附属兵种层。
4. 仅当该主将附属兵种编制归零后，常规攻击才能命中该主将。
5. 带“直击主将”关键词的卡牌/技能可绕过附属兵种层。

### 3) 兵种系统（附属编制方案）

- 兵种通过召唤卡挂载到主将后，以附属编制层形式生效。
- 自动逻辑：
  1. 优先可攻击的克制目标；
  2. 其次攻击默认威胁最高目标；
  3. 若有将令，按将令目标覆盖。

### 4) 克制系统

软克制倍率：
- 克制 `x1.25`
- 被克 `x0.80`
- 中性 `x1.00`

### 5) 地形系统（节点内动态变化）

- 节点有初始地形标签。
- 战斗中可由事件/卡牌触发地形变化。
- 地形影响兵种机动、伤害、状态。

### 6) 卡牌升级与离场机制

- 每种卡牌支持一次升级（Lv1→Lv2），不可重复升级。
- 升级后保留卡牌定位，仅提升数值或追加轻量效果。
- 离场机制分为：
  1. 使用后移除：该战斗中移出循环，本局结束回到卡组；
  2. 消耗卡：当前地图仅可成功使用一次，进入下一张地图重置。

### 7) 负面体系

- 负面卡数量提高（见 `cards-design.md`）。
- 负面卡可通过商店/技能/卡牌净化。
- 特定武将可“利用负面卡”转化收益（如司马懿）。
- 新增“施加敌方负面”流派武将（见 `heroes-design.md`）。

### 8) 统御容量定义（明确解释）

**统御容量 = 当前可维持“所有已挂载附属兵种”总编制点数上限。**

示例：
- 武将容量=10
- 步兵占2，骑兵占3
- 可挂载：骑兵(3)+骑兵(3)+步兵(2)+步兵(2)=10
- 超出则不能继续挂载新兵种。

设计目的：
- 防止无限铺兵；
- 让玩家在“少量精兵 vs 多量轻兵”间做策略选择。

### 9) 1v3画面可读性策略

在新模型下，最大同屏核心目标为“我方1主将，对方最多3主将”，兵种以附属层显示。
为防拥挤，采用：
1. 主将立绘为主视觉；
2. 附属兵种以主将侧边“编制条+兵种图标”显示；
3. 默认仅显示关键状态（护甲、负面层、附属编制值）；
4. 动效优先线性反馈，避免复杂位移。

## Formulas

### F1. 兵种最终伤害

`FinalDamage = BaseDamage × CounterMod × TerrainMod × BuffMod`

### F2. 兵种存活更新

`TroopHP_next = TroopHP_now - IncomingDamage + Reinforce`

### F3. 容量约束

`TotalAttachedTroopCost ≤ CommanderCapacity`

### F4. 将令机会

`OrderGain = BaseOrderGain + HeroTraitBonus + ItemBonus`

- BaseOrderGain 默认 = 1/回合
- 将令上限 = 2

### F5. 卡牌升级收益

`UpgradeGain = Lv2Power / Lv1Power`

- 建议区间：1.20~1.35

### F6. 地图消耗卡可用性

`MapConsumableUsable = !UsedOnMap(cardId, mapId)`

- false 时该卡在当前地图不可再打出

## Edge Cases

1. 目标主将附属槽满且容量不足时召唤：卡牌灰显不可释放。
2. 将令可用但无可用附属兵种：按钮可见但禁用。
3. 动态地形在结算中途切换：从切换后事件开始使用新地形修正。
4. 负面卡抽到后被即时净化：按“先触发抽到效果，再净化”规则。
5. 多敌将目标冲突（同优先级）：按速度值或固定顺序决策。
6. 升级卡被复制：复制体继承当前等级，不额外获得升级次数。
7. 使用后移除卡被回收：默认不可回收，除非卡牌文本明确允许。
8. 消耗卡跨战斗：同地图首用后锁定，切换下一图解锁。

## Dependencies

- **卡牌战斗系统**：提供手牌、费用、出牌、结算框架。
- **地图节点系统**：提供地形标签、历史节点事件入口。
- **状态效果系统**：承载地形Buff/Debuff与将令状态。
- **AI目标选择模块**：兵种自动作战需要目标优先级支持。
- **武将特性系统**：实现诸葛亮“额外将令”能力及负面流特性。
- **平衡配置系统（数据驱动）**：存放克制矩阵、地形参数、容量参数。
- **系统文档依赖**：`heroes-design.md` / `cards-design.md` / `equipment-design.md` / `map-design.md`

## Acceptance Criteria

1. 战斗采用“我方1主将 vs 敌方最多3主将 + 附属兵种层”结构并稳定运行。
2. 将令系统不进入手牌且每回合可验证充能。
3. 统御容量规则可阻止超限挂载。
4. 负面卡具备“惩罚+可利用”双属性玩法。
5. 1v3场面下，玩家可清晰识别主将目标与附属兵种状态。
6. 每种卡牌支持且仅支持一次升级（Lv1→Lv2）。
7. “使用后移除”与“消耗卡”可被系统正确追踪并按规则重置。

## Scope and Feasibility (v3)

### MVP最小验证包（建议）

1. 新战场：我方1将，对方最多3将；兵种为主将附属编制层
2. 兵种：步兵/枪兵/骑兵（3类）
3. 地形：平原/山地/林地（3类，支持动态切换）
4. 将令：集火/固守（2个）
5. 武将特性：诸葛亮额外将令 + 1名负面流武将

### 最大风险

- **可读性风险**：玩家无法快速理解“自动兵种为何选该目标”。

### 风险缓解

- 战斗日志显示关键修正来源：克制、地形、将令、负面。
- 兵种状态简化显示：仅保留关键图标与层数。
