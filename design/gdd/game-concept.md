# Game Concept v3: 三国称雄 (Three Kingdoms Ascendant)

*Updated: 2026-03-27*
*Status: Draft v3（两排战场 + 负面流扩展）*

---

## Elevator Pitch

> 一款三国题材Roguelike卡牌游戏。玩家扮演历史武将，在树形战役地图中推进节点，通过“武将卡组决策 + 兵种召唤自动作战 + 地形克制博弈”赢得战斗。战场采用“将+兵”双排简化模型：武将为主，兵种为辅，将令不占手牌，负面体系可被经营与反制。

---

## Overview

本作是单人PC向中型策略卡牌项目，核心体验为“运筹帷幄之中，决胜千里之外”。

v3关键更新：
1. 兵种定位为“辅助武将作战”，不喧宾夺主。
2. 战场从多格前后排简化为“两排（将位+兵位）”。
3. 负面卡体系扩展：既可作为风险，也可作为构筑资源。
4. 地图采用3张树形小地图（每张10-15层，单Boss终点）。

## Player Fantasy

**“我是主将，兵种听令辅战，借地利与谋略制敌。”**

玩家核心决策顺序：
- 先用武将卡组做主决策；
- 再用召唤兵种稳定战线；
- 关键回合用将令改写兵种行为；
- 结合地形和克制形成战术优势。

---

## Detailed Rules

### 1) 战斗资源层

- **手牌（主资源）**：武将行动卡 + 兵种召唤卡。
- **费用（行动点）**：每回合刷新。
- **将令槽（独立资源）**：
  - 不占手牌，不消耗抽牌位。
  - 默认每回合+1，将令上限2。
  - 通用将令：集火 / 固守 / 突击。

### 2) 两排战场模型（已简化）

采用“将+兵”双排：
- 我方：**武将位1** + **兵种位最多2**
- 敌方：**敌将位1** + **敌兵位最多2**

规则要点：
1. 武将是主输出/主决策核心。
2. 兵种主要承担补伤害、护卫、联动触发。
3. 近战默认优先敌兵位；远程可指定敌将位。
4. 敌兵清空后，敌将更易被集火。

### 3) 兵种系统（S方案）

- 兵种通过召唤卡上场后自动作战。
- 自动逻辑：
  1. 优先可攻击的克制目标；
  2. 其次攻击默认威胁最高目标；
  3. 若有将令，按将令目标覆盖。

### 4) 克制系统

软克制倍率：
- 克制 `x1.25`
- 被克 `x0.80`
- 中性 `x1.00`

### 5) 地形系统（节点内动态变化）

- 节点有初始地形标签。
- 战斗中可由事件/卡牌触发地形变化。
- 地形影响兵种机动、伤害、状态。

### 6) 负面体系

- 负面卡数量提高（见 `cards-design.md`）。
- 负面卡可通过商店/技能/卡牌净化。
- 特定武将可“利用负面卡”转化收益（如司马懿）。
- 新增“施加敌方负面”流派武将（见 `heroes-design.md`）。

### 7) 容量定义（明确解释）

**容量 = 当前可维持兵种总编制点数上限。**

示例：
- 武将容量=10
- 步兵占2，骑兵占3
- 可上场：骑兵(3)+骑兵(3)+步兵(2)+步兵(2)=10
- 超出则不能召唤新兵种。

设计目的：
- 防止无限铺兵；
- 让玩家在“少量精兵 vs 多量轻兵”间做策略选择。

### 8) 1v3画面可读性策略

在双排模型下，最大同屏目标可出现“我方1主将+2兵，对方1主将+2兵”。
为防拥挤，采用：
1. 武将立绘主视觉，兵种缩放显示；
2. 兵种以“编队单位”表现（数量图标替代多模型）；
3. 默认仅显示关键状态（护甲、负面层）；
4. 动效优先线性反馈（箭线/冲锋线），避免复杂位移。

## Formulas

### F1. 兵种最终伤害

`FinalDamage = BaseDamage × CounterMod × TerrainMod × BuffMod`

### F2. 兵种存活更新

`TroopHP_next = TroopHP_now - IncomingDamage + Reinforce`

### F3. 容量约束

`TotalCapacityUsed ≤ CommanderCapacity`

### F4. 将令机会

`OrderGain = BaseOrderGain + HeroTraitBonus + ItemBonus`

- BaseOrderGain 默认 = 1/回合
- 将令上限 = 2

## Edge Cases

1. 兵位满且容量满时召唤：卡牌灰显不可释放。
2. 将令可用但无友军兵种：按钮可见但禁用。
3. 动态地形在结算中途切换：从切换后事件开始使用新地形修正。
4. 负面卡抽到后被即时净化：按“先触发抽到效果，再净化”规则。
5. 双排目标冲突（同优先级）：按速度值或固定顺序决策。

## Dependencies

- **卡牌战斗系统**：提供手牌、费用、出牌、结算框架。
- **地图节点系统**：提供地形标签、历史节点事件入口。
- **状态效果系统**：承载地形Buff/Debuff与将令状态。
- **AI目标选择模块**：兵种自动作战需要目标优先级支持。
- **武将特性系统**：实现诸葛亮“额外将令”能力及负面流特性。
- **平衡配置系统（数据驱动）**：存放克制矩阵、地形参数、容量参数。
- **系统文档依赖**：`heroes-design.md` / `cards-design.md` / `equipment-design.md` / `map-design.md`

## Tuning Knobs

| Knob | Safe Range | Affects |
| ---- | ---- | ---- |
| 克制倍率 | 1.15~1.35 | 兵种存在感 |
| 地形修正 | 0.85~1.20 | 地形战略价值 |
| 将令上限 | 1~3 | 战术频率 |
| 负面卡占比 | 10%~20% | 风险/收益节奏 |
| 武将容量 | 8~12 | 兵种编制深度 |

## Acceptance Criteria

1. 战斗采用双排结构并稳定运行（将位+兵位）。
2. 将令系统不进入手牌且每回合可验证充能。
3. 容量规则可阻止超限召唤。
4. 负面卡具备“惩罚+可利用”双属性玩法。
5. 1v3场面下，玩家可清晰识别目标与关键状态。

## Scope and Feasibility (v3)

### MVP最小验证包（建议）

1. 双排战场：我方1将+2兵，敌方1将+2兵
2. 兵种：步兵/枪兵/骑兵（3类）
3. 地形：平原/山地/林地（3类，支持动态切换）
4. 将令：集火/固守（2个）
5. 武将特性：诸葛亮额外将令 + 1名负面流武将

### 最大风险

- **可读性风险**：玩家无法快速理解“自动兵种为何选该目标”。

### 风险缓解

- 战斗日志显示关键修正来源：克制、地形、将令、负面。
- 兵种状态简化显示：仅保留关键图标与层数。
