# Game Concept v3: 三国称雄 (Three Kingdoms Ascendant)

_Updated: 2026-03-27_
_Status: Draft v5（兵种卡牌化 + 统帅3-6体系）_

---

## Elevator Pitch

> 一款三国题材Roguelike卡牌游戏。玩家扮演历史武将，在树形战役地图中推进节点，通过”武将卡组决策 + 兵种卡辅战 + 地形克制博弈”赢得战斗。兵种以卡牌形式加入卡组，由玩家主动打出，诅咒体系可被经营与反制。

---

## Overview

本作是单人PC向中型策略卡牌项目，核心体验为“运筹帷幄之中，决胜千里之外”。

v5关键更新：

1. 兵种改为**卡牌形式**：步兵/骑兵/弓兵/谋士/盾兵卡通过军营节点选入卡组，由玩家在战斗中主动打出。
2. 统帅值重定义为**3~6**，代表该武将卡组中可携带的兵种卡上限张数（历史官职高者上限高）。
3. 战场采用”我方1主将 vs 敌方同时最多3主将，精英战或BOSS战可能超过3人”。
4. 诅咒卡体系扩展：既可作为风险，也可作为构筑资源。
5. 根据每个武将生平，设计5张大地图，每张地图采用3张树形小地图（每张10-15层节点，单Boss终点）。
6. 在地图中，移动到下一个节点，消耗粮草5~9（上限100，每张大地图开局默认加满，可在商店购买，Boss战胜利奖励小部分粮草）

## Player Fantasy

**“我是主将，借地利与谋略制敌。”**

玩家核心决策顺序：

- 先用武将卡组做主决策；
- 结合地形和兵种卡形成战术优势。

## Detailed Rules

### 1) 战斗资源层

- **手牌（主资源）**：近战攻击卡/远程攻击卡/技能卡/兵种卡/诅咒卡。
- **费用（行动点）**：每局战斗刷新。

### 2) 战场模型

- 我方：**主将位1**
- 敌方：**敌将位最多3**（每名敌将携带附属兵种层，作为承伤防御体）

规则要点：

1. 武将是主输出/主决策核心。
2. **玩家侧无附属兵种层**：兵种以卡牌形式加入卡组，由玩家主动打出，按卡牌文本结算效果。
3. **敌方保留附属兵种层**：普通攻击默认先命中敌将的附属兵种层，兵种层清空后才可命中该敌将。
4. 带”直击主将”关键词的卡牌可绕过敌方兵种层，直接命中敌将。

### 2.1) 单位效果作用范围规则（统一裁定）

**”目标单位”= 一名敌将（含其附属兵种层）或我方主将，按效果类型分别结算。**

| 效果类型 | 默认作用范围 | 例外说明 |
| -------- | ------------ | -------- |
| 状态（Buff/Debuff） | 作用于目标主将 | 卡牌文本含”仅兵种层”时才作用于附属兵种层 |
| 回复效果 | 作用于目标主将 | 同上 |
| 持续伤害（中毒/灼烧等） | 每回合对目标主将结算 | 同上 |
| 普通攻击伤害 | **优先减少敌将附属兵种层值**；兵种层归零后对敌将造成HP伤害 | 带”直击主将”文本时绕过兵种层 |
| “直击主将”攻击 | 直接对敌将造成HP伤害（不影响兵种层） | — |
| 兵种卡效果 | 按卡牌文本指定目标，不依赖附属层逻辑 | — |

**”所有敌方单位” / “敌方全体”**：指所有敌方主将单位（不含其附属兵种层）。

**”周围单位”**：在1v3战场模型中，指其他敌方主将单位。

### 3) 兵种卡系统

- **玩家侧**：步兵/骑兵/弓兵/谋士/盾兵以**卡牌**形式存在，在军营节点选入卡组，由玩家主动打出。
- 卡组中兵种卡数量上限由武将**统帅值（3~6）**决定。
- 兵种卡按卡面文本指定目标，打出后进入弃牌堆（与普通卡相同）。

### 4) 地形天气系统（节点内动态变化）

- 节点有初始地形/天气标签，根据战争史实设定，默认为平原/晴。
- 战斗中可由事件/卡牌触发天气变化。
- 地形/天气影响兵种伤害、状态效果。

### 5) 卡牌升级与离场机制

- 每种卡牌支持一次升级（Lv1→Lv2），不可重复升级。
- 升级后保留卡牌定位，仅提升数值或追加轻量效果。
- 离场机制分为：
  1. 使用后移除：该战斗中**移除**（不在抽牌堆/弃牌堆中），本局结束回到卡组；
  2. 消耗卡：仅可成功使用一次。

### 6) 诅咒体系

- 诅咒卡数量提高（见 `cards-design.md`）。
- 诅咒卡可通过商店净化/技能净化/卡牌净化（操作结果为**移出卡组**）。
- 特定武将可“利用诅咒卡”转化收益（如司马懿）。

### 7) 统帅定义（明确解释）

**统帅 = 该武将卡组中可携带兵种卡的上限张数。**

武将核心数值范围：

- **HP：30~50（含边界）**
- **统帅：3~6（含边界）**

示例：

- 武将统帅=4，卡组中最多放入4张兵种卡。
- 超出统帅上限时，军营节点无法再选择新的兵种卡。

## Formulas

### F1. 兵种最终伤害

`FinalDamage = BaseDamage × CounterMod × TerrainMod × BuffMod`

### F2. 兵种存活更新

`TroopHP_next = TroopHP_now - IncomingDamage + Reinforce`

### F3. 统帅约束

`TroopCardCountInDeck ≤ CommanderLeadership`

### F4. 卡牌升级收益

`UpgradeGain = Lv2Power / Lv1Power`

- 建议区间：1.20~1.35

### F5. 地图消耗卡可用性

`MapConsumableUsable = !UsedOnMap(cardId, mapId)`

- false 时该卡在当前地图不可再打出

---

## Edge Cases

1. 兵种卡超出统帅上限时：军营节点该选项灰显不可选。
2. 动态地形在结算中途切换：从切换后事件开始使用新地形修正。
3. 诅咒卡抽到后被即时净化：按”先触发抽到效果，再移出卡组”规则。
4. 多敌将目标冲突（同优先级）：按速度值或固定顺序决策。
5. 升级卡被复制：复制体继承当前等级，不额外获得升级次数。
6. 使用后移除卡被回收：默认不可回收，除非卡牌文本明确允许。
7. 消耗卡跨战斗：同地图首用后锁定，切换下一图解锁。

## Dependencies

- **卡牌战斗系统**：提供手牌、费用、出牌、结算框架。
- **地图节点系统**：提供地形标签、历史节点事件入口。
- **状态效果系统**：承载地形Buff/Debuff与状态效果。
- **AI目标选择模块**：兵种自动作战需要目标优先级支持。
- **武将特性系统**：实现武将被动特性及诅咒流特性。
- **平衡配置系统（数据驱动）**：地形参数、统帅参数。
- **系统文档依赖**：`heroes-design.md` / `cards-design.md` / `equipment-design.md` / `map-design.md`

## Tuning Knobs

| Knob           | Safe Range | Affects       |
| -------------- | ---------- | ------------- |
| 地形修正       | 0.85~1.20  | 地形战略价值  |
| 诅咒卡占比     | 10%~20%    | 风险/收益节奏 |
| 统帅           | 3~6        | 可携带兵种卡数量  |
| 升级收益系数   | 1.20~1.35  | 单卡成长幅度  |

| 地图消耗卡占比 | 3%~8%      | 地图资源节奏  |

## Acceptance Criteria

1. 战斗采用”我方1主将 vs 敌方最多3主将 + 附属兵种层”结构并稳定运行。
2. 统帅规则可阻止卡组中兵种卡超限加入。
3. 诅咒卡具备”惩罚+可利用”双属性玩法。
4. 1v3场面下，玩家可清晰识别主将目标与附属兵种状态。
5. 每种卡牌支持且仅支持一次升级（Lv1→Lv2）。
6. “使用后移除”与”消耗卡”可被系统正确追踪并按规则重置。

## Scope and Feasibility (v3)

### MVP最小验证包（建议）

1. 新战场：我方1将，对方最多3将；敌方保留附属兵种层
2. 兵种卡：步兵/骑兵/弓兵/谋士/盾兵（5类，暂定）
3. 地形：平原/山地/森林（3类，暂定，不支持动态切换）
4. 天气：晴/风/雨（3类，暂定，支持动态切换）
