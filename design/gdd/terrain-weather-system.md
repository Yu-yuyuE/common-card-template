# D1 — 地形天气系统（Terrain Weather System）

_Created: 2026-04-07_
_Status: Complete_
_Priority: MVP / 功能层_

---

## Overview

地形天气系统为每场战斗提供环境背景层，通过节点预设的**地形标签**（7种）和**天气标签**（4种）在战斗开始时施加环境效果，并在战斗中支持天气的动态切换。系统本身不直接处理伤害结算——它向卡牌战斗系统（C2）、状态效果系统（C1）和兵种卡系统（D2）提供修正系数，作为战场环境的被动规则层。地形在战斗中**固定不变**；天气可由特定卡牌或事件在战斗中动态切换，新天气效果从切换后的下一次结算起生效。7种地形与4种天气构成28种组合，每种组合对战局节奏产生差异化影响，强化三国战争的历史地理代入感。

**系统职责边界：**
- 负责：地形/天气标签的持有、初始化效果结算、天气切换时机管理、修正系数的对外接口
- 不负责：具体伤害数值计算（由 C2 处理）、状态施加逻辑（由 C1 处理）

## Player Fantasy

**"我知道此地地形，我知道今日天时——这是我的优势。"**

玩家在地形天气系统中应感受到两种核心幻想：

**① 读图博弈感**
进入节点前，玩家已可看到地形/天气标签，据此选择是否携带对应兵种卡、决定路线取舍。骑兵卡在平原如虎添翼，在山地却成负担——"看地形配牌"是策略深度的来源，而非随机惩罚。

**② 历史代入感**
官渡是平原、赤壁是水域、剑阁是关隘——地形不只是机制标签，更是历史场景的召唤。踏上雪地节点时，凛冽的冻伤压力让玩家感受到并州冬季战役的真实重量。

**不应有的感受：**
- 地形天气是随机惩罚（每种地形的应对方式应是可学习的）
- 28种组合让人眼花缭乱（每个地形/天气效果最多1~2条规则，保持可记忆性）

## Detailed Rules

### 3.1 标签结构

每个战斗节点持有两个独立标签：

| 标签类型 | 数量 | 可变性 | 默认值 |
|---------|------|--------|--------|
| 地形标签 | 1个（固定） | 战斗中**不可改变** | 平原 |
| 天气标签 | 1个（可变） | 可由卡牌/事件切换 | 晴 |

- 地形由地图编辑器按历史背景预设，战斗中锁定
- 天气切换后，新效果从**下一次结算**起生效（不追溯当前结算）
- 同一战斗中天气可多次切换，无次数上限（受冷却约束）

---

### 3.2 地形效果定义（7种）

| 地形 | 效果 | 生效时机 | 主要对应州 |
|------|------|---------|-----------|
| **平原** | 骑兵卡伤害 +50% | 出牌结算时 | 冀州、兖州、豫州 |
| **山地** | 骑兵卡伤害 -50% | 出牌结算时 | 益州、并州 |
| **森林** | 灼烧伤害 +50%；每回合结束对所有单位施加 1 层中毒 | 持续（每回合结束） | 荆州、青州 |
| **水域** | 战斗开始时所有单位获得 1 层滑倒 | 战斗初始化时（一次性） | 扬州、荆州、交州 |
| **沙漠** | 骑兵卡费用 -1；每回合结束对所有单位施加 1 层灼烧 | 出牌时 / 持续 | 凉州、幽州北部 |
| **关隘** | 守城方战斗开始获得 10 点护甲 + 2 层坚守 | 战斗初始化时（一次性） | 司州、益州（剑阁）、并州 |
| **雪地** | 战斗开始时玩家获得 2 层冻伤；灼烧伤害 -50% | 初始化（一次性） / 结算时 | 幽州、并州（冬季战役） |

> **守城方定义**：当前 MVP 阶段默认敌方为守城方（玩家始终为攻方）。

---

### 3.3 天气效果定义（4种）

| 天气 | 效果 | 生效时机 |
|------|------|---------|
| **晴** | 无额外效果 | — |
| **风** | 处于灼烧状态的单位，每回合结束对其周围未灼烧的单位施加 1 层灼烧 | 持续（每回合结束） |
| **雨** | 灼烧层数每回合额外消减 1 层（叠加在正常消减之上） | 持续（每回合结束） |
| **雾** | 战斗开始时所有单位获得 2 层盲目 | 战斗初始化时（一次性） |

---

### 3.4 天气切换规则

- **触发来源**：特定技能卡或地图事件
- **切换时机**：卡牌结算完毕后立即切换，不中断当前结算
- **生效时机**：切换后**下一次**受该天气影响的结算起生效
- **持续时间**：无限持续，直到被再次切换
- **冷却约束**：同一来源的天气切换冷却至少 2 回合（防止反复横跳）
- **切换为相同天气**：无效果，不重置任何状态

## Formulas

### F1. 地形伤害修正（骑兵卡）

```
FinalDamage = BaseDamage × TerrainMod

TerrainMod（骑兵卡）：
  平原 → ×1.5
  山地 → ×0.5
  其他 → ×1.0
```

### F2. 地形伤害修正（灼烧）

```
FinalBurnDamage = BurnDamage × TerrainMod

TerrainMod（灼烧）：
  森林 → ×1.5
  雪地 → ×0.5
  其他 → ×1.0
```

### F3. 天气灼烧消减（雨天）

```
BurnLayerDecay = 正常消减(1层) + 额外消减(1层) = 2层/回合
```

### F4. 沙漠骑兵卡费用修正

```
FinalCost = max(0, BaseCost - 1)    // 仅沙漠地形，仅骑兵卡
```

### F5. 天气风灼烧传播

```
每回合结束，对每个处于灼烧状态的单位：
  for each adjacent in AdjacentUnits(unit):
    if NOT adjacent.hasBurn:
      adjacent.applyBurn(1层)
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **天气切换为当前相同天气** | 无效果，不重置任何状态（如已有盲目不再叠加） |
| E2 | **雾天切换至其他天气** | 盲目层数不移除，正常倒计时消耗；雾的初始化效果不重复触发 |
| E3 | **水域/雪地/关隘的一次性效果，多阶段战斗** | 仅在战斗初始化时触发一次，阶段切换不重复触发 |
| E4 | **雾天后再切换回雾天** | 不重新施加盲目（同 E1，相同天气无效果） |
| E5 | **森林中毒 + 玩家已有中毒** | 中毒属同类状态，按刷新规则处理（取较高层数），不额外叠加 |
| E6 | **沙漠灼烧 + 雨天同时存在** | 两者均生效：每回合结束先施加1层灼烧（沙漠），再额外消减1层（雨天），净效果视当前层数而定 |
| E7 | **风天 + 场上只有1名敌人（无相邻单位）** | 风的传播不触发（无合法目标），效果自然跳过 |
| E8 | **天气切换冷却中再次触发切换** | 冷却期间该来源的切换无效；不同来源的切换不共享冷却 |
| E9 | **关隘护甲与其他初始护甲叠加** | 关隘护甲在初始化时直接添加，与其他来源护甲正常叠加（受护盾上限约束） |
| E10 | **骑兵卡费用 -1 后费用为 0** | 最低费用为 0，不产生负数行动点返还 |
| E11 | **地形修正与状态结算的叠加顺序** | 地形修正系数在 C2 出牌结算时作为乘数应用；C1 负责状态施加/倒计时；两者不冲突 |

## Dependencies

### 系统依赖

| 系统 | 方向 | 接口内容 |
|------|------|---------|
| 卡牌战斗系统（C2） | 读 | 出牌结算时查询当前地形/天气修正系数；天气切换后通知 C2 更新环境状态 |
| 状态效果系统（C1） | 写 | 地形/天气初始化时委托 C1 施加状态（滑倒/冻伤/中毒/盲目等） |
| 兵种卡系统（D2） | 读 | 兵种卡出牌时查询地形修正系数（骑兵卡平原/山地）和费用修正（骑兵卡沙漠） |
| 敌人系统（C3） | 读 | 敌人行动伤害结算时查询地形修正 |
| 地图节点系统（M1） | 读 | 节点初始化时提供地形/天气标签给本系统完成战斗初始化 |
| 事件系统（M4） | 写 | 地图事件可触发天气切换，调用本系统的天气切换接口 |

### 数据依赖

| 文件 | 用途 |
|------|------|
| `design/detail/hero_campaign_maps.csv` | 每个节点的地形/天气预设值 |
| `design/gdd/status-design.md` | 地形/天气触发的状态定义参考 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 平原骑兵伤害加成 | +50% | +30%~+70% | 骑兵卡在平原的价值强度 |
| 山地骑兵伤害折减 | -50% | -30%~-60% | 山地对骑兵卡的惩罚力度 |
| 森林灼烧伤害加成 | +50% | +30%~+70% | 灼烧流在森林的爆发上限 |
| 森林每回合中毒层数 | 1层 | 1~2层 | 森林持续压力强度 |
| 水域初始滑倒层数 | 1层 | 1~2层 | 水域开局控制压力 |
| 沙漠每回合灼烧层数 | 1层 | 1~2层 | 沙漠持续灼烧压力 |
| 沙漠骑兵费用减少 | -1 | -1（固定） | 沙漠骑兵流速度优势 |
| 关隘守城方初始护甲 | 10 | 6~15 | 关隘防守方初始优势强度 |
| 关隘守城方初始坚守层数 | 2层 | 1~3层 | 关隘节点难度 |
| 雪地初始冻伤层数 | 2层 | 1~3层 | 雪地开局压力 |
| 雪地灼烧伤害折减 | -50% | -30%~-60% | 雪地克制灼烧流的力度 |
| 雨天灼烧额外消减 | 1层/回合 | 1~2层 | 雨天削弱灼烧流的速度 |
| 天气切换冷却 | 2回合 | 2~3回合 | 天气反复横跳的频率上限 |

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 每个节点战斗开始时正确加载地形/天气标签并立即结算初始化效果 | 集成测试：进入水域节点，确认所有单位获得1层滑倒 |
| AC2 | 地形在战斗中固定不变，无任何途径可修改地形标签 | 单元测试：尝试切换地形，确认接口返回拒绝 |
| AC3 | 天气切换后新效果从下一次结算起生效，不追溯当前结算 | 单元测试：出牌中途切换天气，确认本次伤害用旧天气系数 |
| AC4 | 骑兵卡在平原伤害 ×1.5，在山地伤害 ×0.5，其他地形 ×1.0 | 单元测试：F1 公式三种地形各一个用例 |
| AC5 | 沙漠骑兵卡费用正确 -1，最低为 0 | 单元测试：费用1骑兵卡在沙漠显示费用0 |
| AC6 | 森林地形每回合结束正确施加1层中毒给所有单位 | 战斗测试：回合结束后检查所有单位状态列表 |
| AC7 | 雨天灼烧每回合消减2层（1正常+1额外） | 单元测试：有4层灼烧，1回合后剩2层 |
| AC8 | 风天灼烧传播：灼烧单位周围未灼烧单位获得1层灼烧 | 战斗测试：1v3场景，1名敌人灼烧，回合结束后确认相邻敌人获得灼烧 |
| AC9 | 关隘地形敌方初始护甲+10、坚守+2，仅触发一次 | 进入关隘节点测试；多阶段战斗确认第二阶段不重复触发 |
| AC10 | 天气切换为相同天气无效，不重置状态 | 单元测试：晴→晴，确认无任何状态变化 |
| AC11 | 地形/天气标签在节点信息界面对玩家可见（进入战斗前可查阅） | UI 测试：地图节点显示地形天气图标 |
