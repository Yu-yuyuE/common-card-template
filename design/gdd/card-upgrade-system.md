# M5 — 卡牌升级系统（Card Upgrade System）

_Created: 2026-04-07_

_Priority: 垂直切片 / 元游戏层_

## Overview

卡牌升级系统（Card Upgrade System）定义《三国称雄》中卡牌从 **Lv1 升级为 Lv2** 的触发入口、规则约束与收益标准。每张卡牌支持且仅支持**一次**升级，在商店节点内通过消耗金币完成。升级后的 Lv2 卡牌在本局游戏的所有后续战斗中保持 Lv2 状态。

**系统职责边界：**
- 负责：升级触发条件、升级次数约束（仅一次）、升级收益标准（单维度提升原则）、升级与离场机制的交互规则、Lv2 状态的持久化记录
- 不负责：商店节点的UI布局和价格定价（M2 商店系统）、卡牌的具体 Lv2 数值（`cards-design.md` 及各卡牌 CSV）、兵种卡升级（兵种卡遵循相同规则但在 D2 中定义）

## Player Fantasy

**"每张升级的卡牌都是一次专精承诺——我知道这局我要怎么打。"**

**① 构筑定型感（策略层）**
选择升级哪张卡是对当前构筑方向的表态：升级"骑兵"意味着平原地形主攻流；升级"火矢兵"意味着森林灼烧链。金币有限，升级即放弃其他可能，这种机会成本本身就是策略的一部分。

**② 进步可视感（成长层）**
升级后的卡牌在手牌中有明确的 Lv2 视觉标识，每次打出都能感受到"这张卡变强了"。相比开局时的 Lv1 卡组，战役后期手持数张 Lv2 卡的玩家感受到明显的成长曲线。

**③ 决策满足感（资源层）**
金币永远不够用（升级 vs 买新卡 vs 净化诅咒 vs 买装备），在商店节点的每一次分配都是有意义的权衡，做出正确决策后的满足感是元游戏的核心乐趣。

**不应有的感受：**
- 某张卡必须升级（每张卡的 Lv2 应该有适合的情境，而非绝对强于 Lv1）
- 升级总是次优选择（升级需要和买卡/净化真正竞争）
- Lv2 和 Lv1 效果基本一样感受不到差异（每次升级需要有清晰的感知提升）

## Detailed Rules

### 1. 升级基础规则

| 规则项 | 说明 |
|--------|------|
| **升级次数** | 每张卡牌仅支持且仅支持一次升级（Lv1 → Lv2），不可重复升级已是 Lv2 的卡 |
| **升级入口** | 商店节点内的"升级卡牌"功能；玩家选择卡组中任意一张 Lv1 卡付费升级 |
| **升级持久性** | 升级在整局游戏内持续有效（战役层状态），进入下一张地图后仍为 Lv2 |
| **升级目标** | 玩家可升级卡组中任意 Lv1 卡牌；诅咒卡**不可升级** |
| **兵种卡升级** | 兵种卡遵循相同升级规则（Lv1→Lv2，一次性），在军营节点可触发升级（详见 D2） |
| **武将专属卡升级** | 武将专属卡可以升级，遵循相同的单维度收益规则 |

---

### 2. 升级收益标准（设计约束，强制执行）

所有卡牌的 Lv2 效果必须遵循以下约束：

**A. 单维度提升原则（强制）**
Lv2 效果只能在**一个维度**上提升，禁止同时增益多个效果，且禁止新增 Lv1 没有的收益类别。

| 合规示例 | 违规示例 |
|---------|---------|
| Lv1护盾8 → Lv2护盾12（单纯数值提升）| Lv1护盾8 → Lv2护盾12+抽1张牌（新增收益类别）|
| Lv1伤害7 → Lv2伤害9（单纯数值提升）| Lv1伤害7 → Lv2伤害9+施加1层灼烧（新增状态）|
| Lv1施加2层灼烧 → Lv2施加3层灼烧（层数提升）| Lv1施加2层灼烧 → Lv2施加2层灼烧+2层中毒（新增状态类型）|

**B. 取消移除规则（强制）**
若 Lv1 有"使用后移除"→ Lv2 取消移除时，其他数值**不得**同时提升。取消移除本身即为升级收益（可重复使用是核心价值）。

| 情况 | 规则 |
|------|------|
| Lv1"使用后移除" → Lv2"不移除" | 其余数值/效果保持与 Lv1 完全一致 |
| Lv1 无移除 → Lv2 无移除 | 正常单维度数值提升 |

**C. 升级收益系数参考区间**

```
UpgradeGain = Lv2Power / Lv1Power

伤害/护甲/回复类：1.20~1.35（+20%~+35%）
状态层数类：+1层 或 延长1回合
费用折扣类：-1费
```

---

### 3. 商店节点升级流程

1. 玩家进入商店节点
2. 界面展示"升级卡牌"区域，列出玩家卡组中所有 Lv1 卡（诅咒卡不显示）
3. 每张可升级卡旁显示：当前 Lv1 效果 / 升级后 Lv2 效果 / 升级价格（金币）
4. 玩家选择一张并确认支付
5. 该卡立即变为 Lv2，本局游戏后续战斗中使用 Lv2 效果

> 商店节点每次访问可升级**多张**卡（金币足够即可），无每次访问升级数量上限。

## Formulas

### F1. 升级收益计算

```
// 伤害/护甲/回复类
Lv2Power = Lv1Power × UpgradeMultiplier
UpgradeMultiplier ∈ [1.20, 1.35]

// 示例：弓兵 Lv1 伤害7 → Lv2 全体伤害7（效果升级，非单纯数值）
// 示例：盾兵 Lv1 护盾8 → Lv2 护盾8+坚守（地形联动卡例外，按卡牌文本）
```

### F2. 升级成本参考（M2 定义权威值）

```
UpgradeCost_reference：
  普通卡（攻击/技能）：50~80 金
  兵种卡：60~100 金
  武将专属卡：80~120 金

// 以上为参考区间，M2 商店系统最终定义实际价格
```

### F3. Lv2 状态持久化

```
// 战役层记录
CardUpgradeRecord[cardId] = {
    isUpgraded: bool,
    upgradedInMap: mapIndex
}

// 战斗层读取
On BattleInit:
    ForEach card in PlayerDeck:
        If CardUpgradeRecord[card.id].isUpgraded:
            card.level = 2
            card.applyLv2Effects()
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **尝试升级已是 Lv2 的卡** | 升级按钮禁用/不显示该卡在可升级列表中 |
| E2 | **卡组中有多张同名卡（如3张骑兵）** | 每张卡牌独立记录升级状态；升级其中一张，另两张仍为 Lv1 |
| E3 | **升级武将专属卡** | 允许，遵循相同规则；专属卡的 Lv2 效果在 `heroes_exclusive_decks.csv` 中定义 |
| E4 | **诅咒卡在升级列表中出现** | 不允许；诅咒卡在商店升级界面中不显示 |
| E5 | **金币不足以支付升级费用** | 该卡升级按钮置灰，显示所需金币；不允许部分支付 |
| E6 | **升级后离开商店再进入** | 已升级的卡保持 Lv2 状态，商店列表不再显示该卡可升级 |
| E7 | **Lv2 卡牌被净化（移出卡组）后** | 该卡的升级记录随卡牌一起失效（移出卡组=本局不再存在）|
| E8 | **Lv1 有"使用后移除"→ Lv2 取消移除，但 Lv2 数值意外高于 Lv1** | 违规，需修正；取消移除时其余效果必须与 Lv1 完全一致 |
| E9 | **同一张卡在一次商店访问中被升级两次** | 第一次升级后立即变为 Lv2，后续不可再次选择升级该卡 |
| E10 | **兵种卡在商店节点和军营节点都可升级** | 允许，两个入口均可升级；升级记录共用（一张牌无论在哪里升级都记录 Lv2）|

## Dependencies

### 系统依赖

| 系统 | 方向 | 接口内容 |
|------|------|---------|
| 商店系统（M2） | 读 | M2 商店节点提供升级 UI 入口，调用 M5 的升级接口；升级价格由 M2 定义 |
| 卡牌战斗系统（C2） | 写 | 战斗初始化时 M5 的升级记录驱动卡牌加载 Lv2 效果；C2 战斗引擎使用升级后的效果数值 |
| 地图节点系统（M1） | 读 | 商店节点（M1 的节点类型之一）是升级的触发场景；M5 不感知节点本身，只提供升级接口 |
| 兵种卡系统（D2） | 弱依赖 | 兵种卡升级遵循相同规则；军营节点也可触发兵种卡升级（与商店节点共用 M5 接口）|
| 存档持久化系统（F1） | 写 | Lv2 升级状态作为战役进度的一部分，需由 F1 持久化存储 |

### 数据依赖

| 文件 | 用途 |
|------|------|
| `design/gdd/cards-design.md` | 升级规则的来源文档（3.0 节）；本文档为其权威展开 |
| `design/detail/all_cards/attack_cards.csv` | 攻击卡 Lv2 效果数值 |
| `design/detail/all_cards/skill_cards.csv` | 技能卡 Lv2 效果数值 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 普通卡升级成本 | 50~80 金 | 40~100 金 | 升级决策频率；过低则玩家总是升级，失去取舍 |
| 兵种卡升级成本 | 60~100 金 | 50~120 金 | 兵种流构筑的升级投入门槛 |
| 武将专属卡升级成本 | 80~120 金 | 60~150 金 | 专属流的升级投入是否值得 |
| 升级收益系数（伤害类）| 1.20~1.35 | 1.15~1.40 | 升级后感知提升强度；过低则升级无感 |
| 升级收益（状态层数类）| +1层 | +1~+2层 | 状态流升级后的控制/DoT强度提升 |
| 升级收益（费用折扣类）| -1费 | 固定 -1费 | 高费卡升级后的出牌效率提升 |
| 每次商店可升级数量上限 | 无限制（金币足够即可）| 无限/或 1~3 | 单次商店的升级集中度 |

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 每张卡牌仅可升级一次，已是 Lv2 的卡不出现在可升级列表 | UI测试：升级一张卡后，该卡在商店升级列表中不再显示 |
| AC2 | 诅咒卡不出现在升级列表中 | UI测试：卡组含诅咒卡时，商店升级界面不显示诅咒卡 |
| AC3 | 支付金币后卡牌立即升级为 Lv2，当场战斗（下次战斗起）使用 Lv2 效果 | 战役测试：商店升级骑兵后，下一场战斗打出骑兵确认效果为 Lv2（击退+眩晕）|
| AC4 | 升级状态在整局游戏内持续，进入下一张地图后仍为 Lv2 | 战役测试：第1张地图升级的卡，进入第2张地图后仍为 Lv2 |
| AC5 | Lv2 效果只在一个维度提升，无新增收益类别 | 设计审查：逐一检查所有卡牌 Lv2 效果文本，确认无违规双重收益 |
| AC6 | 取消"使用后移除"的升级收益，Lv1/Lv2 数值完全一致 | 数据验证：遍历所有 Lv1 有移除属性的卡，确认 Lv2 其他数值不变 |
| AC7 | 金币不足时升级按钮置灰，显示所需金币 | UI测试：金币=30，升级需80金时，升级按钮不可点击并显示"需要80金" |
| AC8 | 卡组中多张同名卡各自独立记录升级状态 | 功能测试：2张骑兵，升级1张后：1张Lv2、1张Lv1，在战斗中各自效果不同 |
| AC9 | 升级后的卡牌有明确的 Lv2 视觉标识（角标/边框等）| UI测试：Lv2 卡牌在手牌、卡组浏览界面中有明显 Lv2 标识 |
| AC10 | 兵种卡在军营节点和商店节点均可升级，共用升级记录 | 功能测试：在军营节点升级骑兵后，商店节点的升级列表中该骑兵不再显示 |
