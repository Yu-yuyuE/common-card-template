# 卡牌设计文档（Cards Design）

_Updated: 2026-03-27_
_Status: Draft v2（负面卡增强 + 施咒/转收益扩展）_

---

## Overview

常规卡规则：

- 与武将专属卡分离
- 不设稀有度
- 玩家自由组卡
- 每局结束常规卡3选1

本版调整：

- 提升负面卡占比与数量
- 增强“对敌施加负面”与“负面转收益”体系

---

## Player Fantasy

玩家可构筑：

- 常规稳定流
- 负面控制流（给敌人上负面）
- 负面经营流（自己吃负面换收益）

## Detailed Rules

### 1) 卡池规模与类别（目标 330）

- 近战攻击卡：50
- 远程攻击卡：50
- 技能卡：175（含防御/战术/兵种卡协同/地形操控等）
- 诅咒卡：55

合计：330

### 2) 常规卡基础设计（补回）

#### 2.0 升级与离场机制（新增）

- 每种卡牌可升级1次（Lv1 → Lv2），不可重复升级。
- 升级后保留原卡身份与标签，仅提升数值或追加1条轻量效果（不改变核心定位）或-1费或使用后移除效果取消。
- 升级收益建议：
  - 伤害/护甲类：+20% ~ +35%
  - 施加层数类：+1层或延长1回合

离场机制分为两类：

A. **使用后移除（战斗移除）**

- 卡牌打出后从本场战斗**移除**（不在抽牌堆/弃牌堆中）。
- 该卡在本局游戏结束后自动回到玩家卡组。
- 用于高爆发或高功能卡，防止单场循环滥用。

B. **消耗卡（地图一次）**

- 卡牌在当前地图（M1~M5）内仅可成功使用1次。
- 一旦使用，标记为“已消耗”，本地图后续战斗不可再使用。
- 进入下一张地图时重置可用次数。

追踪要求：

- 战斗内需记录“战斗移除”状态。
- 战役层需记录“地图消耗”状态（按卡ID+地图ID）。

#### 2.1 攻击卡子类型与目标规则

- 攻击卡分为：**近战** / **远程**。
- **近战攻击卡**：
  - 可出现0费卡；
  - 默认对目标敌将造成伤害，优先结算其附属兵种层（兵种层清空后才命中敌将本体）；
  - 带”直击主将”关键词时可直接命中敌将。
- **远程攻击卡**：
  - 可指定敌方可见主将目标；默认先结算其附属兵种层，特殊文本可直击主将；
  - 默认不允许0费（若后续出现，需在卡面与平衡表单独标注）。

#### 2.2 常规卡通用字段

每张常规卡至少包含：

- 卡名
- 等级（Lv1/Lv2）
- 费用
- 类型（近战攻击/远程攻击/技能/兵种/诅咒）
- 目标规则（主将/全体/兵种层/任意）
- 基础效果值（伤害/护甲/层数/抽牌）
- 离场机制（常规/使用后移除/消耗卡）
- 关键标签（近战/远程/召唤/净化/施咒等）

#### 2.3 常规卡模板（示例）

A. 近战攻击卡（示例8）

1. 破阵斩（0费）：对目标敌将造成小额伤害（优先结算其兵种层）。
2. 追身突刺（1费）：对目标造成伤害，若目标有负面状态则额外+伤。
3. 横扫千军（2费）：对敌方所有主将造成群体伤害（各自优先结算兵种层）。
4. 破甲重击（2费）：对目标造成伤害并附破甲。
5. 冲锋陷阵（1费）：对目标造成伤害并冲锋至其身后。
6. 致命一击（2费）：对目标造成高额伤害。
7. 连环刀（2费）：对目标造成3次小额伤害。
8. 吸血斩（3费）：对目标造成伤害并恢复等量生命。

B. 远程攻击卡（示例8）

1. 穿云箭（1费）：对任意敌方主将目标造成伤害（优先结算其兵种层）。
2. 连珠箭（2费）：随机两次命中敌方可见目标。
3. 定点狙袭（3费）：指定敌将造成高伤（直击主将）。
4. 火矢齐发（3费）：敌方全体主将目标小伤并附1层灼烧。
5. 散射（2费）：对敌方全体造成小额伤害。
6. 毒箭（2费）：对目标造成伤害并附加3层中毒。
7. 冰霜箭（2费）：对目标造成伤害并附加2层迟缓。
8. 雷击（3费）：对目标造成高额伤害并附加1层脆弱。

C. 技能卡（示例10）

1. 举盾（1费）：获得护甲。
2. 守势反击（1费）：获得少量护甲，若本回合受击则反伤。
3. 军阵固守（2费）：我方全体获得护甲。
4. 铁壁（2费）：获得高额护甲并免疫一次位移。
5. 调度（1费）：抽1并重置1张低费卡。
6. 诈降（2费）：目标攻击下降1回合。
7. 佯攻转实（2费）：本回合下一张攻击卡伤害提升。
8. 号令齐发（2费）：本回合打出的兵种卡与武将攻击获得协同增伤。
9. 扬沙（1费）：地形转为沙地并降低命中。
10. 火线蔓延（2费）：地形转为火地并附持续灼烧。

D. 诅咒卡（示例4）

1. 毒药（1费）：使目标中毒3层。
2. 虚弱（1费）：使目标虚弱2层。
3. 混乱（2费）：使目标混乱2层。
4. 迟缓（1费）：使目标迟缓2层。

### 3) 诅咒卡核心规则

负面卡分为三种类型：

| 类型           | 触发时机                             | 持续方式                                     | 示例                              |
| -------------- | ------------------------------------ | -------------------------------------------- | --------------------------------- |
| **抽到触发型** | 被抽入手牌时立即触发，触发后正常丢弃 | 单次触发，丢弃即加入弃牌堆                   | 毒药（抽到→扣2血）                |
| **常驻牌库型** | 存在于牌库中即持续生效               | **直到被净化操作移出卡组才消除，丢弃不消除** | 韬晦（在库→最大生命-3）           |
| **常驻手牌型** | 进入手牌后持续占位                   | 需支付指定费用才能丢弃，否则常驻占用手牌位   | 泥泞（需1费丢弃，否则锁住手牌格） |

通用规则：

- 负面卡可来自事件、敌人、特定卡牌或武将被动。
- 负面卡**无法主动打出**，**不可升级**。
- 负面卡仅可通过：商店净化 / 技能净化 / 特定卡牌净化，操作结果为**移出卡组**。

### 4) 诅咒玩法分支

A. 对敌施咒：给敌方施加中毒、虚弱、混乱、迟缓等。

B. 自污经营：主动将负面塞入自身牌库，换取费用/抽牌/增伤收益。

## Formulas

### 近战/远程目标合法性

`TargetLegal = (CardRangeType == Melee && Target == EnemyFrontFirst) || (CardRangeType == Ranged && Target ∈ EnemyVisibleSet)`

- `EnemyVisibleSet`：敌方当前可被选中的全部主将目标（各目标携带附属兵种层，伤害优先结算兵种层）

### 近战0费强度约束

`ZeroCostMeleeBudget = ZeroCostMeleeDamage / AvgTurnDamage`

建议安全区：`0.10 ~ 0.18`

### 升级收益系数

`UpgradeGain = Lv2Power / Lv1Power`

建议区间：`1.20 ~ 1.35`

### 地图消耗卡可用性

`MapConsumableUsable = !UsedOnMap(cardId, mapId)`

- 当 `MapConsumableUsable = false` 时，卡牌在当前地图不可再打出。

### 负面收益评分

`CurseValue = (PenaltyScore × -1) + (SynergyBonus + ConversionGain)`

用于平衡：确保负面流不是无脑最优。

### 施咒覆盖率

`DebuffCoverage = DebuffCardsPlayed / TotalTurns`

目标建议：0.35~0.55。

## Edge Cases

1. 负面卡被复制：复制体保留“负面”标签。
2. 敌方免疫某类负面：施加失败但触发“施咒尝试”类被动。
3. 同回合既净化（移出卡组）又利用负面：先移出卡组后计算”持有量”收益。
4. 负面卡来源不明：日志需标明来源（事件/敌方/自加）。
5. 升级卡被复制：复制体继承当前等级，但不额外触发升级次数。
6. 使用后移除卡被回收效果影响：若文本未明确“可回收移除卡”，则不可被回收。
7. 消耗卡在同图多战斗中：首次成功打出后锁定，直到切换下一张地图才解锁。

## Tuning Knobs

- 负面卡总量：45~70
- 每局新增负面来源数：2~8
- 净化价格：中高（防止白给）
- 负面转收益倍率：0.8~1.4
- 升级收益系数：1.20~1.35
- 使用后移除卡占比：8%~18%
- 消耗卡占比：3%~8%

---

## Acceptance Criteria

1. 卡池总量维持300+（目标330）。
2. 负面卡数量显著提升（≥45）。
3. 具备“施咒敌方”与“负面转收益”两套可构筑框架。
4. 负面卡仍可被商店/技能/特效净化，规则一致。
5. 每种卡牌均支持且仅支持一次升级（Lv1→Lv2）。
6. “使用后移除”卡在战斗内移除、局结束回卡组。
7. “消耗卡”在单张地图内仅可成功使用一次，跨地图重置。
