## Overview

军营系统（Barracks System）定义《三国称雄》中**军营节点**的全部操作逻辑与规则约束。军营节点是玩家管理兵种卡的专属场所，提供三类功能：**添加兵种卡**（从候选列表中选择入组）、**升级兵种卡**（Lv1→Lv2，以及触发Lv2→Lv3分支选择）、**移出卡牌**（从卡组中移出任意卡牌，包括非兵种卡）。金币是兵种卡升级的唯一消耗资源（用于军队操练），与地图粮草完全独立。

**系统职责边界：**

- **负责**：候选兵种卡的生成与权重计算、添加/升级/移出操作的执行与约束、Lv2→Lv3分支选择的UI触发与确认流程、金币升级费用的扣除
- **不负责**：兵种卡自身的战斗效果与数值（D2）、Lv3分支卡的具体效果定义（D2）、金币资源的存储与跨战持久化（F2）、武将统帅值的定义（D3）、卡牌Lv1→Lv2升级的通用规则（M5）、地图节点的导航与路径逻辑（M1）

## Player Fantasy

**"这是我的军队——我来决定它的构成。"**

**① 编制自主感（策略层）**
军营节点是玩家最直接控制卡组构成的时刻。添加哪种兵种、移出哪张多余的卡、决定是否将步兵升级为虎贲卫——每个决策都是对下一场战斗的提前布局。军营让玩家感受到"将领调兵遣将"的主导感，而不是被动接受随机卡牌。

**② 专精成就感（成长层）**
Lv2→Lv3分支升级是军营独有的仪式感时刻。从5~6个特殊精锐中选定一个方向，代表着对这场战役核心战术的最终承诺。虎豹骑还是陌刀骑？刺客还是舞女？选择的那一刻，军队有了真正的"个性"。

**③ 卡组整洁感（效率层）**
移出功能让玩家可以清理前期加入的过渡卡、不适合当前流派的卡，甚至战略性地在军营主动清除某张诅咒卡（虽然代价高昂）。一套精炼的卡组比一套臃肿的卡组更有掌控感。

**不应有的感受：**

- 候选兵种卡与当前武将流派完全无关（武将倾向权重应确保出现匹配兵种的概率更高）
- 粮草消耗过高导致升级不起（粮草升级应与地图粮草收益保持可行性）
- 移出功能让玩家感到"被迫瘦身"而非"主动优化"

## Detailed Rules

### 1. 进入流程

```
玩家移动至军营节点
  → 扣除移动粮草消耗（由地图节点系统执行）
  → 显示军营主界面（三个功能区并列展示）
  → 玩家自由操作（任意顺序、可多次执行，资源足够即可）
  → 玩家点击"离开军营" → 本次访问结束，保存所有变更
```

### 2. 功能A：添加兵种卡

#### 2a. 候选卡生成规则

| 规则项       | 说明                                                                   |
| ------------ | ---------------------------------------------------------------------- |
| **候选数量** | 每次访问展示3张候选兵种卡                                              |
| **等级分布** | 通常为Lv1（约85%），偶尔出现Lv2（约15%）                               |
| **权重来源** | 受武将兵种倾向权重影响（D3定义），倾向步兵的武将更高概率出现步兵系候选 |
| **去重规则** | 3张候选卡不重复（卡牌ID唯一）                                          |
| **固定时机** | 进入军营节点时生成，同一次访问内固定，不自动刷新                       |

#### 2b. 添加规则

| 规则项         | 说明                                                           |
| -------------- | -------------------------------------------------------------- |
| **添加费用**   | 免费（兵种卡通过军营加入，不消耗金币或粮草）                   |
| **统帅限制**   | 卡组中兵种卡总数 ≤ 武将统帅值（3~6），达到上限时添加按钮灰显   |
| **无数量限制** | 单次访问可添加多张（只要统帅值允许）                           |
| **立即生效**   | 添加后立即进入卡组，本次访问剩余操作中即可在移出功能中看到该卡 |

### 3. 功能B：升级兵种卡

#### 3a. Lv1→Lv2 升级

| 规则项           | 说明                                                     |
| ---------------- | -------------------------------------------------------- |
| **触发条件**     | 玩家卡组中存在Lv1兵种卡                                  |
| **费用**         | 消耗粮草（见Formulas F1）                                |
| **效果**         | 兵种卡升级为高级兵种（效果见D2 troop-cards-design.md）   |
| **每次访问次数** | 不限（粮草足够即可）                                     |
| **界面**         | 展示卡组中所有Lv1兵种卡，每张旁显示Lv2效果预览和粮草费用 |

#### 3b. Lv2→Lv3 分支选择

| 规则项           | 说明                                                          |
| ---------------- | ------------------------------------------------------------- |
| **触发条件**     | 玩家卡组中存在Lv2兵种卡                                       |
| **费用**         | 消耗粮草（见Formulas F2）                                     |
| **效果**         | 从当前Lv2兵种对应的5~6个Lv3特殊精锐分支中选择1个，替换该Lv2卡 |
| **分支不可撤销** | 选定Lv3分支后本局不可更改，可选择"暂不升级"留到下次军营节点   |
| **界面**         | 对每张Lv2卡展示全部Lv3分支卡预览（效果、费用、地形联动）      |

**各兵种Lv3分支列表：**

| 基础兵种 | Lv3分支（5个）                                           |
| -------- | -------------------------------------------------------- |
| 步兵     | 虎贲卫 / 刀盾手 / 戟兵 / 山地步兵 / 游侠                 |
| 骑兵     | 轻骑兵 / 火骑兵 / 象兵 / 虎豹骑 / 陌刀骑                 |
| 弓兵     | 连弩兵 / 火矢兵 / 投石兵 / 弩车兵 / 火兵 / 猎人（共6个） |
| 谋士     | 细作 / 方士 / 刺客 / 舞女 / 黄巾信徒                     |
| 盾兵     | 重甲盾兵 / 水兵 / 壁垒工兵 / 藤甲兵 / 铁甲步兵           |

### 4. 功能C：移出卡牌

| 规则项           | 说明                                                               |
| ---------------- | ------------------------------------------------------------------ |
| **可移出范围**   | 卡组中**任意**卡牌：攻击卡、技能卡、兵种卡、武将专属卡、诅咒卡     |
| **费用**         | 无费用，但移出后该卡**本局永久消失**                               |
| **诅咒卡移出**   | 诅咒卡视为普通卡处理，直接移出即为"净化"；无需额外付费，无次数限制 |
| **界面**         | 展示完整卡组列表（含诅咒卡），可滚动选择；移出前弹出确认窗口       |
| **不可移出**     | 无限制（任何卡均可移出，含武将专属卡）                             |
| **每次访问次数** | 不限                                                               |

## Formulas

### F1. Lv1→Lv2 升级金币费用

```
UpgradeCost_Lv1toLv2 = BaseUpgradeCost_Lv2

变量：
  BaseUpgradeCost_Lv2 = 50（金币）

示例：
  玩家有3张Lv1兵种卡，依次升级：每次消耗50金币，3次共消耗150金币
```

### F2. Lv2→Lv3 分支升级金币费用

```
UpgradeCost_Lv2toLv3 = BaseUpgradeCost_Lv3

变量：
  BaseUpgradeCost_Lv3 = 50（金币）

设计意图：
  Lv3升级费用高于Lv2，强化分支选择的决策重量。
  单张地图平均金币收益约400（见shop-system.md F6参考区间），
  Lv3升级消耗约12.5%的单图收益，属于有意义但不过度的消耗。
```

### F3. 候选兵种卡权重计算

```
CardWeight(card) = BaseWeight × HeroAffinityMult(card.type)

变量：
  BaseWeight          = 1.0（各兵种基础权重相等）
  HeroAffinityMult    = 武将对该兵种类型的倾向系数（由D3武将系统定义）
                        高倾向：2.0 / 中倾向：1.0 / 低倾向：0.5

抽取流程：
  候选池 = 全部可用兵种卡（排除统帅已满时的超限情况）
  按CardWeight归一化概率，无放回抽取3张（不重复）

示例：
  曹操（步兵高倾向×2.0，其余×1.0）：
  步兵系出现概率显著提升，候选中约40~50%为步兵系卡牌
```

### F4. Lv2候选卡出现概率

```
Lv2Chance(slot) = BaseLv2Chance

变量：
  BaseLv2Chance = 15%（每张候选卡独立判定）

示例：
  3张候选卡，每张各15%概率为Lv2 → 平均每次军营访问约0.45张Lv2候选卡出现
```

## Edge Cases

| #   | 边界情况                                                  | 裁定规则                                                                                             |
| --- | --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| E1  | **统帅值已满（如统帅=3，卡组已有3张兵种卡），仍进入军营** | 添加功能中所有候选卡按钮灰显，显示"兵种卡已达统帅上限（3/3）"；升级和移出功能正常可用                |
| E2  | **金币不足以支付Lv1→Lv2升级费用（<50金币）**              | 对应升级按钮置灰，显示"金币不足（需50，当前X）"；不允许升级                                          |
| E3  | **金币不足以支付Lv2→Lv3升级费用（<50金币）**              | 分支选择界面中确认按钮置灰；可浏览分支预览但不可确认；可选择"暂不升级"                               |
| E4  | **Lv2→Lv3分支选择后后悔（同一军营节点内）**               | 不可撤销；选定后立即生效，Lv2卡替换为选定的Lv3卡；设计意图：分支承诺是核心策略张力                   |
| E5  | **移出卡牌后卡组只剩1张卡**                               | 允许移出（无最小卡组限制）；UI警告"当前卡组仅剩X张卡，继续移出将影响战斗"；玩家自行决策              |
| E6  | **移出武将专属卡**                                        | 允许，弹出额外警告"武将专属卡移出后本局不可恢复，确认？"；确认后移出                                 |
| E7  | **候选兵种卡出现玩家已拥有的同名卡**                      | 允许选择（卡组可有多张同名兵种卡，受统帅值约束）；UI显示"已持有N张"提示                              |
| E8  | **卡组中兵种卡全部为Lv3（无Lv1/Lv2可升级）**              | 升级功能区显示"所有兵种卡已达最高级"；移出和添加功能正常可用                                         |
| E9  | **同一次访问内先添加一张Lv1兵种卡再立即升级**             | 允许；添加后该卡立即进入卡组，升级功能可见并可对其操作；金币扣除正常                                 |
| E10 | **弓兵Lv3有6个分支（非标准5个）**                         | 正常展示全部6个分支预览；玩家从6选1；不产生UI异常                                                    |
| E11 | **移出所有兵种卡后统帅槽清空，再添加兵种卡**              | 允许；统帅值重新计算，移出后槽位腾空，可正常添加                                                     |
| E12 | **进入军营节点时存档，操作中途退出**                      | 存档在"离开军营"时写入；中途退出读取本次军营进入前的状态；所有操作需重新执行（类似战斗中途退出规则） |

## Dependencies

### 系统依赖

| 系统                     | 方向         | 接口内容                                                                                                                 |
| ------------------------ | ------------ | ------------------------------------------------------------------------------------------------------------------------ |
| **兵种卡系统（D2）**     | 读           | 候选卡从D2兵种卡池抽取；Lv1→Lv2效果、Lv2→Lv3分支列表与效果均由D2定义；本系统调用D2数据展示                               |
| **武将系统（D3）**       | 读           | 统帅值（卡组兵种卡上限）由D3提供；武将兵种倾向权重（HeroAffinityMult）由D3定义，本系统用于候选卡权重计算                 |
| **资源管理系统（F2）**   | 读/写        | 金币当前值由F2管理；Lv1→Lv2和Lv2→Lv3升级操作通过F2扣除金币；F2负责金币值的合法性校验（不低于0）                          |
| **卡牌战斗系统（C2）**   | 读/写        | 卡组的增删（添加/移出兵种卡）通过C2管理的卡组接口执行；升级后的卡牌状态更新到C2卡组记录                                  |
| **卡牌升级系统（M5）**   | 弱依赖       | 军营节点的Lv1→Lv2升级遵循M5定义的通用升级规则（单维度提升原则）；Lv2→Lv3分支不经过M5（分支选择是军营独有逻辑，M5不感知） |
| **地图节点系统（M1）**   | 读（被触发） | M1检测玩家进入军营节点后触发本系统；本系统向M1注册"军营操作完成"事件                                                     |
| **存档持久化系统（F1）** | 写           | 军营操作结果（卡组变更、升级状态）在"离开军营"时通过F1持久化；跨地图保留                                                 |

## Tuning Knobs

| 调节项                 | 当前值 | 安全区间  | 影响维度                                                |
| ---------------------- | ------ | --------- | ------------------------------------------------------- |
| 候选兵种卡数量         | 3张    | 2~4张     | 选择宽度；过少则无选择感，过多则统帅限制失去意义        |
| 候选Lv2出现概率        | 15%    | 10%~25%   | Lv2候选的惊喜频率；过高则Lv1→Lv2路径被架空              |
| Lv1→Lv2升级金币费用    | 50金币 | 40~60金币 | 升级频率；过低则无决策感，过高则升级路径形同虚设        |
| Lv2→Lv3升级金币费用    | 50金币 | 30~80金币 | 分支选择的稀缺性；过低则无承诺感，过高则玩家无法触达Lv3 |
| 武将倾向系数（高倾向） | 2.0×   | 1.5×~3.0× | 武将差异化的强度；过高则候选卡单一，过低则武将特色消失  |
| 武将倾向系数（低倾向） | 0.5×   | 0.2×~0.8× | 非主流兵种的出现率；过低则某些兵种永远不出现            |

## Acceptance Criteria

| #    | 验收条件                                                                       | 验证方式                                                                                   |
| ---- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------ |
| AC1  | 进入军营节点时，展示3张候选兵种卡，卡牌ID不重复                                | 功能测试：访问军营100次，统计同一次候选中出现重复卡的次数（应为0）                         |
| AC2  | 统帅值已满时，添加按钮全部灰显，无法点击                                       | UI测试：统帅=3，卡组3张兵种卡时，确认3个候选卡按钮均不可点击                               |
| AC3  | Lv1兵种卡升级消耗50金币，升级后卡牌变为Lv2效果                                 | 功能测试：升级步兵（Lv1→Lv2），确认金币-50，卡组中该步兵变为"高级步兵"效果                 |
| AC4  | Lv2兵种卡展示5~6个Lv3分支预览，选择分支后消耗50金币，Lv2卡替换为选定Lv3卡      | 功能测试：对高级步兵选择"虎贲卫"，确认金币-50，卡组中高级步兵变为虎贲卫                    |
| AC5  | Lv3分支选定后不可撤销，同一军营访问内无"取消"选项                              | UI测试：确认Lv3分支选择弹窗无撤销按钮；确认后卡牌立即替换                                  |
| AC6  | 移出功能可移出卡组中任意卡牌（含武将专属卡/诅咒卡），移出前弹出确认窗口        | 功能测试：分别移出攻击卡、诅咒卡、武将专属卡，确认各自触发不同确认文本且移出后卡组正确更新 |
| AC7  | 军营内所有操作（添加/升级/移出）在"离开军营"后通过F1持久化，存档重载后状态一致 | 存档测试：军营内升级1张卡后强制退出，重载存档后确认该卡仍为Lv1（中途退出不保存）           |
| AC8  | 候选卡权重受武将倾向影响，高倾向兵种出现频率明显高于低倾向                     | 统计测试：用曹操（步兵高倾向）访问军营200次，步兵系卡出现率应在35%~55%之间                 |
| AC9  | Lv2候选卡出现概率约15%，每张候选位独立判定                                     | 统计测试：记录1000次候选卡生成，Lv2出现率在10%~20%之间                                     |
| AC10 | 金币不足时升级按钮置灰，显示"金币不足（需X，当前Y）"，不允许执行升级           | UI测试：金币=10，尝试点击Lv1→Lv2升级（需50金币），确认按钮不可点击且显示正确提示           |
