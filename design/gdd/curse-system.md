## Overview

诅咒系统（Curse System）定义《三国称雄》中**诅咒卡**的类型规则、来源机制、净化流程，以及与司马懿被动的事件钩子接口。诅咒卡是对玩家不利的卡牌，通过事件/敌人行动/特定卡牌被强制注入玩家卡组，是负面经营玩法的核心机制。

诅咒卡分为3种类型：**抽到触发型**（抽入手牌立即触发）、**常驻牌库型**（在库持续生效）、**常驻手牌型**（占用手牌位，需付费才能丢弃）。

**系统职责边界：**
- 负责：3种诅咒类型的触发/持续/消除规则、诅咒卡注入来源接口、净化操作的执行定义（移出卡组）、`OnCurseCardDrawn` 事件钩子（供司马懿被动监听）
- 不负责：净化的具体费用/价格（商店系统 M2 定义）、司马懿被动的隐忍层数计算（武将系统 D3）、诅咒卡的具体卡面数值设计（`cards-design.md` 及 `curse_cards.csv`）

## Player Fantasy

**"诅咒卡是敌人的武器，但有人能把它变成自己的弹药。"**

**① 威胁与压力感（普通玩家视角）**
诅咒卡出现在手牌或牌库中是持续的战略威胁——"韬晦不净化，最大HP会一直往下掉"；"泥泞锁住了一个手牌位，我这回合少打一张"。净化节点的资源决策（净化 vs 买卡 vs 回血）是高压策略的核心张力，每一个金币都是机会成本。

**② 资产化与逆向心理（司马懿隐忍流）**
司马懿把诅咒卡视为"隐忍层数的充能燃料"——每次诅咒卡进入手牌，都在积累爆发所需的隐忍层数。玩家在商店节点主动**不净化**（甚至可能通过特定卡牌主动获取诅咒卡），是一种高阶策略选择。诅咒从威胁变成了资产，这种反转感是司马懿流的核心乐趣。

**不应有的感受：**
- 诅咒卡是纯粹的惩罚，完全没有利用可能（司马懿必须感受到反转感）
- 净化永远是最优解（应该存在"不净化"价值更高的情境）
- 诅咒卡太少，几乎感受不到压力（基础图鉴25张诅咒卡需要覆盖不同危害程度）

## Detailed Rules

### 1. 三种诅咒类型

| 类型 | 触发时机 | 持续方式 | 消除方式 | 示例 |
|------|---------|---------|---------|------|
| **抽到触发型** | 被抽入手牌时立即触发效果，触发后正常丢弃 | 每次被抽到都触发（循环中持续存在） | 净化（移出卡组）| 毒药（抽到→扣2HP）|
| **常驻牌库型** | 存在于牌库/弃牌堆中即持续生效（不需要被抽到）| 在本局流通范围内（牌库+弃牌堆）持续生效 | **仅净化**（丢弃不消除，仍持续生效）| 韬晦（在库→最大HP-3）|
| **常驻手牌型** | 被抽入手牌后持续占位，需支付费用才能丢弃 | 常驻占用手牌位，不支付则无法清除 | 支付丢弃费用（丢至弃牌堆，下局仍在）或净化（移出卡组）| 泥泞（需1费丢弃）|

**通用规则：**
- 诅咒卡**无法主动打出**（不同于支付费用的"丢弃"操作），**不可升级**
- 战斗结束时，诅咒卡进入弃牌堆，下场战斗重新洗牌后仍在卡组中循环
- 诅咒卡卡面标有 `[诅咒]` 标签，UI中以红色边框区分

---

### 2. 诅咒卡注入来源

| 来源类型 | 时机 | 注入位置 | 典型数量 |
|---------|------|---------|---------|
| 敌人行动（B/C类行动） | 敌人出牌阶段结算 | 玩家弃牌堆（下次洗牌时进入牌库）| 1~2张 |
| 地图事件（负面事件选项）| 进入事件节点并选择/触发对应结果 | 直接注入玩家牌库 | 1~3张 |
| 特定卡牌效果 | 玩家/敌方打出带有"注入诅咒"效果的卡时 | 按卡牌文本：弃牌堆或手牌 | 1张 |
| 司马懿初始卡组 | 游戏开始时 | 牌库（初始配置2张韬晦）| 2张（固定）|

> 注：司马懿的初始韬晦诅咒卡是**牌库预置**，不属于运行期"注入"；其他来源才走 `OnCurseInjected` 事件钩子。

---

### 3. 净化规则

**净化**（Purify）= 将诅咒卡从玩家卡组**永久移出**（本局游戏内不再出现）。

术语规范：
- **净化**是操作名称
- **移出卡组**是净化的结果（详见 CLAUDE.md 卡牌术语标准）
- 不得用"删除"、"消除"代替"净化"

| 净化来源 | 时机 | 能否指定 | 限制 |
|---------|------|---------|------|
| 商店节点净化 | 战役层节点 | 可指定具体诅咒卡 | 消耗金币（价格由 M2 商店系统定义）|
| 技能卡净化效果 | 战斗中打出对应技能卡 | 按技能文本（指定/随机）| 当回合消耗费用 |
| 特定事件奖励 | 事件节点完成 | 按事件文本 | 无额外条件 |

**净化结果**：目标诅咒卡进入"移出卡组"状态，本局剩余所有战斗不再出现于牌库/弃牌堆/手牌中。

---

### 4. 诅咒卡事件钩子（系统对外接口）

D4 系统需向外暴露以下事件，供其他系统监听：

| 事件名 | 触发时机 | 当前监听方 | 备注 |
|--------|---------|---------|------|
| `OnCurseCardDrawn` | 任意诅咒卡被抽入手牌时（三种类型均触发）| 司马懿被动（增加隐忍层数）| 每次抽到均触发，包括重复被抽到 |
| `OnCurseInjected` | 诅咒卡被运行期注入玩家卡组时 | 无（预留，供未来系统扩展）| 初始配置预置不触发此事件 |
| `OnCursePurified` | 诅咒卡被净化移出时 | 无（预留）| — |

## Formulas

### F1. 常驻牌库型持续效果计算

```
// 每回合开始时触发一次，统计牌库+弃牌堆中的常驻牌库型诅咒卡数量
ActivePersistentCurses = COUNT(CurseCards WHERE Type = PersistentLibrary
                               AND Location IN {Deck, DiscardPile})

// 示例：韬晦
MaxHP_effective = MaxHP_base - (3 × COUNT(韬晦 in {Deck ∪ DiscardPile}))

// 净化后立即重新计算（净化当回合生效）
```

> 常驻牌库型诅咒卡被抽入手牌时效果不额外触发（区别于抽到触发型）。
> 该卡在手牌中时，仍计入 `ActivePersistentCurses`（在手牌 = 仍在本局流通范围内）。

### F2. 常驻手牌型占位与丢弃约束

```
HandSlots_occupied = COUNT(CurseCards in Hand WHERE Type = PersistentHand)
HandSlots_available = HandLimit - HandSlots_occupied

// 丢弃成本（由各诅咒卡文本定义）
To discard: pay DiscardCost（通常 1~2 费）
If ActionPoints < DiscardCost: 该回合无法主动清除，继续占位

// 战斗结束时：常驻手牌型诅咒卡自动进入弃牌堆（下场战斗仍在卡组中）
```

### F3. 净化成本参考区间（M2 定义权威值，此处记录上下限参考）

```
// 净化成本 = 商店金币消耗，由 M2 商店系统最终定义
PurificationCost_reference：
  抽到触发型：8~10 金（危害最低，净化需求较低）
  常驻手牌型：12~15 金（占位持续危害，净化价值中等）
  常驻牌库型：15~20 金（持续削弱属性，危害最高，净化价值最大）

// 司马懿专属经济分析：隐忍层数价值 ≥ 净化成本时，玩家应优先不净化
// 此公式用于平衡设计评审，非引擎计算公式
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **常驻牌库型在弃牌堆时效果是否继续生效** | 继续生效（弃牌堆属于本局流通范围；只有净化才能消除，丢弃无效）|
| E2 | **常驻牌库型被抽入手牌时持续效果是否变化** | 不变化，不额外触发（手牌中仍计入 ActivePersistentCurses，效果持续）|
| E3 | **常驻手牌型战斗结束时仍在手牌中** | 战斗结束时自动进入弃牌堆，下场战斗洗牌后仍在卡组中循环（非净化，效果不消除）|
| E4 | **常驻手牌型费用恰好等于当前行动点** | 可以支付丢弃；丢弃后该诅咒卡进入弃牌堆（非移出卡组，下局仍在）|
| E5 | **同一张诅咒卡同回合被抽到两次**（极少见：重置牌库后立刻洗回）| 每次抽到独立结算（抽到触发型触发两次；`OnCurseCardDrawn` 发出两次信号）|
| E6 | **多张同名诅咒卡同时在牌库中** | 各自独立计数（F1 公式中 COUNT 按实际张数，非去重；2张韬晦 = MaxHP - 6）|
| E7 | **净化时指定的卡牌在当前手牌而非牌库/弃牌堆中** | 允许净化（净化目标是卡牌本身，无论其当前所处位置）|
| E8 | **同一回合注入诅咒卡后立刻净化** | 允许（先注入 → 触发 `OnCurseInjected` → 净化 → 触发 `OnCursePurified`；该回合常驻效果即时消除）|
| E9 | **诅咒卡被注入弃牌堆，当回合立刻洗牌** | 若该回合洗牌将弃牌堆并入牌库，则该诅咒卡随之进入牌库（常驻牌库型立即生效）|
| E10 | **司马懿同回合抽到3张诅咒卡** | `OnCurseCardDrawn` 触发3次，隐忍层数+3（每次独立计数，受上限4层约束，超出丢弃）|
| E11 | **抽到触发型诅咒卡的效果导致玩家死亡（HP归零）** | 触发结算后按正常死亡流程处理（诅咒效果不因死亡而中断，死亡后战斗结束）|
| E12 | **净化操作成本可以被折扣（如特殊事件奖励）** | 净化成本以最终结算价为准；折扣机制由 M2/事件系统定义，D4 不做约束 |

## Dependencies

### 系统依赖

| 系统 | 方向 | 接口内容 |
|------|------|---------|
| 卡牌战斗系统（C2） | 读 | 诅咒卡在战斗中的抽牌/弃牌/手牌位管理均由 C2 执行；`OnCurseCardDrawn` 事件在 C2 抽牌逻辑中触发 |
| 状态效果系统（C1） | 弱依赖 | 若诅咒卡效果涉及施加标准状态（如中毒），委托 C1 处理；诅咒系统本身不管理状态的持续/消除 |
| 武将系统（D3） | 写 | 发出 `OnCurseCardDrawn` 事件，供司马懿被动监听；D3 注册监听后自行计算隐忍层数 |
| 商店系统（M2） | 弱依赖 | M2 商店节点调用 D4 的净化接口；净化成本由 M2 定义，D4 只执行"移出卡组"操作 |
| 地图节点系统（M1） | 弱依赖 | 事件节点触发诅咒注入时调用 D4 的注入接口；D4 不感知节点类型 |
| 敌人系统（C3） | 读 | 敌人 B/C 类行动中"向玩家注入诅咒卡"使用 D4 提供的注入接口 |
| 资源管理系统（F2） | 读 | 常驻牌库型效果（如最大HP削减）修改的是 F2 管理的 MaxHP 属性 |

### 数据依赖

| 文件 | 用途 |
|------|------|

| `design/gdd/cards-design.md` | 诅咒卡类型定义的来源文档（3.1节）；本文档为其权威展开 |
| `design/gdd/heroes-design.md` | 司马懿的初始韬晦配置（2张）和 `OnCurseCardDrawn` 监听注册位置 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 基础图鉴诅咒卡数量 | 25张（CC0001~CC0025）| 20~35张 | 玩家在战役中遭遇诅咒的频率和多样性 |
| 高阶图鉴诅咒卡数量 | 30张 | 20~40张 | 后期战役诅咒压力强度 |
| 敌人单次注入数量 | 1~2张 | 1~3张 | 单场战斗的诅咒污染速度 |
| 事件单次注入数量 | 1~3张 | 1~4张 | 地图节点选择的风险上限 |
| 常驻牌库型效果强度（以韬晦为例）| MaxHP-3/张 | MaxHP-2~-4/张 | 每张诅咒卡对玩家生存能力的削减幅度 |
| 常驻手牌型丢弃成本 | 1~2费 | 1~3费 | 清除占位诅咒的行动点代价；过高则严重限制出牌节奏 |
| 净化成本参考区间 | 8~20金（按类型分档）| 6~25金 | 净化决策的经济门槛；过低则净化总是最优解，破坏司马懿逆向策略 |
| 司马懿初始韬晦数量 | 2张 | 1~3张 | 司马懿开局隐忍层数积累速度 |

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 抽到触发型诅咒卡被抽入手牌时立即触发效果，触发后进入弃牌堆正常循环 | 战斗测试：含毒药的卡组，确认每次抽到毒药时 HP 立即减少，且毒药进入弃牌堆 |
| AC2 | 常驻牌库型诅咒卡存在于牌库/弃牌堆时持续生效，丢弃不消除效果 | 单元测试：2张韬晦在牌库中→ MaxHP = BaseHP - 6；将韬晦打入弃牌堆后 MaxHP 不变 |
| AC3 | 常驻手牌型诅咒卡占用手牌位，支付费用可丢弃，不支付则无法清除 | 战斗测试：手持泥泞（1费丢弃），行动点=0时无法丢弃，行动点=1时可丢弃；丢弃后手牌位释放 |
| AC4 | 诅咒卡无法被主动打出（不同于普通卡的打出操作）| UI测试：诅咒卡无"打出"按钮；仅常驻手牌型有"丢弃（1费）"按钮 |
| AC5 | 净化操作将诅咒卡移出卡组，本局后续战斗不再出现 | 战役测试：净化一张诅咒卡后完成3场战斗，确认该卡未出现于牌库/弃牌堆/手牌 |
| AC6 | 净化常驻牌库型诅咒卡后，MaxHP 在同回合立即恢复 | 单元测试：持有2张韬晦 → MaxHP = BaseHP - 6；净化1张后当回合 MaxHP = BaseHP - 3 |
| AC7 | `OnCurseCardDrawn` 事件在每次诅咒卡进入手牌时触发（含重复抽到）| 集成测试：选择司马懿，确认每次抽到诅咒卡时隐忍层+1（受上限4约束）|
| AC8 | 司马懿初始卡组含2张韬晦，游戏开始时已在牌库中 | 配置测试：司马懿战斗初始化后，确认牌库含2张CC类韬晦卡 |
| AC9 | 多张同名诅咒卡效果独立叠加，不合并 | 数值测试：3张韬晦在牌库中 → MaxHP = BaseHP - 9（不是 - 3）|
| AC10 | 诅咒卡不可升级，在铁匠铺节点不显示该卡的升级选项 | UI测试：铁匠铺节点的卡牌列表中，诅咒卡不出现或显示为"不可升级" |
