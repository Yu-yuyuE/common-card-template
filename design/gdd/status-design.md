# C1 — 状态效果系统（Status Effect System）

_Created: 2026-03-28_
_Updated: 2026-04-07_
_Status: Complete_
_Priority: MVP / 核心层_

---

## Overview

状态效果系统（Status Effect System）定义《三国称雄》中所有**正面状态（Buff）**与**负面状态（Debuff）**的统一机制：状态如何被施加、持续、消耗、刷新与覆盖。状态是战斗中临时改变单位属性/行为的核心机制，直接影响战斗节奏与策略构筑方向。

当前共定义 **7种Buff + 13种Debuff**，共20种状态。所有状态均遵循本文件的基础机制，无例外。

**系统职责边界：**

- 负责：状态的施加/持续/消耗/覆盖/刷新规则，状态数值定义，同类叠加与不同类互斥裁定
- 不负责：具体卡牌/技能的施加来源设计（`cards-design.md`）、武将专属计数器（隐忍/诡谋/嗜血/铁壁，由 D3 管理）、诅咒卡的触发/持续（由 D4 管理）

## Player Fantasy

**"状态是战场上看不见的第二条战线——谁能更好地施加、维持、转化它，谁就掌控节奏。"**

**① 状态构筑流（攻势）**
通过叠加多层灼烧或中毒，在不需要行动点的情况下对敌人持续造压；破甲+高伤打出单回合爆发上限；精准施加状态打出连招效果。状态流玩家感受到的是"我的每张卡都在为最终爆发铺路"。

**② 状态对抗流（防势）**
坚守减伤+格挡叠盾构筑近乎无解的防线；免疫在关键时刻挡住一波控制连招；在敌方即将施加破甲时优先打出格挡，感受到预判成功的满足感。

**③ 状态爆发流（引爆）**
虎贲卫在滑倒目标上打出额外5点伤害；猎人在灼烧/中毒目标上直击额外8点；黄巾信徒在中毒目标上触发虚弱链——状态积累完成后，一张卡引爆所有层数，打出"终局时刻"的爽感。

**不应有的感受：**

- 状态太多，玩家记不住哪些在生效（UI必须清晰显示当前所有激活状态）
- 某状态组合无解（不同类状态互斥规则防止无限叠加锁死）
- 状态只是数字，没有策略感（每种状态应有明确的适用情境）

## Detailed Rules

### 1. 状态基础机制

| 机制项       | 规则                                                                                                                                 |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| **状态类型** | 正面状态（Buff） / 负面状态（Debuff）                                                                                                |
| **状态来源** | 专属技能、卡牌效果、敌人施加、事件触发                                                                                               |
| **状态层数** | 代表持续回合数，每回合末层数-1                                                                                                       |
| **状态刷新** | 获得新相同状态时，旧状态层数刷新（取较高层数）                                                                                       |
| **状态叠加** | **同类状态**（如中毒+中毒）：叠加层数；**不同类状态**（如中毒+破甲）：互斥，后加的状态**覆盖**前一种                                 |
| **状态失效** | 层数降为0时自动移除                                                                                                                  |
| **状态上限** | 无全局层数上限，但部分状态有特殊说明上限                                                                                             |
| **状态强度** | 状态效果的强度等级，分为普通/中等/强力，影响平衡性，普通可一次施加较高层数（5-10），中等能施加中等层数（2-4）和强力只能施加（1-2）层 |

### 4. 状态叠加与刷新规则

| 情况                      | 规则                                                                                                                             |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 施加同类状态              | 默认刷新（取较高层数）                                                                                                           |
| 剧毒 vs 中毒              | 互斥：施加剧毒时覆盖中毒，层数合并                                                                                               |
| 灼烧 vs 中毒              | **不同状态，效果不同**：灼烧每回合伤害**计算护盾**（护盾先抵挡），中毒每回合伤害**穿透护盾**（直接扣HP）；两者互斥，后加覆盖前者 |
| 同类状态（如中毒+中毒）   | 层数叠加（两次施加2层中毒 = 4层中毒）                                                                                            |
| 不同类状态（如中毒+破甲） | 互斥：后加的覆盖前一种（施加破甲后，中毒被移除）                                                                                 |

> **贾诩特殊规则**：当贾诩的"毒士"被动触发时，施加的新负面状态会覆盖目标现有的负面状态，但新状态的层数 = 现有状态层数 + 新施加层数。这是贾诩被动技能的特殊例外，不影响其他系统的标准状态规则。

### 5. 特殊状态交互规则

| 交互类型     | 规则                                      |
| ------------ | ----------------------------------------- |
| 穿透 vs 格挡 | 穿透攻击可绕过格挡                        |
| 免疫 vs 所有 | 免疫状态下无法获得任何状态                |
| 多层状态消耗 | 消耗型状态（穿透/格挡/混乱）每次仅消耗1层 |

## Formulas

### F1. 状态持续时间

```
Duration = StatusLayers  // 每回合末 -1 层
状态在层数降为 0 时于当回合末自动移除
```

示例：获得3层中毒 → 第1回合末2层，第2回合末1层，第3回合末移除。

### F2. 状态伤害结算（持续伤害类）

```
# 对于中毒、剧毒、灼烧、瘟疫：固定伤害（不乘层数）
DotDamage_per_round = DamagePerLayer

# 对于重伤：伤害随层数增长
DotDamage_per_round = StatusLayers × 1
```

| 状态        | DamagePerLayer | 护盾交互                             | 伤害计算方式         |
| ----------- | -------------- | ------------------------------------ | -------------------- |
| 中毒（D1）  | 4              | **穿透**护盾，直接扣HP               | 固定伤害（不乘层数） |
| 剧毒（D2）  | 7              | **穿透**护盾，直接扣HP               | 固定伤害（不乘层数） |
| 灼烧（D9）  | 5              | **走护盾**（护盾先抵挡，溢出才扣HP） | 固定伤害（不乘层数） |
| 瘟疫（D10） | 3              | **穿透**护盾，直接扣HP               | 固定伤害（不乘层数） |
| 重伤（D12） | 1              | **穿透**护盾（叠加型流血）           | 伤害 = 层数 × 1      |

示例：

- 3层灼烧，护盾剩余4点 → 固定5点伤害，护盾抵挡4点，扣HP 1点，护盾归零。
- 3层中毒 → 固定4点伤害（穿透护盾，直接扣HP）。
- 3层重伤 → 3×1=3点伤害（穿透护盾）。

### F3. 伤害修正类状态结算

```
// 破甲（D7）：受到攻击伤害 +25%
FinalDamage_vs_broken = IncomingDamage × 1.25

// 虚弱（D8）：自身攻击伤害 -25%（与 enemies-design 对齐）
FinalDamage_of_weak = BaseDamage × 0.75

// 怒气（B1）：攻击伤害 +25%
FinalDamage_with_fury = BaseDamage × 1.25

// 坚守（B4）：受到伤害 -25%（与 troop-cards-design 和 card-battle-system 对齐）
FinalDamage_vs_defend = IncomingDamage × 0.75
```

> 多个修正叠加时按乘法顺序结算：破甲+虚弱敌方攻击 = BaseDamage × 0.75 × 1.25

### F4. 瘟疫传播

```
// 每回合结束时结算
ForEach unit WHERE PlagueLayer > 0:
    ForEach adjacent_unit:
        adjacent_unit.PlagueLayer += 1
```

相邻定义：前排/后排同排的紧邻单位（具体位置由 C2 战场模型定义）。

### F5. 状态叠加与覆盖结算顺序

```
// 施加新状态时：
IF new_status.type == existing_status.type:
    // 同类叠加
    existing_status.layers += new_status.layers
ELSE:
    // 不同类互斥：覆盖
    Remove existing_status
    Add new_status
```

## Edge Cases

| #   | 边界情况                                         | 裁定规则                                                                                   |
| --- | ------------------------------------------------ | ------------------------------------------------------------------------------------------ |
| E1  | **状态层数降为0**                                | 当回合末自动移除，不触发任何结束效果                                                       |
| E2  | **同类状态叠加超过20层**                         | 无全局上限，按实际层数计算；若数值异常（如超过武将最大HP），视为平衡异常需记录             |
| E3  | **施加状态时目标已有不同类状态**                 | 覆盖：移除现有状态，施加新状态（E.g. 目标有3层中毒，施加破甲 → 中毒移除，破甲2层生效）     |
| E4  | **覆盖时是否触发被覆盖状态的"移除时效果"**       | 不触发（覆盖不等于主动消除，被覆盖状态静默移除）                                           |
| E5  | **免疫状态下被施加负面状态**                     | 所有负面状态施加失败（不扣层数，静默处理，不显示"免疫"弹出）                               |
| E6  | **免疫状态下被施加正面状态**                     | 正面状态正常施加（免疫只阻止负面）                                                         |
| E7  | **混乱单位无友军目标**                           | 混乱攻击自动取消，不消耗行动点，混乱层数-1                                                 |
| E8  | **滑倒与混乱同时存在（理论上不可能：互斥覆盖）** | 若因特殊顺序同时存在：滑倒优先（无法执行攻击动作，混乱无法触发）                           |
| E9  | **穿透（B6）vs 无护甲目标**                      | 穿透层数仍消耗，攻击正常造成伤害（无护甲则等同于正常攻击，无额外效果）                     |
| E10 | **坚守减伤与护盾的结算顺序**                     | 先计算坚守（B4）对传入伤害的减免，然后再用护盾抵挡减免后的伤害。坚守可以保护护盾。         |
| E11 | **破甲+怒气同时存在（不同类，互斥）**            | 后加的覆盖前者：施加破甲后再施加怒气 → 怒气覆盖破甲（破甲被移除）                          |
| E12 | **格挡（B3）能否拦截穿透（B6）攻击**             | 不能。穿透攻击绕过格挡（详见特殊交互规则）                                                 |
| E13 | **瘟疫传播目标已有其他负面状态**                 | 覆盖：瘟疫覆盖目标现有负面状态（不同类互斥规则适用）                                       |
| E14 | **状态来源（如施加状态的敌人）在状态结算前死亡** | 状态独立存在，来源死亡不影响状态继续生效                                                   |
| E15 | **同回合内施加状态与消耗持续伤害的顺序**         | 同回合内：先处理持续伤害（回合开始），后施加新状态（出牌阶段）；新状态当回合不触发持续伤害 |

## Tuning Knobs

| 调节项       | 当前值          | 安全区间     | 影响维度                                   |
| ------------ | --------------- | ------------ | ------------------------------------------ |
| 中毒单层伤害 | 4点（穿透护盾） | 3~6          | 毒流基础持续压力；穿透特性使低层数即有威胁 |
| 剧毒单层伤害 | 7点（穿透护盾） | 5~9          | 高阶毒流爆发上限；强力等级，施加层数应受限 |
| 灼烧单层伤害 | 5点（走护盾）   | 4~7          | 火攻流持续压力；走护盾使护盾流有对抗空间   |
| 瘟疫单层伤害 | 3点（穿透护盾） | 2~5          | 传播流扩散速度；伤害较低，危害来自传播     |
| 重伤单层伤害 | 1点/层（穿透）  | 1~2          | 流血叠加流上限；多层时伤害可观             |
| 坚守减伤比例 | 25%             | 20%~40%      | 防御流生存强度（与D2兵种卡一致）           |
| 破甲增伤比例 | 25%             | 20%~35%      | 破防后单回合爆发增幅                       |
| 怒气增伤比例 | 25%             | 20%~35%      | 进攻流单回合伤害上限                       |
| 虚弱减伤比例 | 25%             | 20%~40%      | 削弱流对目标输出的压制强度                 |
| 盲目命中率   | 50%             | 40%~60%      | 弓兵/远程被致盲后的有效伤害期望            |
| 免疫持续回合 | 1回合           | 1~2          | 负面状态免疫窗口；过高则无法控制           |
| 瘟疫传播范围 | 紧邻单位        | 紧邻（固定） | 固定为紧邻，扩展到全场会使瘟疫无法对抗     |

## Acceptance Criteria

| #    | 验收条件                                      | 验证方式                                                                                          |
| ---- | --------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| AC1  | 所有状态在状态表中明确定义，编号不重复        | 数据验证：遍历 status-design.md 状态表，确认 B1~B7、D1~D13 编号连续无重复                         |
