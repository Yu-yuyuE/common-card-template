# 状态系统设计文档（Status Design）

_Updated: 2026-03-28_
_Status: Initial Draft_

---

## Overview

本系统定义游戏中所有**正/负面状态**（Buffs / Debuffs）的统一机制、命名、效果、交互规则与数值标准。状态是战斗中临时改变单位属性/行为的关键机制，直接影响战斗节奏与策略构筑。

---

## Player Fantasy

- **状态构筑流**：通过施加/维持/转化状态获得战术优势
- **状态对抗流**：通过免疫/清除/反制状态获得生存空间
- **状态爆发流**：通过状态累积/转化/引爆获得瞬间高额收益

## Detailed Rules

### 1. 状态基础机制

| 机制项       | 规则                                                                                                                                 |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| **状态类型** | 正面状态（Buff） / 负面状态（Debuff）                                                                                                |
| **状态来源** | 专属技能、卡牌效果、敌人施加、事件触发                                                                                               |
| **状态层数** | 代表持续回合数，每回合末层数-1                                                                                                       |
| **状态刷新** | 获得新相同状态时，旧状态层数刷新（取较高层数）                                                                                       |
| **状态叠加** | 不同状态不可同时存在；同类状态默认叠加层数，不同类状态无论正面或负面，后加的状态都会顶替前一种（除非特殊说明）                       |
| **状态失效** | 层数降为0时自动移除                                                                                                                  |
| **状态上限** | 无全局层数上限，但部分状态有特殊说明上限                                                                                             |
| **状态强度** | 状态效果的强度等级，分为普通/中等/强力，影响平衡性，普通可一次施加较高层数（5-10），中等能施加中等层数（2-4）和强力只能施加（1-2）层 |

### 4. 状态叠加与刷新规则

| 情况               | 规则                               |
| ------------------ | ---------------------------------- |
| 施加同类状态       | 默认刷新（取较高层数）             |
| 剧毒 vs 中毒       | 互斥：施加剧毒时覆盖中毒，层数合并 |
| 灼烧 vs 中毒       | 效果相同，仅来源命名区分           |
| 同类状态层数叠加   | 后加的同类状态会叠加层数           |
| 不同类状态不能共存 | 后加的不同类状态会覆盖前一种状态   |

---

### 5. 特殊状态交互规则

| 交互类型     | 规则                                      |
| ------------ | ----------------------------------------- |
| 穿透 vs 格挡 | 穿透攻击可绕过格挡                        |
| 免疫 vs 所有 | 免疫状态下无法获得任何状态                |
| 多层状态消耗 | 消耗型状态（穿透/格挡/混乱）每次仅消耗1层 |

## Formulas

### 状态持续时间

`Duration = StatusLayers`

- 例：获得3层中毒 → 持续3回合（每回合末-1层）

### 状态伤害结算

`TotalDamage = StatusLayers × DamagePerLayer`

- 例：2层灼烧 → 每回合造成 2 × 5 = 10点伤害

### 瘟疫传播（D10）

`SpreadTargets = AdjacentUnits`

- 回合结束时，每个带有瘟疫的单位使其相邻单位获得1层瘟疫

## Edge Cases

1. **状态层数为0**：状态自动移除，不触发任何效果
2. **同时获得多个同类状态**：按刷新规则处理，只保留1个
3. **状态来源消失但状态仍存在**：状态继续生效直至层数耗尽
4. **免疫状态下被施加状态**：所有负面状态施加失败，无提示
5. **混乱单位无友军目标**：混乱攻击失败，不消耗行动
6. **滑倒+混乱同时存在**：滑倒优先生效，攻击动作被阻止
7. **穿透攻击 vs 无护甲目标**：穿透仍可消耗，但无额外效果

## Tuning Knobs

| 调节项                 | 安全区间               | 影响 gameplay 维度 |
| ---------------------- | ---------------------- | ------------------ |
| 中毒/剧毒/灼烧单层伤害 | 3~10                   | 持续伤害压力       |
| 瘟疫传播范围           | 相邻单位               | 状态扩散速度       |
| 状态最大层数           | 无上限（除非特殊说明） | 状态堆叠上限       |
| 免疫状态持续回合       | 1回合                  | 负面状态免疫窗口   |
| 混乱命中率             | 100%（强制攻击友军）   | 控制稳定性         |

## Acceptance Criteria

1. 所有状态必须在状态表中明确定义（编号/名称/机制/来源）
2. 状态层数必须按回合递减，0层时自动移除
3. 同类状态默认刷新，不允许无限叠加（除非特殊说明）
4. 状态交互规则必须明确（如穿透vs格挡）
5. 状态来源必须可追溯（日志/战斗记录）
6. 瘟疫必须在回合结束时传播至相邻单位
7. 免疫状态必须能阻止所有负面状态施加
8. 混乱状态必须强制攻击友军（优先最近单位）
