_Created: 2026-04-08_
_Updated: 2026-04-08_
_Status: Complete_
_Priority: Alpha / 元游戏层_

---

## Overview

事件系统定义《三国称雄》中**奇遇节点**触发的历史随机事件的内容结构、分类框架、选项结算规则与后果类型。每个奇遇节点从事件库中随机抽取一个事件，**大多数事件直接触发固定后果**，少数事件提供2~3个选项供玩家选择，产生不同结果。后果类型包括：资源变化、战斗触发、卡组变化、天气修改（不修改地形）。

事件库共分5大类（政治/军略/民心/人物/转折），**首版目标50+事件**。

**系统职责边界：**

- 负责：事件内容定义（文本、选项、后果）、事件库管理、随机抽取逻辑、后果结算接口
- 不负责：奇遇节点的触发时机（M1 定义）、地形天气修改的执行（D1 定义）、装备发放（M3 定义）、商店开启（M2 定义）

## Player Fantasy

**"历史在这一刻分岔——我的选择会改写战局。"**

**① 叙事沉浸感（代入层）**
事件不是冰冷的数值交换，而是三国历史的真实片段。"招降旧部"让玩家感受到曹操收编青州兵的抉择；"天候突变"重现赤壁之战前的东南风。每个事件都有历史原型，玩家不是在"刷数值"，而是在"经历三国"。

**② 风险与收益的张力（决策层）**
少数可选事件带来真正的两难：接受谋士来投获得强力卡牌，但可能触发战斗；赈济灾民损失粮草但获得民心加成。这些选择没有绝对正确答案，取决于当前战局和玩家风格。

**③ 不可控的戏剧性（惊喜层）**
大多数事件直接触发——"援军迟到"突然让下一场战斗变难；"伪诏风波"意外获得装备。这种不可控性是 Roguelike 的核心魅力，让每局都有独特的故事线。

**不应有的感受：**
- 事件文本与三国背景无关（每个事件都应有历史感）
- 所有事件都是"选A得好处，选B吃亏"的明牌选择
- 事件后果对战局毫无影响（每个后果都应有战术意义）

## Detailed Rules

### 1. 事件分类（5大类，50+事件）

| 类别 | 主题 | 首版数量 | 代表事件 |
|------|------|---------|---------|
| **政治抉择** | 外交、联盟、权谋 | 10 | 招降旧部、联盟使者、伪诏风波 |
| **军略抉择** | 战术、奇袭、布阵 | 10 | 夜袭营寨、伏兵山道、火攻时机 |
| **民心抉择** | 民生、征募、赈济 | 10 | 赈济灾民、征粮保军、安抚流民 |
| **人物抉择** | 人才、将领、谋士 | 10 | 谋士来投、旧将请战、间者求见 |
| **转折事件** | 突发、天灾、变故 | 10 | 援军迟到、天候突变、兵变流言 |

### 2. 事件触发流程

```
玩家移动到奇遇节点
  → 从事件库随机抽取1个事件（按权重，排除已触发事件）
  → 显示事件文本与历史背景
  → 判断事件类型：
    - 无选项事件：直接结算固定后果
    - 有选项事件：展示2~3个选项，玩家选择后结算对应后果
  → 后果立即生效（资源变化/战斗触发/卡组变化/天气修改）
  → 事件结束，玩家继续地图推进
```

---

### 3. 后果类型定义

#### 3a. 资源变化

| 资源类型 | 变化范围 | 示例 |
|---------|---------|------|
| 金币 | ±20~100 | 地方豪族献粮：+80金币 |
| 粮草 | ±10~50 | 赈济灾民：-30粮草 |
| HP | ±5~20 | 征粮保军：-10 HP（民怨） |

#### 3b. 战斗触发

- 事件后果直接触发一场战斗（敌人配置由事件定义）
- 战斗胜利后正常获得金币奖励，失败则按战斗失败规则处理
- 示例：夜袭营寨成功 → 触发1场精英战斗

#### 3c. 卡组变化

| 变化类型 | 说明 | 示例 |
|---------|------|------|
| 获得卡牌 | 从指定卡池随机获得1~2张卡牌 | 谋士来投：获得1张高级技能卡 |
| 移除卡牌 | 玩家选择移除卡组中1张卡牌 | 净化仪式：移除1张诅咒卡 |
| 升级卡牌 | 玩家选择升级卡组中1张 Lv1 卡牌为 Lv2 | 兵法传授：升级1张攻击卡 |

#### 3d. 天气修改

- 修改**下一场战斗**的天气（不修改地形）
- 天气修改在进入下一个战斗节点时生效，战斗结束后恢复正常
- 示例：天候突变 → 下一场战斗天气强制为"雾"

---

### 4. 事件选项结构（少数可选事件）

有选项的事件提供2~3个选项，每个选项包含：

| 字段 | 说明 |
|------|------|
| 选项文本 | 玩家看到的选择描述 |
| 后果类型 | 资源变化/战斗触发/卡组变化/天气修改 |
| 后果数值 | 具体效果（如+50金币、触发战斗、获得1张卡） |

**示例事件：谋士来投（可选）**

- 选项A：接纳谋士 → 获得1张高级技能卡 + 触发1场战斗（谋士旧主追击）
- 选项B：婉拒谋士 → +30金币（谋士献上盘缠）

**示例事件：援军迟到（无选项）**

- 固定后果：下一场战斗天气强制为"雾"（能见度降低，敌方首先获得2层盲目）

---

### 5. 事件抽取规则

- 每个事件有基础权重（1~3），见各事件条目定义
- 同一张小地图内，已触发的事件**不再重复抽取**
- 部分事件设有**前置条件**（满足条件才进入抽取池），示例：
  - "赈济灾民"：粮草 ≥ 30 才可触发
  - "断粮奇袭"：当前地形为山地或森林才可触发

## Formulas

### F1. 事件抽取权重

```
EventProbability(e) = BaseWeight(e) / ΣBaseWeight(available)

变量：
  BaseWeight(e)          = 事件 e 的基础权重（1~3，定义在事件条目中）
  available              = 当前小地图内未触发、且满足前置条件的事件集合
  ΣBaseWeight(available) = 所有可用事件权重之和

示例：
  可用事件10个，权重分别为 [3,2,2,2,2,1,1,1,1,1] = 16
  权重为3的事件触发概率 = 3/16 = 18.75%
  权重为1的事件触发概率 = 1/16 = 6.25%
```

### F2. 后果数值参考区间

```
资源变化后果：
  金币收益：+20~+100（低价值事件20~40，中等40~70，高价值70~100）
  金币损失：-20~-60
  粮草收益：+10~+50
  粮草损失：-10~-50
  HP回复：  +5~+20
  HP损失：  -5~-20

卡组变化后果：
  获得卡牌：1~2张（普通卡或高级卡，由事件等级决定）
  移除卡牌：玩家选择移除1张
  升级卡牌：玩家选择升级1张 Lv1→Lv2
```

### F3. 有选项事件收益期望对比（设计约束）

```
// 选项之间应无绝对优劣，期望收益相近
// 高收益选项 = 伴随更高风险或更多损失

示例：谋士来投
  选项A（接纳）：获得高级技能卡（价值≈75金）+ 触发精英战斗（风险）
  选项B（婉拒）：+30金币（稳定收益）
  → A的期望价值 > B，但A有风险；选择A需要有战斗胜算
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **当前小地图所有事件已触发（无可用事件）** | 从全局事件库重置本图已触发记录，重新抽取（极端情况，正常游戏不应触发） |
| E2 | **前置条件导致可用事件池为空（如所有事件都要求粮草≥30而玩家粮草=0）** | 忽略所有前置条件，从全库无条件抽取一个事件 |
| E3 | **战斗触发型事件：触发的战斗失败** | 按标准战斗失败处理（扣除粮草，不回退节点）；事件视为已触发，不重复 |
| E4 | **卡组变化：获得卡牌时卡组为空（极端情况）** | 正常获得，卡组无上限；此情况不会导致错误 |
| E5 | **卡组变化：移除卡牌时卡组中无可移除的卡（如全是诅咒卡且事件要求移除普通卡）** | 事件后果改为跳过移除步骤，直接结算其他后果（如有）；向玩家提示"无可移除卡牌" |
| E6 | **卡组变化：升级卡牌时卡组中无 Lv1 卡** | 同E5，升级步骤跳过；向玩家提示"无可升级卡牌" |
| E7 | **天气修改：下一节点不是战斗节点（如是商店/酒馆）** | 天气修改状态保持，直到下一个战斗节点生效为止（跨越非战斗节点传递） |
| E8 | **连续两个奇遇节点均触发天气修改事件** | 后触发的天气覆盖前一个，以最后触发的天气为准进入战斗 |
| E9 | **有选项事件：玩家不选择直接关闭界面** | 视为选择"无后果"选项（若无此选项则强制要求玩家选择，不可关闭） |
| E10 | **同一事件在同一张小地图的多个奇遇节点均可触发** | 第一次触发后标记为已触发，同图内不再抽到该事件 |

## Dependencies

### 系统依赖

| 系统 | 方向 | 接口内容 |
|------|------|---------|
| 地图节点系统（M1） | 读 | M1 触发奇遇节点时调用本系统的事件抽取接口；本系统不感知节点本身 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 首版事件总数 | 50 | 40~80 | 事件重复率；过少则同图频繁重复，过多则长尾事件几乎不出现 |
| 事件基础权重范围 | 1~3 | 1~5 | 稀有事件与常见事件的出现频率差距 |
| 无选项事件占比 | ~70% | 60%~80% | 叙事节奏 vs 决策密度；过多无选项则玩家感觉被动 |
| 有选项事件占比 | ~30% | 20%~40% | 决策感强度；过多有选项则节奏拖慢 |
| 资源变化后果：金币收益上限 | +100 | +60~+150 | 与商店购买力的比较；过高则事件成为主要金币来源 |
| 资源变化后果：粮草损失上限 | -50 | -30~-80 | 对玩家推进节奏的压力上限 |
| 战斗触发后果：难度 | 精英级（约50%）/ 普通（约50%） | 可调 | 事件战斗的平均难度与奖励期望 |
| 天气修改后果：可选天气类型 | 雨/风/雾（不含晴） | 可扩展 | 天气修改的策略价值；晴通常不作为奖励天气 |
| 前置条件事件比例 | ~20% | 10%~30% | 条件事件的出现率；过高则部分玩家几乎看不到某类事件 |
