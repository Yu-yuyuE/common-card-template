## Overview

商店系统定义《三国称雄》中**商店节点**的全部交易行为与规则约束。商店是战役中唯一可以**购买卡牌、升级卡牌、购买装备**的场所，金币是唯一通货，金币由战斗获得并在整局游戏内跨小地图累积保留。

**三项功能：**

1. **买卡** — 从通用卡池随机生成货架，玩家用金币购入加入卡组
2. **升级卡** — 展示卡组中所有可升级卡，玩家付金币执行 Lv1→Lv2（规则由 M5 定义，价格由本系统定义）
3. **买装备** — 从装备池随机生成货架，玩家付金币购入（装备效果由 M3 定义）

**系统职责边界：**

- 负责：货架生成逻辑、价格定义、金币扣除执行、刷新机制
- 不负责：卡牌升级规则（M5）、装备效果（M3）、金币来源与奖励数值（C2 战斗结算）、诅咒净化（军营节点负责）

## Player Fantasy

**"这家军需官手里有好东西——但金币永远不够用。"**

**① 抉择张力（资源层）**
每次进入商店，玩家面对的不是"买什么"，而是"放弃什么"。买一张强力攻击卡，还是把这笔钱留给装备？升级手里的技能卡锁定当前流派，还是先买一件防御装备撑过下一场精英战？金币跨整局累积但永远紧张，每一次花销都是一次承诺。

**② 构筑定型感（策略层）**
商店是卡组成形的关键节点。战役前半段积攒金币、在商店里逐步强化核心牌；战役后半段商店决定能否补全构筑短板。好的商店访问让玩家感觉"这局的路线清晰了"。

**③ 惊喜与期待（发现层）**
每次商店货架随机刷新，偶尔出现正好契合当前流派的强力卡或装备，带来"正好需要这个"的满足感。货架的不确定性是 Roguelike 核心乐趣之一。

**不应有的感受：**

- 商店货架与当前武将流派完全无关（每次都买不到想要的方向）
- 金币太多导致没有取舍（所有东西都买得起）
- 金币太少导致商店形同虚设（进去出来什么都没做）

## Detailed Rules

### 1. 金币系统

| 规则项       | 说明                                   |
| ------------ | -------------------------------------- |
| **来源**     | 战斗胜利奖励（数值由 C2 战斗结算定义） |
| **持久性**   | 跨小地图累积，整局游戏保留；不跨局保留 |
| **存储上限** | 无上限                                 |
| **消耗**     | 仅在商店节点通过购买/升级消耗          |

---

### 2. 货架生成规则

进入商店节点时，系统自动生成三个独立货架区：

| 货架区     | 内容                                          | 数量      |
| ---------- | --------------------------------------------- | --------- |
| **卡牌区** | 攻击卡 + 技能卡（随机抽取）                   | 固定 4 张 |
| **升级区** | 玩家卡组中所有 Lv1 攻击卡、技能卡、武将专属卡 | 全部展示  |
| **装备区** | 随机装备                                      | 固定 2 件 |

- 进入商店时自动生成，**同一次访问内货架固定，不自动刷新**
- 购买后该位置**不补货**（空出即空）
- 本版本**不提供付费刷新**

---

### 3. 卡牌区生成逻辑

#### 3a. 卡池来源

每张卡牌货位独立从以下卡池按权重抽取：

| 卡池                             | 比例   |
| -------------------------------- | ------ |
| 通用攻击卡（`attack_cards.csv`） | 约 60% |
| 通用技能卡（`skill_cards.csv`）  | 约 40% |

#### 3b. 等级抽取（每张货位独立判定）

| 结果       | 概率 |
| ---------- | ---- |
| **Lv1 卡** | ~85% |
| **Lv2 卡** | ~15% |

- Lv2 卡直接以升级状态出售，加入卡组后即为 Lv2，**不占用 M5 升级次数**（该卡已是 Lv2，M5 无需再处理）
- Lv2 卡的价格高于同名 Lv1 卡（见 Tuning Knobs）

#### 3c. 州专属卡过滤

每张货位在确定卡池后，进行额外的**州专属卡判定**：

- 以一定内在概率，将该货位的卡替换为当前战役所在州的专属卡
- 若当前州无专属卡、或该州所有专属卡玩家均已持有，跳过此判定
- 具体概率见 Tuning Knobs

#### 3d. 防重复规则

- 同一货架的 4 张卡**不重复**（卡牌 ID 唯一）
- 若卡池剩余不足 4 张不重复卡（极端情况），允许同名卡以不同等级（Lv1/Lv2）各自占位

---

### 4. 买卡规则

- 选择卡牌 → 确认金币 → 卡牌立即加入卡组（以展示等级加入，Lv1 或 Lv2）
- 购买不受卡组总数限制（卡组无上限）
- 不可退货

---

### 5. 升级区规则

- 展示卡组中**所有 Lv1 攻击卡、技能卡、武将专属卡**
- **不展示**：诅咒卡、兵种卡（兵种卡升级在军营节点）
- 升级规则、收益标准完全遵照 **M5 卡牌升级系统**
- 同一次商店访问可升级多张（金币足够即可）
- 升级后该卡从升级区移除（已是 Lv2，不可再升级）

---

### 6. 装备区规则

- 展示 2 件随机装备（来自 M3 装备池）
- 购买后立即装备（槽位与替换规则由 M3 定义）
- 若武将装备槽已满：购买前提示"装备槽已满，将替换现有装备"

---

### 7. 商店访问流程

```
进入商店节点
  → 生成货架：卡牌区×4 / 升级区（全卡组可升级卡） / 装备区×2
  → 玩家自由操作（任意顺序、可多次交易，金币足够即可）
    - 每笔交易：选择 → 确认 → 金币扣除 → 商品到手
  → 玩家"离开商店" → 货架状态清除（下次访问重新生成）
```

## Formulas

### F1. 卡牌购买价格

```
PriceLv1(card) = BasePrice × RarityMult

变量：
  BasePrice = 50（金币）
  RarityMult：普通=1.0 / 高级=1.5 / 稀有=2.0

示例：
  普通攻击卡 Lv1 = 50 × 1.0 = 50 金
  高级技能卡 Lv1 = 50 × 1.5 = 75 金
  稀有技能卡 Lv1 = 50 × 2.0 = 100 金
```

### F2. Lv2 卡直购价格

```
PriceLv2(card) = PriceLv1(card) × 1.8

示例：
  普通攻击卡 Lv2 = 50 × 1.8 = 90 金
  高级技能卡 Lv2 = 75 × 1.8 = 135 金
```

> **设计意图**：直购 Lv2（90金）比"买Lv1+首次升级（50+60=110金）"便宜 20 金，
> 但货架刷出概率仅 ~15%，且不一定是想要的卡。

### F3. 卡牌升级价格（累进翻倍）

```
UpgradePrice(n) = BaseUpgradePrice × CardTypeMult × 2^(n-1)

变量：
  BaseUpgradePrice = 60（金币，本次访问首次升级基准价）
  CardTypeMult：攻击卡/技能卡=1.0 / 武将专属卡=1.5
  n = 本次商店访问内已执行的升级次数（含本次，从 1 开始）
  重置条件：玩家离开商店后 n 重置为 1

示例（普通攻击卡/技能卡，CardTypeMult=1.0）：
  本次访问第1次升级：60 × 1.0 × 2^0 = 60 金
  本次访问第2次升级：60 × 1.0 × 2^1 = 120 金
  本次访问第3次升级：60 × 1.0 × 2^2 = 240 金
  本次访问第4次升级：60 × 1.0 × 2^3 = 480 金

示例（武将专属卡，CardTypeMult=1.5）：
  本次访问第1次升级：60 × 1.5 × 2^0 = 90 金
  本次访问第2次升级：60 × 1.5 × 2^1 = 180 金
```

### F4. 装备价格

```
EquipmentPrice = EquipmentTier × 40

变量：
  EquipmentTier = 装备等级（M3 定义，范围 1~3）

示例：
  1级装备 = 40 金
  2级装备 = 80 金
  3级装备 = 120 金
```

### F5. 州专属卡出现概率

```
RegionalCardChance(slot) = BaseChance + ProgressBonus

变量：
  BaseChance = 20%（每张货位独立判定）
  ProgressBonus：第1张小地图 = +0% / 第2张 = +5% / 第3张 = +10%

示例：
  第1张地图：每货位 20% 概率为州专属卡
  第3张地图：每货位 30% 概率为州专属卡
```

### F6. 金币战斗奖励（C2 参考值）

```
GoldReward = BaseGold × DifficultyMult

变量：
  BaseGold = 30
  DifficultyMult：普通=1.0 / 精英=1.5 / Boss=2.5

示例：
  普通战斗 = 30 金
  精英战斗 = 45 金
  Boss战斗 = 75 金

单张地图估算（12层：8普通+3精英+1Boss）：
  (30×8) + (45×3) + (75×1) = 450 金/图
```

## Edge Cases

| #   | 边界情况                                                 | 裁定规则                                                                                         |
| --- | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| E1  | **金币不足以购买选中商品**                               | 按钮置灰，显示所需金币；不允许部分支付                                                           |
| E2  | **卡组中没有任何可升级卡（全是 Lv2 / 诅咒卡 / 兵种卡）** | 升级区显示"暂无可升级卡牌"，该区域折叠或置灰                                                     |
| E3  | **商店货架刷出玩家已持有的同名卡**                       | 允许购买（卡组可有多张同名卡）；显示"已持有 N 张"提示                                            |
| E4  | **州专属卡玩家已全部持有（每种至少1张）**                | 跳过州专属卡判定，该货位按普通流程生成                                                           |
| E5  | **升级翻倍后玩家金币不足以支付第 N 次升级**              | 该升级按钮显示实际价格并置灰；玩家可选择先购买其他商品，但升级计数器 n 不重置                    |
| E6  | **玩家升级一张卡后离开商店，再次进入**                   | n 重置为 1，下次首次升级价格回到 60 金（× CardTypeMult）                                         |
| E7  | **装备槽已满时尝试购买装备**                             | 提示"装备槽已满，购买后将替换现有装备"；确认购买则触发替换（替换逻辑由 M3 定义）；取消则不扣金币 |
| E8  | **卡池可用卡不足 4 张（极端情况）**                      | 允许同名卡以 Lv1/Lv2 各自占位；若仍不足，货位显示为空                                            |
| E9  | **直购 Lv2 卡后，同一次访问内升级区存在同名 Lv1 卡**     | 允许；两张卡独立，直购 Lv2 不影响升级区中其他 Lv1 卡的升级资格                                   |
| E10 | **连续升级导致价格超出持有金币（如第5次需960金）**       | 正常显示价格，按钮置灰；提示"本次访问升级价格过高，建议下次访问时优先升级"                       |
| E11 | **战斗奖励使总金币超出 UI 显示上限**                     | 数值正常计算和存储，UI 做上限保护显示（如">9999"），不截断实际金币值                             |

## Dependencies

### 系统依赖

| 系统                 | 方向  | 接口内容                                                                               |
| -------------------- | ----- | -------------------------------------------------------------------------------------- |
| 地图节点系统（M1）   | 读    | 商店节点由 M1 触发；M1 提供当前战役所在州信息（用于州专属卡过滤）                      |
| 资源管理系统（F2）   | 读/写 | 金币作为战役层资源，由 F2 管理持久化；每笔交易通过 F2 扣除金币                         |
| 卡牌战斗系统（C2）   | 读    | 战斗结算后触发金币奖励（C2 定义奖励数值，F2 存储）；购买的卡牌加入 C2 管理的卡组       |
| 卡牌升级系统（M5）   | 读    | 升级区调用 M5 的升级规则和约束（升级次数、单维度提升等）；M2 定义价格，M5 执行升级逻辑 |
| 装备系统（M3）       | 读    | 装备区从 M3 装备池读取可售装备；购买后交由 M3 处理槽位与替换逻辑                       |
| 存档持久化系统（F1） | 写    | 金币余额、本次商店访问的升级计数器 n 需由 F1 持久化（跨地图保留）                      |

## Tuning Knobs

| 调节项             | 当前值     | 安全区间        | 影响维度                                               |
| ------------------ | ---------- | --------------- | ------------------------------------------------------ |
| 货架卡牌数量       | 4 张       | 3~5 张          | 每次商店的选择宽度；过少则无选择感，过多则稀释金币压力 |
| 货架装备数量       | 2 件       | 1~3 件          | 装备获取频率；过多则装备决策比例过重                   |
| Lv2 卡出现概率     | ~15%       | 10%~25%         | 直购 Lv2 的惊喜频率；过高则买卡+升级路线被架空         |
| 州专属卡基础概率   | 20%        | 10%~35%         | 地域主题的强度；过高则普通卡难以出现                   |
| 州专属卡进度加成   | +5%/张地图 | +0%~+10%        | 战役后期地域特色强化程度                               |
| 攻击卡:技能卡比例  | 60%:40%    | 50%:50%~70%:30% | 货架构成风格；偏攻击或偏功能                           |
| Lv2 卡溢价系数     | 1.8×       | 1.5×~2.2×       | 直购 Lv2 相对"买+升"路线的性价比                       |
| 升级基准价格       | 60 金      | 40~80 金        | 升级的基础门槛；影响第1次升级的抉择权重                |
| 升级翻倍基数       | 2×         | 1.5×~3×         | 多次升级的价格增速；影响单次商店能合理升级的数量上限   |
| 武将专属卡升级系数 | 1.5×       | 1.2×~2.0×       | 专属卡升级相对普通卡的溢价；体现专属卡稀缺性           |
| 普通卡基础购买价   | 50 金      | 40~70 金        | 买卡门槛；影响单图内可购卡数量                         |
| 装备每级价格       | 40 金/级   | 30~60 金/级     | 装备消费在金币分配中的占比                             |
| 战斗金币基础奖励   | 30 金      | 20~50 金        | 单图总收入；决定玩家商店消费能力上限                   |

## Acceptance Criteria

| #    | 验收条件                                                            | 验证方式                                                                                |
| ---- | ------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| AC1  | 进入商店自动生成卡牌区×4、升级区、装备区×2                          | 功能测试：访问商店节点，确认三区均正常显示                                              |
| AC2  | 卡牌区 4 张不重复（ID 唯一）                                        | 功能测试：刷新 100 次商店，统计同一货架出现重复卡牌 ID 的次数（应为 0，极端情况除外）   |
| AC3  | 每张货位独立判定 Lv2，概率约 15%                                    | 统计测试：记录 1000 次货位生成，Lv2 出现率应在 10%~20% 之间                             |
| AC4  | 当前州专属卡以约 20% 概率出现在货位                                 | 统计测试：在固定州地图生成 500 次商店，州专属卡出现率应在 15%~25% 之间                  |
| AC5  | 升级区只展示 Lv1 攻击卡、技能卡、武将专属卡；诅咒卡和兵种卡不出现   | UI 测试：卡组包含诅咒卡和兵种卡时，升级区不显示这两类卡                                 |
| AC6  | 首次升级价格 = BaseUpgradePrice × CardTypeMult；第 N 次升级价格翻倍 | 功能测试：连续升级3张卡，确认价格依次为 60、120、240（普通卡）                          |
| AC7  | 离开商店再进入，升级计数器重置为 1                                  | 功能测试：升级1张卡后离开商店，再次进入后升级另一张卡，价格应为 60 金                   |
| AC8  | 金币不足时购买/升级按钮置灰并显示所需金币                           | UI 测试：持有金币=30，面对需80金的商品时按钮不可点击且显示"需要80金"                    |
| AC9  | 购买卡牌后立即加入卡组（以对应等级加入）                            | 功能测试：购买 Lv2 卡后进入战斗，确认该卡以 Lv2 效果出现在手牌中                        |
| AC10 | 购买装备触发 M3 槽位逻辑；装备槽已满时显示替换提示                  | 功能测试：装备6件后尝试购买第7件，确认出现"装备槽已满"提示且可正常替换                  |
| AC11 | 金币跨小地图累积，整局保留；局结束后清零                            | 战役测试：第1张地图结束后金币余额在第2张地图开始时保持不变；新开一局金币归零            |
| AC12 | 商店离开后货架状态清除，下次访问重新生成                            | 功能测试：第一次访问商店购买1张卡后离开，再次访问确认货架重新随机生成（非保留上次剩余） |
