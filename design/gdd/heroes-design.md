## Overview

武将系统（Hero System）是三国称雄玩法分化的核心驱动力。每位武将通过**基础数值**（HP、费用、统帅）、**1个被动技能**和**专属卡组**三个维度，定义玩家在整局战役中的策略方向。22名武将分属魏、蜀、吴、群雄四阵营，每名武将拥有独立的5张生涯地图，构成独立的叙事与成长线。

本文件定义武将系统：

- 阵营：魏、蜀、吴、群雄
- 每阵营 5~6 名武将（本版22名）
- 每位武将包含：基础值、兵种倾向、**被动技能（仅1个）**、专属卡数量（12~18）、生涯地图（每武将5张地图、每图3幕、每幕12~16层节点、每层1~4节点）

> 已取消”武将主动技能”，以简化操作负担。

**系统职责边界：**
- 负责：武将基础数值初始化、被动技能触发接口、专属卡组注入、生涯地图结构定义、兵种倾向权重
- 不负责：具体状态的施加/消除（由 C1 处理）、战斗回合流程（由 C2 处理）、地图节点生成（由 M1 处理）

## Player Fantasy

**”换武将=换流派，同一张地图，不同将领，是截然不同的战局。”**

玩家在武将系统中应感受到三种核心幻想：

**① 身份代入感（叙事层）**
选择曹操，就是在走一条从陈留起兵到赤壁折戟的历史轨迹。武将不只是数值载体，更是三国历史角色的战场具现。每张专属卡都有历史质感，打出”借东风”时玩家感受的是诸葛亮在赤壁的历史时刻。

**② 流派确定感（策略层）**
选定武将即选定策略方向——曹操是兵种卡+虚弱收割、司马懿是吸纳诅咒的韬晦经营、典韦是以血换暴击的死战狂人。在第一张地图的前三场战斗后，玩家应清楚知道”我这局的玩法是什么”。

**③ 成长差异化（进程层）**
不同武将的5张生涯地图有不同的地形/天气倾向，对应不同的历史战场。曹操在平原骑兵优势如鱼得水；诸葛亮在山地谋略流克制明显。武将与地图的匹配感是策略深度的来源。

**不应有的感受：**
- 武将只是皮肤，换谁打法都一样（被动必须对构筑产生实质影响）
- 某武将明显强于其他（22名武将应有不同适合情境，无绝对强弱）

## Detailed Rules

### 1) 武将通用基础值模板

- 生命值：40~60
- 初始手牌：5
- 每回合费用：3~4
- 统帅：3~6（可携带兵种卡上限）

### 2) 兵种倾向

兵种卡类型：步兵 / 骑兵 / 弓兵 / 谋士 / 盾兵

每位武将定义：主修2项 + 次修1项。

> 兵种倾向影响军营节点的兵种卡出现权重：主修兵种卡的出现概率更高。倾向不限制武将可选的兵种卡类型，仅作为构筑引导。

### 3) 技能结构（已简化）

- 每位武将仅保留 **1个被动技能**。
- 战斗深度由：被动 + 专属卡 + 地形 + 兵种卡协同构成。

### 4) 专属卡组

- 每位武将 **≤12张**专属卡（司马懿≤15张，含初始2张韬晦负面卡）。
- 专属卡不进入常规卡池。

### 5) 生涯地图线（已调整）

- 每位武将生涯固定 **5张地图**（后续可扩展到6+）。
- 每张地图固定 **3幕结构**。
- 每幕至少 **10个节点**（可更多）。

地图命名建议：

- M1 起势图
- M2 扩张图
- M3 转折图
- M4 决战前图
- M5 终局图

## Formulas

### F1. 武将有效生存值（参考）

```
EffectiveHP = BaseHP + Shield_max

Shield_max：
  默认武将   = BaseHP（护盾上限 = 最大 HP）
  曹仁铁壁   = BaseHP + 5 × IronWallStacks（最多额外 +30）
  张角黄天   = 无上限
```

### F2. 统帅约束（兵种卡上限）

```
TroopCardSlots = CommanderLeadership    // 取值范围 3~6
TroopCardCountInDeck ≤ TroopCardSlots   // 军营节点选卡时强制验证
```

### F3. 兵种倾向权重（军营节点兵种卡出现率）

```
基准权重 = 1.0（所有卡类型共享的基准值）

主修兵种（2项）：出现权重 = 基准权重 × 2.0
次修兵种（1项）：出现权重 = 基准权重 × 1.0
非倾向兵种：    出现权重 = 基准权重 × 0.5

示例：曹操（主修步/骑，次修谋）
  步兵卡 ×2.0，骑兵卡 ×2.0，谋士卡 ×1.0，弓兵卡 ×0.5，盾兵卡 ×0.5
```

### F4. 被动层数计数（以司马懿"隐忍"为例）

```
HideLayersGained = COUNT(CurseCardsAddedToHand)  // 本回合进入手牌的诅咒卡数量
HideLayers_next  = min(4, HideLayers_current + HideLayersGained)

隐忍层数持续机制：
- 隐忍层数在当回合结束时统一清零至0（非标准状态，不等同于C1状态系统的层数递减）
- 回合结束时先触发层数对应的效果，然后层数清零
- "持续一回合"的正确理解：层数在当回合内有效，回合结束后重置为0

示例时序：
  回合1抽牌阶段：抽到3张诅咒卡 → HideLayers = 3
  回合1结束阶段：触发3层效果（护甲2×3=6），然后 HideLayers = 0
  回合2开始：重新累积
```

> 其他武将被动的具体数值计算见各武将专属卡 CSV（`heroes_exclusive_decks.csv`）。

### F5. 每回合行动点（费用）

```
BaseCostPerTurn：
  夏侯惇、典韦 = 2
  大多数武将   = 3
  诸葛亮       = 4

行动点上限 = BaseCostPerTurn（与资源管理系统 F2 定义一致）

行动点规则（统一版本）：
- 战斗内每回合之间：行动点不自动重置——可累积（累积上限 = 武将上限）
- 多阶段战斗切换时：行动点重置为武将基础值（与 C2 E8 一致）
- 战斗结束后：行动点清零（与资源管理系统 F2 一致）
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **司马懿隐忍层数超过4层** | 上限4层，超出部分丢弃，不溢出 |
| E2 | **司马懿隐忍”持续一回合”：回合结束前诅咒卡已耗尽** | 隐忍层数在回合结束时统一清零，不提前结算 |
| E3 | **贾诩同一动作施加多种负面状态** | 每种新负面状态各计1层”诡谋”，多个负面按独立来源分别计算 |
| E4 | **典韦”恶来”无法获得护甲：关隘地形初始护甲** | 关隘守城方护甲效果对典韦无效，典韦在关隘依然从0护甲开始 |
| E5 | **典韦”死战”状态下嗜血回复与当前HP超过最大HP** | 回复量以最大HP为上限，不允许超过最大HP |
| E6 | **张角复活机制触发时当前最大HP已极低（如≤1）** | 复活后最大HP = floor(当前最大HP × 50%)，最低为1；HP不允许降至0以下 |
| E6b | **张角复活在多阶段战斗中** | 复活次数上限（3次）和最大HP削减效果（每次减半）在整个单场战斗内持续累积，不因阶段切换而重置 |
| E7 | **关羽”过关斩将”超过5张（5×3=+15上限）** | 卡组中每张算，但攻击伤害加成上限为+15（每张+3，最多5张生效） |
| E8 | **赵云”龙胆”闪避层数：每回合开始+1，出牌阶段闪避消耗** | 闪避在回合开始触发时结算+1，本回合内消耗不影响下回合的+1 |
| E9 | **陆逊”火种”达到3层时被净化（诅咒净化操作）** | 净化操作只对诅咒卡生效；火种是状态效果，不受净化影响 |
| E10 | **武将被动专属计数器与C1标准状态的区别** | 武将专属计数器（隐忍/诡谋/嗜血/铁壁等）是武将本地状态，不在C1标准状态池中；标准状态（虚弱/灼烧等）由C1统一管理 |
| E11 | **地图节点数低于10** | 视为编辑器配置错误，保存验证时拒绝，提示配置问题 |
| E12 | **战役中切换武将** | 每次开局绑定1名武将；战役中不支持武将切换；当前战役结束后可选新武将 |
| E13 | **兵种倾向主修武将进入不利地形（如骑兵主修进山地）** | 倾向权重只影响军营节点兵种卡出现率；地形对已入组的骑兵卡仍正常生效惩罚（-50%伤害）|

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 武将基础 HP | 40~60 | 35~70 | 战斗持久度；过低则一击即溃，过高则战斗拖沓 |
| 武将基础费用（行动点/回合） | 2~4 | 2~5 | 每回合出牌节奏；过低则手牌堆积，过高则每回合只能出1张 |
| 武将统帅（兵种卡上限） | 3~6 | 2~8 | 兵种卡流比重；过低则兵种卡无意义，过高则全组变兵种卡 |
| 武将专属卡数量 | 12~15（司马懿≤15，其他≤12） | 10~20 | 专属卡流派强度；过多则常规卡被压缩，过少则专属感不足 |
| 兵种倾向权重（主修） | ×2.0 | ×1.5~×3.0 | 军营节点倾向引导强度；过高则选卡几乎无随机性 |
| 兵种倾向权重（次修） | ×1.0 | ×0.8~×1.5 | 次修兵种相对主修的稀缺度 |
| 兵种倾向权重（非倾向） | ×0.5 | ×0.3~×0.7 | 非倾向兵种卡的稀缺度 |
| 生涯地图数量/武将 | 5 | 4~6 | 单武将重玩性；过少则很快打完，过多则制作成本陡增 |
| 每图幕数 | 3 | 3（固定） | 节奏结构；固定3幕可预期 |
| 每幕节点下限 | 10 | 10~15 | 每幕内容密度；低于10则选择感不足 |
| 司马懿隐忍层数上限 | 4 | 3~5 | 司马懿后期爆发上限；层数越高后期越强 |
| 贾诩诡谋消耗阈值 | 2层/次 | 1~3 | 贾诩费用折扣触发频率 |
| 典韦嗜血触发阈值（进入死战） | 5层 | 4~6 | 典韦进入死战的触发门槛 |
| 曹仁铁壁层数上限 | 6层（+30护盾上限） | 4~8 | 层数×5=额外护盾上限；层数×2=额外伤害 |
| 张角复活次数上限 | 3次 | 2~4 | 张角续命容错空间 |

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 每位武将仅1个被动技能，无主动技能 | 代码审查：武将配置中无 ActiveSkill 字段；UI 测试：战斗界面不显示主动技能按钮 |
| AC2 | 每阵营>=5名武将，四阵营总数>=20名 | 数据验证：`heroes_passive_skills.csv` 行数>=20，且每阵营行数符合设计 |
| AC3 | 每位武将生涯固定5张地图，每张地图固定3幕 | 地图配置测试：遍历所有武将，确认各自5张地图×3幕结构 |
| AC4 | 每幕节点数≥10 | 编辑器验证：保存时检查节点数，低于10时拒绝保存并报错 |
| AC5 | 司马懿与贾诩的被动技能存在根本性差异 | 量化验证：<br>1. 司马懿被动仅在己方手牌获得诅咒卡时触发（自我受益型），触发条件可写单元测试验证<br>2. 贾诩被动仅在敌方获得负面状态时触发（敌方影响型），触发条件可写单元测试验证<br>3. 在相同游戏情境下，选择司马懿和贾诩会导致不同的最优卡组构建策略（设计评审）<br>验证方式：单元测试 + 设计评审 |
| AC6 | 战斗初始化时正确加载武将 BaseHP、BaseCostPerTurn、HandLimit | 集成测试：选择不同武将开始战斗，确认各项初始值与配置一致 |
| AC7 | 兵种卡数量超过统帅上限时军营节点禁止加入 | 军营节点测试：统帅=3 时第4张兵种卡按钮禁用/灰显 |
| AC8 | 典韦无法通过任何途径获得护甲 | 战斗测试：典韦进入关隘节点，确认初始护甲=0；装备测试：给予护甲装备，确认不生效 |
| AC9 | 张角复活最多3次，每次复活后最大HP减半 | 战斗测试：让张角连续被击倒4次，确认第4次真正死亡且前3次最大HP正确减半 |
| AC10 | 司马懿隐忍层数上限4层，不溢出 | 单元测试：回合内5张诅咒卡进入手牌，确认隐忍=4而非5 |
| AC11 | 兵种倾向权重正确影响军营节点卡牌出现率 | 统计测试：曹操在军营节点抽样100次，骑兵/步兵卡出现率约为其他兵种类的2倍 |
