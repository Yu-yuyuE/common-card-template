# 兵种卡设计文档（Troop Cards Design）

_Updated: 2026-04-03_
_Status: Draft v2（扩展兵种池v3 — 关隘/雪地/沙漠地形补全，共41种兵种卡）_

---

## Overview

本文档定义《三国称雄》中**兵种卡**的完整规则、数值设计与交互机制。

兵种卡是玩家通过军营节点选入卡组的特殊卡牌，与武将专属卡和常规卡共处一个卡组。兵种卡的数量上限由武将的**统帅值（3~6）**决定。玩家在战斗中主动打出兵种卡，按卡牌文本结算效果后进入弃牌堆（与普通卡相同）。

共5种兵种类型：**弓兵、步兵、骑兵、谋士、盾兵**，每种支持一次升级（Lv1→Lv2）。

---

## Player Fantasy

- **兵种构筑流**：根据武将统帅值与地形预判，选择最适合当局战场的兵种组合，形成"武将主动技 + 兵种辅战"的协同体验。
- **前排/后排博弈**：利用骑兵击退改变敌方阵型，配合其他卡牌打出换位红利。
- **防御反击流**：盾兵叠护盾延续主将生命，配合坚守状态构筑高防构筑。
- **弱化控制流**：谋士施加随机Debuff扰乱敌方节奏，为后续爆发铺路。

## Detailed Rules

### 1. 兵种卡通用规则

| 规则项             | 说明                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| **获取方式**       | 军营节点选择，加入玩家卡组                                           |
| **卡组上限**       | 卡组中兵种卡总数 ≤ 武将统帅值（3~6），超限时军营节点该选项灰显不可选 |
| **打出方式**       | 玩家主动打出，按卡牌文本结算效果                                     |
| **打出后去向**     | 进入弃牌堆（与普通卡相同，参与正常循环）                             |
| **伤害类型**       | 所有兵种卡造成的伤害均受护甲影响                                     |
| **升级机制**       | 每种兵种卡支持且仅支持一次升级（Lv1→Lv2），不可重复升级             |
| **升级后保留身份** | 升级后保留原卡身份与标签，仅提升效果，不改变核心定位                 |

### 2. 五种兵种卡完整数值表

| 兵种 | 费用 | Lv1 效果                           | Lv2 效果                                        |
| ---- | ---- | ---------------------------------- | ----------------------------------------------- |
| 弓兵 | 1费  | 对任意目标造成 **7点伤害**（受护甲影响）| 对**所有**敌方目标各造成 **7点伤害**（受护甲影响）|
| 步兵 | 0费  | 对目标造成 **8点伤害**              | 对目标造成 **6点伤害，攻击两次**（共12点，受护甲影响）|
| 骑兵 | 1费  | 对目标造成 **5点伤害**，附带**击退**| 对目标造成 **5点伤害**，附带**击退+1层眩晕**       |
| 谋士 | 1费  | 对任意目标造成 **7点伤害**              | 对任意目标造成 **7点伤害**，施加 **1个随机Debuff**      |
| 盾兵 | 1费  | 获得 **8点护盾**          | 获得 **8点护盾** + **1层坚守**       |

> **护盾（护甲）** = 消耗型护盾池，优先抵挡传入伤害；护盾耗尽后剩余伤害扣除主将HP。
> **坚守** = 持续状态，每层持续1回合，效果：受到伤害-50%（详见 `status-design.md` B4）。
> **眩晕** = 持续状态，每层持续1回合，效果：目标跳过本回合行动（详见 `status-design.md`）。
> **随机Debuff** = 从当前战斗可用的全部Debuff池中随机选取1个，以1层施加给目标。

### 3. 击退机制详细规则

**击退（Knockback）** 是一种位置操作，不是状态效果。

| 规则项             | 说明                                                                     |
| ------------------ | ------------------------------------------------------------------------ |
| **触发条件**       | 骑兵卡命中目标后自动触发                                                 |
| **效果**           | 目标敌将被推至后排                                                       |
| **换位逻辑**       | 若目标已在后排，无效果；若后排已有其他敌将，则两者**互换位置**           |
| **前排/后排定义**  | 由地图/战场系统定义（此文档不做扩展，后续由 `map-design.md` 补充）       |
| **与其他效果交互** | 击退先于同卡牌后续效果结算（骑兵Lv2：先击退，后施加眩晕）               |
| **无法击退的情况** | 如目标有「稳固」类免疫状态，击退无效（由状态系统裁定）                   |

---

### 4. 护盾（护甲）机制详细规则

| 规则项             | 说明                                                                         |
| ------------------ | ---------------------------------------------------------------------------- |
| **护盾来源**       | 盾兵卡、部分武将技能/卡牌                                                    |
| **护盾叠加**       | 多次获得护盾时，数值**累加**（护盾池总量增加）                               |
| **护盾扣除顺序**   | 传入伤害**优先**消耗护盾池；护盾耗尽后，剩余伤害扣除主将HP                  |
| **护盾与坚守叠加** | 先扣护盾，护盾归零后受到的伤害再经过坚守（-50%）减伤后扣HP（见 Edge Case 1）|
| **护盾不跨战斗**   | 每场战斗开始时护盾清零（除非武将被动特殊说明）                               |
| **护盾上限**       | 无全局上限（待平衡验证后配置）                                               |

## Formulas

### F1. 护盾伤害吸收

```
EffectiveDamage = max(0, IncomingDamage - ShieldPool)
ShieldPool_next = max(0, ShieldPool - IncomingDamage)
HP_next = HP_now - EffectiveDamage
```

示例：护盾池 = 8，受到12点伤害 → 护盾归零，HP扣除 4点。

### F2. 护盾 + 坚守叠加减伤

```
Step1: ShieldAbsorb = min(ShieldPool, IncomingDamage)
Step2: Overflow = IncomingDamage - ShieldAbsorb
Step3: FinalDamage = Overflow × (1 - 0.50)   // 坚守生效
HP_next = HP_now - FinalDamage
ShieldPool_next = ShieldPool - ShieldAbsorb
```

示例：护盾池 = 8，坚守1层，受到20点伤害 → 护盾吸收8点，剩余12点经坚守减半 = 6点扣HP。

### F3. 步兵Lv2 双击伤害

```
TotalDamage = BaseDamage × 2   // 8 × 2 = 16
每次攻击独立走护甲扣除逻辑
```

> 两次攻击各自独立计算护甲，不合并为一次16点计算。
> 示例：护甲4点，步兵Lv2攻击 → 第一次 8-4=4点伤害（护甲耗尽），第二次 8点伤害（护甲已无）= 共12点HP伤害。

### F4. 弓兵Lv2 全体伤害

```
ForEach enemy in AllEnemies:
    EffectiveDamage = max(0, BaseDamage - enemy.ShieldPool)
    // 每个敌将独立计算护甲吸收
```

## Edge Cases

1. **护盾与坚守同时生效**：护盾优先吸收，溢出伤害再经坚守减半后扣HP（见 F2）。不可颠倒顺序。
2. **击退目标已在后排**：击退无效，不产生位置变化，也不触发其他衍生效果（骑兵Lv2眩晕仍正常施加）。
3. **击退时后排有多名敌将**：与最近一名后排敌将换位，具体"最近"定义由战场系统决定（待 `map-design.md` 补充）。
4. **步兵Lv2双击时目标在第一击死亡**：第二次攻击无效（无目标），不溢出到其他敌将。
5. **弓兵Lv2对单一敌将的战场**：效果等同于Lv1（只有1个目标，单体7点伤害）。
6. **谋士Lv2随机Debuff为"混乱"**：混乱会使目标下次攻击打友军；若敌方只有1名武将，混乱的攻击无友军可打则无效。
7. **护盾叠加上限**：暂无上限，但单局护盾总量超过武将最大HP的2倍时（即 ≥ 100），视为平衡异常，需记录。
8. **盾兵卡打出时主将已满血且无坚守**：护盾正常添加，不因满血而失效。
9. **随机Debuff池为空**（理论不可能，但需处理）：谋士Lv2跳过施加Debuff步骤，仍造成7点伤害。

## Acceptance Criteria

1. 卡组中兵种卡数量超过对应武将统帅值时，军营节点兵种卡选项灰显且无法选择。
2. 步兵Lv2打出后，目标承受**两次独立的8点伤害**计算（护甲分别独立扣除，不合并）。
3. 弓兵Lv2打出后，**所有**场上敌将均承受7点伤害（每人独立计算护甲）。
4. 骑兵打出后，目标被推至后排；若后排已有敌将，两者互换位置；骑兵Lv2同时对目标施加1层眩晕。
5. 盾兵打出后，我方主将护盾池数值增加8；若已有护盾，则累加（不覆盖）。
6. 盾兵Lv2打出后，我方主将同时获得1层坚守状态，与护盾效果同回合共存。
7. 谋士Lv2打出后，目标随机获得1个Debuff（从当前Debuff池随机抽取，层数为1）。
8. 护盾存在时，传入伤害**优先扣护盾**，护盾耗尽后剩余伤害才扣HP。
9. **守城步兵**：关隘地形下护盾量≥12（Lv1）且触发反击效果。
10. **铁甲重骑**：关隘地形下骑兵伤害-50%惩罚被完全取消，按正常伤害值结算。
11. **沙漠游骑**：沙漠地形下费用为0且打出后施加灼烧；若目标已有灼烧，额外穿透伤害触发。
12. **沙漠盾卫**：沙漠地形下打出时清除我方所有灼烧层（Lv1）；Lv2额外施加坚守。
13. **关城守弩**：关隘地形下打出时效果从单体扩展为对所有敌将造成伤害。
14. **雪地三卡**（寒地猎手/雪原猎弓/寒冬军师）：雪地地形下效果升级，且不受雪地全体伤害-20%的衰减（作为地形专属卡，免疫对应地形的负面修正）。
15. 七大地形（平原/山地/森林/水域/关隘/雪地/沙漠）各至少有3张兵种卡具有专属联动效果。
