## Overview

兵种卡系统（Troop Card System）定义《三国称雄》中5大兵种类型（弓兵/步兵/骑兵/谋士/盾兵）的三层升级路径与地形/天气联动机制。基础兵种（Lv1）在军营节点加入卡组，可升级为高级兵种（Lv2），再于军营节点选择分支升级为特殊精锐（Lv3）。兵种卡数量上限受武将**统帅值（3~6）**约束，是构筑层（卡组构成）与地图层（地形/天气环境）之间的核心连接点。

**系统职责边界：**
- 负责：兵种卡三层升级路径定义（Lv1→Lv2→Lv3分支）、各级效果数值、7大地形×4天气联动修正规则、击退位置操作规则、护盾+坚守叠加伤害计算
- 不负责：护盾池的全局生命周期管理（F2/C2）、状态效果的持续/消除/叠加规则（C1）、军营节点兵种卡出现率（由D3武将兵种倾向权重驱动）、前排/后排位置底层定义（C2战场模型）

## Player Fantasy

**"这支军队是我的，我按战场来选兵，不是按兵选战场。"**

**① 构筑选择感（策略层）**
进入军营节点时，玩家面对的不是随机奖励，而是战略决策：再加一张骑兵凑满平原爆发，还是换一张谋士为后面的山地关卡做准备？统帅值上限（3~6）使每个兵种卡槽位都是真实的机会成本，选了这张就意味着放弃另一张。

**② 地形匹配感（叙事/策略层）**
看到下一张地图是山地，就感受到"是时候换陌刀骑压制骑兵了"；在沙漠地图拿到沙漠游骑，觉得"这张卡终于等到了它的主场"。地形与兵种的匹配是策略深度的核心来源，也是武将生涯5张地图差异化的体现。

**③ 协同爆发感（战斗层）**
骑兵击退把目标推到后排 → 戟兵前排清场 → 火矢兵跟进引燃：兵种卡之间的组合打出"职业军队"协同作战的感觉，而不是各自独立的技能按钮。

**不应有的感受：**
- 兵种卡只是伤害数字不同的换皮攻击（每种兵种需在某些情境下明显优于其他）
- 统帅上限让人感觉受到惩罚（应感觉像"专精路线"而非"被限制"）
- 地形效果是随机惊喜，而不是可预期的战略规划

## Detailed Rules

### 1. 兵种卡通用规则

| 规则项             | 说明                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| **获取方式**       | 军营节点选择，加入玩家卡组                                           |
| **卡组上限**       | 卡组中兵种卡总数 ≤ 武将统帅值（3~6），超限时军营节点该选项灰显不可选 |
| **打出方式**       | 玩家主动打出，按卡牌文本结算效果                                     |
| **打出后去向**     | 进入弃牌堆（与普通卡相同，参与正常循环）                             |
| **伤害类型**       | 所有兵种卡造成的伤害均受护甲影响                                     |
| **升级机制**       | 每种兵种卡支持三层升级路径：Lv1（基础兵种）→ Lv2（高级兵种）→ Lv3（特殊精锐分支，每种基础兵种有5个以上分支可选） |
| **统帅占用**       | 一张兵种卡无论处于 Lv1 / Lv2 / Lv3，始终只占 **1个统帅槽**，升级不额外占用 |
| **升级后保留身份** | 升级后保留原卡身份与标签，仅提升效果，不改变核心定位                 |

---

### 1a. 三层升级路径

```
Lv1（基础兵种）→ Lv2（高级兵种）→ Lv3（特殊精锐分支）
```

| 层级 | 定义 | 获取方式 | 代表卡例 |
|------|------|---------|---------|
| **Lv1** | 基础兵种，核心功能明确，费用低廉 | 军营节点首次选择加入卡组 | 步兵、骑兵、弓兵、谋士、盾兵 |
| **Lv2** | 高级兵种，效果增强或新增附加机制 | 铁匠铺节点升级已有Lv1卡 | 步兵Lv2（双击）、骑兵Lv2（眩晕）等 |
| **Lv3** | 特殊精锐，分支化专精路线，具备独特地形/战术联动 | 军营节点对**已有Lv2卡**选择分支方向升级 | 虎贲卫、虎豹骑、关城守弩等 |

**各基础兵种的Lv3分支数量：**

| 基础兵种 | Lv3分支数 | 分支卡列表 |
|---------|---------|-----------|
| 步兵 | 5 | 虎贲卫、刀盾手、戟兵、山地步兵、游侠 |
| 骑兵 | 5 | 轻骑兵、火骑兵、象兵、虎豹骑、陌刀骑 |
| 弓兵 | 6 | 连弩兵、火矢兵、投石兵、弩车兵、火兵、猎人 |
| 谋士 | 5 | 细作、方士、刺客、舞女、黄巾信徒 |
| 盾兵 | 5 | 重甲盾兵、水兵、壁垒工兵、藤甲兵、铁甲步兵 |

> 弓兵因内容丰富性保留6个Lv3分支，详见 Edge Case 14。

---

### 1b. Lv2→Lv3 分支选择规则

| 规则项 | 说明 |
|--------|------|
| **触发时机** | 玩家到达**军营节点**时，系统检查卡组中是否有Lv2兵种卡 |
| **界面展示** | 对每张Lv2卡，弹出该兵种对应的全部Lv3分支预览（含名称、效果概览、地形联动提示） |
| **选择规则** | 玩家从弹出预览中选择**一个**分支方向，当前Lv2卡升级为该Lv3分支卡（不可同一节点为同一张卡选择多个分支） |
| **每节点限制** | 每个军营节点可对**所有**符合条件的Lv2卡各选择一次分支（卡数量不限，但每张卡只能选一次） |
| **跳过选项** | 玩家可主动选择"暂不升级"，保留该Lv2卡，待下次军营节点再选择 |
| **分支唯一性** | 一张卡升级为某Lv3分支后，该分支不可更改（详见 Edge Case 16） |

---

### 1c. Lv3分支卡费用规则

| 规则项 | 说明 |
|--------|------|
| **基础费用继承** | Lv3分支卡费用**默认继承**对应基础兵种Lv1的费用 |
| **专精溢价** | 高战斗力或高专精度的分支卡费用可比基础兵种 **+1费**（上限+1，不可超过） |
| **典型示例** | 虎贲卫（步兵系）→ 1费（步兵基础0费 +1）；虎豹骑（骑兵系）→ 2费（骑兵基础1费 +1）；连弩兵（弓兵系）→ 2费（弓兵基础1费 +1） |
| **零费兵种例外** | 游侠/山地步兵（步兵系0费分支）维持0费，不因分支化而强制加费 |

### 2. 五种兵种卡完整数值表

| 兵种 | 费用 | Lv1 效果                           | Lv2 效果                                        |
| ---- | ---- | ---------------------------------- | ----------------------------------------------- |
| 弓兵 | 1费  | 对任意目标造成 **7点伤害**（受护甲影响）| 对**所有**敌方目标各造成 **7点伤害**（受护甲影响）|
| 步兵 | 0费  | 对目标造成 **8点伤害**              | 对目标造成 **6点伤害，攻击两次**（共12点，受护甲影响）|
| 骑兵 | 1费  | 对目标造成 **5点伤害**，附带**击退**| 对目标造成 **5点伤害**，附带**击退+1层眩晕**       |
| 谋士 | 1费  | 对任意目标造成 **7点伤害**              | 对任意目标造成 **7点伤害**，施加 **1个随机Debuff**      |
| 盾兵 | 1费  | 获得 **8点护盾**          | 获得 **8点护盾** + **1层坚守**       |

> **护盾（护甲）** = 消耗型护盾池，优先抵挡传入伤害；护盾耗尽后剩余伤害扣除HP。
> **坚守** = 持续状态，每层持续1回合，效果：受到伤害-25%（详见 `status-design.md` B4）。
> **眩晕** = 持续状态，每层持续1回合，效果：目标跳过本回合行动（详见 `status-design.md`）。
> **随机Debuff** = 从当前战斗可用的全部Debuff池中随机选取1个，以1层施加给目标。

### 3. 击退机制详细规则

**击退（Knockback）** 是一种位置操作，不是状态效果。

| 规则项             | 说明                                                                     |
| ------------------ | ------------------------------------------------------------------------ |
| **触发条件**       | 骑兵卡命中目标后自动触发                                                 |
| **效果**           | 目标敌将被推至后排                                                       |
| **换位逻辑**       | 若目标已在后排，无效果；若后排已有其他敌将，则两者**互换位置**           |
| **前排/后排定义**  | 由地图/战场系统定义（此文档不做扩展，后续由 `map-design.md` 补充）       |
| **与其他效果交互** | 击退先于同卡牌后续效果结算（骑兵Lv2：先击退，后施加眩晕）               |
| **无法击退的情况** | 如目标有「稳固」类免疫状态，击退无效（由状态系统裁定）                   |

### 4. 护盾（护甲）机制详细规则

| 规则项             | 说明                                                                         |
| ------------------ | ---------------------------------------------------------------------------- |
| **护盾来源**       | 盾兵卡、部分武将技能/卡牌                                                    |
| **护盾叠加**       | 多次获得护盾时，数值**累加**（护盾池总量增加）                               |
| **护盾扣除顺序**   | 传入伤害**优先**消耗护盾池；护盾耗尽后，剩余伤害扣除HP                  |
| **护盾与坚守叠加** | 先用坚守（-25%）计算最终伤害，然后再扣除护盾，护盾归零后剩余伤害扣HP（见 Edge Case 1）|
| **护盾不跨战斗**   | 每场战斗开始时护盾清零（除非武将被动特殊说明）                               |
| **护盾上限**       | 遵循卡牌战斗系统（C2）定义：默认=MaxHP，曹仁=MaxHP+30，张角无上限                                               |

## Formulas

### F1. 护盾伤害吸收

```
EffectiveDamage = max(0, IncomingDamage - ShieldPool)
ShieldPool_next = max(0, ShieldPool - IncomingDamage)
HP_next = HP_now - EffectiveDamage
```

示例：护盾池 = 8，受到12点伤害 → 护盾归零，HP扣除 4点。

### F2. 护盾 + 坚守叠加减伤

```
Step1: FinalDamage = IncomingDamage × (1 - 0.25)   // 坚守生效
Step2: ShieldAbsorb = min(ShieldPool, FinalDamage)
Step3: HP_Damage = FinalDamage - ShieldAbsorb
HP_next = HP_now - HP_Damage
ShieldPool_next = ShieldPool - ShieldAbsorb
```

示例：护盾池 = 8，坚守1层，受到20点伤害 → 伤害先被坚守减免至 15 点，护盾吸收 8 点后破盾，剩余 7 点扣除HP。

### F3. 步兵Lv2 双击伤害

```
TotalDamage = BaseDamage × 2   // 8 × 2 = 16
每次攻击独立走护甲扣除逻辑
```

> 两次攻击各自独立计算护甲，不合并为一次16点计算。
> 示例：护甲4点，步兵Lv2攻击 → 第一次 8-4=4点伤害（护甲耗尽），第二次 8点伤害（护甲已无）= 共12点HP伤害。

### F5. 地形/天气伤害修正

```
// 地形修正（兵种卡打出时判断，不追溯已打出的卡）
BaseDamage_modified = BaseDamage × TerrainMultiplier

TerrainMultiplier：
  卡牌有对应地形专属联动 → 按卡牌文本单独定义（通常 ×1.4~2.0）
  无联动 + 不利地形惩罚   → ×0.5（如骑兵在山地/关隘）
  无联动 + 中性地形       → ×1.0

// 天气修正（叠加在地形修正之上）
弓兵类（含谋士远程效果）：
  雨天 → ×0.5（灼烧消减，不利于引燃后续）
  雾天 → ×0.5（盲目状态：弓兵命中率下降，等效伤害-50%）
  晴天/风天 → ×1.0

近战类（步兵/骑兵/盾兵）：
  雾天 → ×1.0（盲目不影响近战，无额外修正）
  其他天气 → ×1.0

> 雾天的"盲目"状态（2层，战斗开始施加）由 terrain-weather-system.md 定义；
> 此处 ×0.5 是盲目效果对弓兵卡的等效数值表述，不是独立的伤害修正。

// 灼烧扩散（风天特殊规则）
风天：回合结束时，场上每个有灼烧的目标将1层灼烧扩散至随机相邻目标
```

### F6. 直击（穿透伤害）计算

```
// "直击"关键字：无视护盾，直接扣HP
PenetrationDamage = FixedValue（按卡牌文本）
HP_next = HP_now - PenetrationDamage
// 护盾池不变，不参与此次计算
```

> 直击与普通伤害在同一卡上出现时（如猎人"额外直击4点"），分两步结算：先走护盾计算，再走直击计算。

### F7. 升级路径层级判定

```
// 系统判断兵种卡当前层级（用于军营节点UI展示与分支解锁）
CardTier(card):
  if card.upgradeCount == 0 → Lv1（基础兵种）
  if card.upgradeCount == 1 → Lv2（高级兵种，Lv1→Lv2已完成）
  if card.isBranchCard    → Lv3（分支精锐，Lv2→Lv3已完成）

// 军营节点：对每张Lv2卡展示分支选项
ShowBranchOptions(card):
  if CardTier(card) == Lv2 → 展示 BranchPool[card.baseType]（该兵种的全部Lv3分支）
  else → 不展示分支UI

// 统帅槽占用：无论Lv几，始终1槽
CommandSlotCost(card) = 1   // 不因升级而变化

// 升级总次数上限
MaxUpgradeCount = 2   // Lv1→Lv2算第1次；Lv2→Lv3算第2次；超出不可再升级
```

> 设计要点：统帅槽占用全程固定1槽，是为了让"专精精锐路线"感觉是**主动升华**而非**额外负担**。
> 升级收益系数（1.20~1.35）适用于Lv1→Lv2（铁匠铺节点）；Lv2→Lv3为分支替换，不套用同一系数——分支卡的效果由卡牌文本直接定义，无通用放大倍率。

## Edge Cases

1. **护盾与坚守同时生效**：先计算坚守减伤，减伤后的最终伤害再由护盾吸收，护盾耗尽后的剩余伤害扣HP（见 F2）。不可颠倒顺序。
2. **击退目标已在后排**：击退无效，不产生位置变化，也不触发其他衍生效果（骑兵Lv2眩晕仍正常施加）。
3. **击退时后排有多名敌将**：与最近一名后排敌将换位，具体"最近"定义由战场系统决定（待 `map-design.md` 补充）。
4. **步兵Lv2双击时目标在第一击死亡**：第二次攻击无效（无目标），不溢出到其他敌将。
5. **弓兵Lv2对单一敌将的战场**：效果等同于Lv1（只有1个目标，单体7点伤害）。
6. **谋士Lv2随机Debuff为"混乱"**：混乱会使目标下次攻击打友军；若敌方只有1名武将，混乱的攻击无友军可打则无效。
7. **护盾叠加上限**：遵循卡牌战斗系统（C2）定义：默认=MaxHP，曹仁=MaxHP+30，张角无上限。超出上限时丢弃超额部分。
8. **盾兵卡打出时已满血且无坚守**：护盾正常添加，不因满血而失效。
9. **随机Debuff池为空**（理论不可能，但需处理）：谋士Lv2跳过施加Debuff步骤，仍造成7点伤害。
10. **地形修正结算时机**：兵种卡打出瞬间判断当前地形和天气，结算时使用该值；天气中途切换（如诸葛亮借东风）不追溯同回合已结算的卡牌效果；已入手牌的卡牌以打出时的地形/天气为准。
11. **武将兵种倾向权重 vs 卡牌战斗效果**：倾向权重仅影响军营节点的卡牌出现率，对卡牌在战斗中的实际效果无任何影响（倾向弓兵的武将打出的弓兵卡，效果与其他武将打出相同）。
12. **Lv2升级后是否保留地形联动**：升级后完整保留全部地形/天气联动效果；升级不会移除、削弱或修改任何联动条件，仅在原联动基础上增加新效果。
13. **同名兵种卡重复入组**：允许卡组中存在多张同名兵种卡（例如两张"骑兵"），受统帅值上限约束；打出效果各自独立结算，护盾效果累加，不产生特殊叠加规则。
14. **弓兵Lv3有6个分支（非标准5个）**：弓兵Lv3保留连弩兵、火矢兵、投石兵、弩车兵、火兵、猎人共6个分支，比其他兵种多1个。这是为了保留猎人的"状态收割"特色定位（与其他5个弓兵差异明显，专注于Debuff触发直击，无地形专属联动，是通用弓兵流的爆发收尾卡）。设计上视为弓兵类专精内容更丰富的例外，军营节点展示6个分支预览，规则不变。
15. **盾兵第5个Lv3分支——铁甲步兵**：盾兵Lv3新增分支"铁甲步兵"，历史原型为三国重装铁甲步卒（曹魏虎豹骑步兵护卫队形象）。效果如下：<br>**费用：2费**<br>**Lv3效果：** 我方获得 **10点护盾**，施加**嘲讽**状态（持续2回合：敌方所有攻击类卡牌必须以我方该武将为目标，无法指向其他我方单位）。若嘲讽结束时护盾仍存在，额外获得 **1层坚守**。<br>**设计意图：** 嘲讽效果使铁甲步兵成为盾兵大类中唯一的"仇恨集中"式坦克，配合护盾+坚守体系在多武将局面下主动吸收敌方火力，保护输出位武将。克制方向：直击穿透（投石兵/刺客类）可绕过护盾直接扣HP；消除嘲讽的技能（如舞女迷乱）可提前终止嘲讽效果。
16. **Lv3分支选择后能否反悔**：**不可更改**。一旦在军营节点确认选择某Lv3分支，该卡即替换为分支卡，本局不可撤销。UI层需在选择前展示确认弹窗，告知玩家"此选择本局内不可更改"，避免误操作。若玩家在军营节点选择"暂不升级"，保留Lv2卡，下一个军营节点可重新选择。
17. **Lv3分支卡是否可继续升级**：Lv3是终点，分支卡不可再升级（upgradeCount最大为2，参见F7）。铁匠铺节点对已是Lv3的卡不展示升级选项，该槽位显示灰色锁定标识。

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 卡组中兵种卡数量超过对应武将统帅值时，军营节点兵种卡选项灰显且无法选择 | UI测试：统帅=3 时第4张兵种卡按钮不可点击；尝试通过API强制加入应被服务器端拒绝 |
| AC2 | 步兵Lv2打出后，目标承受**两次独立的8点伤害**计算（护甲分别独立扣除，不合并）| 单元测试：目标护盾=4，步兵Lv2攻击 → 第一次 8-4=4伤+护盾归零，第二次完整8伤=共12HP伤害 |
| AC3 | 弓兵Lv2打出后，**所有**场上敌将均承受7点伤害（每人独立计算护甲）| 战斗测试：3名敌将各有不同护盾值，确认各自独立计算，互不影响 |
| AC4 | 骑兵打出后目标被推至后排；若后排已有敌将，两者互换位置；骑兵Lv2同时对目标施加1层眩晕 | 位置测试：前排A、后排B，骑兵命中A → A到后排，B到前排；Lv2时A有1层眩晕，B无眩晕 |
| AC5 | 盾兵打出后，我方护盾池数值增加8；若已有护盾，则累加（不覆盖）| 数值测试：现有护盾5，打出盾兵 → 护盾=13 |
| AC6 | 盾兵Lv2打出后，我方同时获得1层坚守状态，与护盾效果同回合共存 | 状态测试：打出盾兵Lv2 → 护盾+8，坚守层=1；坚守和护盾同时活跃，受到伤害时先扣护盾再减免25% |
| AC7 | 谋士Lv2打出后，目标随机获得1个Debuff（从当前Debuff池随机抽取，层数为1）| 功能测试：打出谋士Lv2 × 10次，记录施加的Debuff类型分布，确认多种Debuff均有出现 |
| AC8 | 护盾存在时，传入伤害**优先扣护盾**，护盾耗尽后剩余伤害才扣HP | 单元测试：护盾=8，受到12伤 → 护盾归零，HP-4（非HP-12）|

| AC10 | **铁甲重骑**：关隘地形下骑兵伤害-50%惩罚被完全取消，按正常伤害值结算 | 地形测试：普通骑兵关隘=5伤（×0.5=2.5→2）；铁甲重骑关隘=10伤（无惩罚，完整结算）|
| AC11 | **沙漠游骑**：沙漠地形下费用为0且打出后施加灼烧；若目标已有灼烧，额外穿透伤害触发 | 地形测试：沙漠地形打出 → 消耗0费，目标受击退+灼烧；再次打出时对已有灼烧目标确认额外直击4伤 |
