# C2 — 卡牌战斗系统（Card Battle System）

_Created: 2026-04-07_
_Status: Complete_
_Priority: MVP / 核心层_

---

## Overview

卡牌战斗系统是三国称雄的核心引擎，定义战斗的完整生命周期——从战斗初始化、手牌管理、出牌结算，到回合循环、胜负判定与战斗结束清算。系统承载"我方 1 主将 vs 敌方最多 3 主将"的 1vN 战场结构，统一管理卡牌的抽取/使用/丢弃/移除全流程，并为地形天气、兵种卡、武将被动、诅咒机制提供钩子接口。卡牌战斗系统不直接处理具体的伤害数值或状态定义——这些由资源管理系统（F2）与状态效果系统（C1）负责——它的职责是**结算顺序的权威仲裁者**：规定什么时候、以什么顺序、由谁执行哪种效果。

---

## Player Fantasy

**"我是三军主将，手中每张牌都是我的决策。"**

玩家在战斗中应感受到三种核心幻想：

**① 运筹帷幄感（策略层）**
面对 1 vs 3 的压力战场，玩家通过手牌规划、行动点分配和出牌顺序逐步瓦解敌阵。同时面对三名公开意图的敌将，决定"先打谁、用什么牌打、留什么牌防"是每回合的核心决策张力。

**② 地利与博弈感（战术层）**
兵种卡不是通用伤害输出，而是有条件的战术工具——骑兵卡在平原上事半功倍，弓兵卡在雨天选择保留。地形天气是可读懂、可利用的规律，而非随机噪声。

**③ 武将身份感（叙事层）**
不同武将的专属卡组让每局战斗风格迥异：曹操的虚弱战术（兵种卡叠 Debuff、攻击收割）、诸葛亮的谋略控制、司马懿的诅咒反制。打出一张专属技能卡时，应感受到"这就是我的武将在战场上的个性"。

**不应有的感受：**
- ❌ 被随机性愚弄（运气大于策略）
- ❌ 不知道为什么输了（伤害来源不透明）
- ❌ 1 vs 3 的数量劣势感（应是挑战，而非绝望）

## Detailed Rules

### 3.1 战场结构

| 位置 | 内容 |
|------|------|
| 我方 | 主将位 ×1（玩家操控） |
| 敌方 | 敌将位 ×1~3（当前阶段配置，同时在场） |
| 战场 | 地形标签 ×1 + 天气标签 ×1（节点预设，可动态切换） |

- 敌将按**从左到右**排列，行动顺序同左到右
- 敌将各自拥有独立的 HP、护盾、状态

---

### 3.2 多阶段战斗

部分节点（精英/Boss）由多个**阶段**组成，每个阶段是一批独立的敌人：

| 字段 | 规则 |
|------|------|
| **阶段数** | 前期节点 1 阶段；后期精英/Boss 节点可有 2~3 阶段 |
| **切换条件** | 当前阶段所有敌人 HP ≤ 0 → 自动进入下一阶段 |
| **玩家资源** | HP、护盾、行动点、手牌**全部保留**，无任何重置 |
| **状态保留** | 玩家身上的所有 Buff/Debuff 保留并继续倒计时 |
| **地形天气** | 保持不变（除非新阶段配置中明确切换） |
| **阶段提示** | 进入新阶段时显示提示（"第 2 阶段"），公示新敌人意图 |

> **设计意图**：多阶段战斗是持久战考验，资源不重置意味着前期节省的 HP 和手牌会在后阶段成为优势——玩家的跨阶段资源管理本身就是战术决策。

---

### 3.3 战斗初始化

1. 读取节点配置：阶段数、各阶段敌人列表、地形、天气
2. 地形/天气初始效果立即结算（如关隘给守方护甲、雾天给全体盲目）
3. 玩家卡组洗牌
4. 抽初始手牌（补至手牌上限）
5. 行动点设为武将基础行动点上限（满值开始）
6. 护盾清零（战斗开始无护盾，除非地形/装备给予）

---

### 3.4 回合流程（严格顺序）

```
【玩家回合】
  1. 回合开始触发（武将被动、状态触发等）
  2. 抽牌（补至手牌上限：默认5张，袁绍6张，装备效果可能扩展）
  3. 出牌阶段（玩家自由出牌，直到行动点耗尽或主动结束）
  4. 回合结束触发（状态层数-1、DoT结算、武将被动等）

【敌人回合】
  5. 敌人按左→右顺序各执行1个行动
  6. 敌人行动后公示下一回合意图

【阶段检查】
  7. 若当前阶段所有敌人 HP ≤ 0 → 进入下一阶段（或进入胜利结算）
  8. 否则回到步骤 1
```

**手牌上限：**

| 情况 | 手牌上限 |
|------|---------|
| 默认（所有武将） | 5 |
| 袁绍（四世三公被动） | 6 |
| 装备效果扩展 | 待设计 |

> 每回合开始时补足至当前手牌上限。若手牌已达上限，本回合不抽牌。

---

### 3.5 卡牌生命周期

卡牌在战斗中存在于以下五个区域之一：

| 区域 | 说明 |
|------|------|
| **卡组（抽牌堆）** | 洗牌后的待抽区，顺序随机 |
| **手牌** | 当前可出牌区，上限 = 武将手牌上限 |
| **弃牌堆** | 使用后的普通归宿，抽牌堆耗尽时洗回 |
| **移除区** | 本场战斗期间不参与循环，战斗结束后回卡组 |
| **消耗区** | 永久消耗，整个存档周期内不可再用，除非再次获得该卡 |

**卡牌流转规则：**
- 普通打出 → 弃牌堆
- 标注"使用后移除" → 移除区（战斗结束后回卡组）
- 标注"消耗" → 消耗区（永久消耗，除非再次获得）
- 抽牌堆为空时 → 弃牌堆洗牌成新抽牌堆
- 手牌溢出上限时 → 溢出的牌强制丢弃至弃牌堆

---

### 3.6 出牌规则

- 费用不足时无法打出该卡（灰显）
- 每张卡打出后立即结算效果，再进入对应区域
- **兵种卡**：卡组中兵种卡总数 ≤ 武将统帅值；打出后进弃牌堆；效果按卡面文本结算；支持地形/天气联动加成
- **诅咒卡**：部分类型抽入手牌时立即触发效果（详见诅咒系统 GDD）
- 同一回合可连续打出多张卡，无出牌次数上限（仅受行动点约束）

---

### 3.7 胜负判定

| 条件 | 结果 |
|------|------|
| 最后阶段所有敌将 HP ≤ 0 | **玩家胜利**，进入战斗结算 |
| 玩家主将 HP ≤ 0 | **玩家失败**，本局结束 |

**战斗结算（胜利后）：**
1. 护盾清零
2. 行动点清零
3. 移除区的卡回归卡组
4. 发放奖励（金币/卡牌/粮草，由节点类型决定）
5. HP 保留当前值，带入下一节点

## Formulas

### F1. 攻击伤害结算

```
有效伤害 = BaseDamage × BuffMod × DebuffMod

护盾先扣：
  若 Shield > 0：
    Shield_next = max(0, Shield - 有效伤害)
    HP溢出伤害  = max(0, 有效伤害 - Shield)
    HP_next     = HP - HP溢出伤害
  若 Shield = 0：
    HP_next = HP - 有效伤害

带"无视护甲"关键词：
    HP_next = HP - 有效伤害（跳过护盾）
```

---

### F2. 兵种卡地形/天气伤害修正

```
最终伤害 = BaseDamage × TerrainMod × WeatherMod × BuffMod
```

> 出牌结算时取当前地形天气快照，动态切换在切换后的下一次结算生效。

---

### F3. 手牌抽取数量

```
DrawCount = max(0, HandLimit - CurrentHandSize)

HandLimit：
  默认武将 = 5
  袁绍     = 6（四世三公被动）
  装备扩展 = 待设计（+N）
```

---

### F4. 统帅约束

```
TroopCardCountInDeck ≤ CommanderLeadership（3~6）
```

> 军营节点选卡时验证；超出上限时该选项禁止加入。

---

### F5. DoT 每回合伤害

```
DoT_damage = StatusLayers × DamagePerLayer
```

> 中毒不经过护盾，直接扣 HP；灼烧经过护盾。详见状态效果系统（C1）。

---

### F6. 多阶段战斗胜利条件

```
PhaseCleared = ALL(enemy.HP ≤ 0) for enemies in CurrentPhase
BattleWon    = PhaseCleared AND CurrentPhase == TotalPhases
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **行动点不足以打出任何牌** | 玩家可主动结束回合；若手牌全部无法负担，自动提示"行动点不足，建议结束回合" |
| E2 | **抽牌时手牌已达上限** | 本回合不抽牌（DrawCount = 0），不触发弃牌 |
| E3 | **抽牌堆不足，弃牌堆为空** | 将抽牌堆剩余所有卡加入手牌（即使少于手牌上限），本次抽牌结束；不报错 |
| E4 | **多张卡同时结算目标死亡** | 当前卡结算完毕、目标 HP ≤ 0 后立即标记死亡；后续同回合对该目标的效果不再结算 |
| E5 | **阶段切换时玩家正在回合中** | 阶段切换在【阶段检查】步骤统一判定，不在出牌阶段中途打断 |
| E6 | **护盾累积超过最大 HP** | 护盾上限 = 角色当前最大 HP；超出部分丢弃，不允许超过最大 HP |
| E7 | **消耗卡被复制效果复制** | 复制体视为新获得的该卡，使用后按消耗规则处理（永久消耗） |
| E8 | **"使用后移除"卡的归还路径** | 默认在战斗全部结束后回卡组；特定卡牌效果可提前将移除区的卡回到卡组或手牌（待设计） |
| E9 | **手牌溢出上限（被动/效果强制抽牌）** | 超出上限的牌强制丢弃至弃牌堆，丢弃顺序从最新抽到的牌开始 |
| E10 | **敌人行动目标死亡（行动执行前目标已死）** | 若为混乱类攻击友方，重新随机选存活目标；若无存活目标，行动跳过 |
| E11 | **诅咒卡抽入手牌触发效果后被净化** | 先触发抽到效果，再执行净化（移出卡组）；两步顺序不可颠倒 |
| E12 | **多阶段：前阶段状态进入下阶段** | 玩家身上状态全部保留，继续按回合倒计时；新阶段敌人以无状态状态登场 |

## Dependencies

### 强依赖（C2 无法独立运行，必须先存在）

| 系统 | ID | 依赖原因 |
|------|----|---------|
| 资源管理系统 | F2 | HP / 护盾 / 行动点的数值定义与结算规则由 F2 提供 |
| 状态效果系统 | C1 | Buff/Debuff 的施加、倒计时、DoT 结算由 C1 提供；C2 仅触发钩子 |

### 弱依赖（运行时需要，但可以用占位实现先原型）

| 系统 | ID | 依赖原因 |
|------|----|---------|
| 地形天气系统 | D1 | 出牌结算时读取当前地形/天气修正系数 |
| 敌人系统 | C3 | 战斗初始化时读取敌人配置；敌人回合行动执行由 C3 提供 |

### 被依赖方（这些系统依赖 C2 才能工作）

| 系统 | ID | 依赖原因 |
|------|----|---------|
| 兵种卡系统 | D2 | 兵种卡在战斗中的打出、结算、统帅约束验证依赖 C2 出牌框架 |
| 武将系统 | D3 | 武将被动的触发时机（回合开始/结束/出牌时）由 C2 回合流程钩子触发 |
| 诅咒系统 | D4 | 诅咒卡的抽入触发、净化操作依赖 C2 卡牌生命周期管理 |
| 卡牌升级系统 | M5 | 升级后的卡牌在战斗中的效果差异由 C2 卡牌结算框架执行 |
| 商店系统 | M2 | 净化（移出卡组）操作依赖 C2 定义的卡牌区域模型 |
| 装备系统 | M3 | 装备被动效果（如手牌上限扩展）在 C2 初始化时注册并影响战斗参数 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 手牌上限（默认） | 5 | 4~6 | 每回合可用选择数量、策略深度 |
| 手牌上限（袁绍） | 6 | 固定 | 袁绍专属优势强度 |
| 行动点上限（默认） | 3~4（按武将） | 2~5 | 每回合出牌张数、节奏快慢 |
| 护盾上限（默认） | = 当前最大 HP | 50%~100% MaxHP | 防御流强度上限 |
| 护盾上限（曹仁铁壁） | 最多 +30（6层×5） | 按被动设计 | 护盾专精武将天花板 |
| 护盾上限（张角黄天当立） | 无上限 | 固定（被动设计） | 张角后期护盾累积无限制 |
| 多阶段战斗阶段数 | 1（前期）/ 2~3（精英/Boss） | 1~3 | 单场持久战压力 |
| DoT 护盾绕过（中毒/剧毒） | 直接扣 HP | 固定 | 中毒流稳定压力 |
| DoT 经过护盾（灼烧） | 经过护盾 | 固定 | 灼烧克制护盾流的力度 |
| 消耗卡永久性 | 永久消耗 | 固定 | 消耗卡稀缺感与风险感 |

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 战场支持 1 vs 1~3 敌将同时在场，各自独立 HP/护盾/状态 | 战斗场景测试：配置 3 名敌人，确认各自独立结算 |
| AC2 | 每回合开始补至手牌上限（默认5，袁绍6），不超出上限 | 单元测试：回合开始时 DrawCount = HandLimit - CurrentHandSize |
| AC3 | 行动点不足时无法打出灰显卡牌 | UI 测试：行动点 = 0 时所有卡灰显，无法点击 |
| AC4 | 攻击伤害优先扣护盾，溢出才扣 HP；"无视护甲"直接扣 HP | 单元测试：F1 公式覆盖有护盾/无护盾/无视护甲三种情况 |
| AC5 | 护盾上限默认 = MaxHP；曹仁铁壁层数可突破上限；张角无护盾上限 | 武将专项测试：各自护盾上限行为符合被动描述 |
| AC6 | 抽牌堆耗尽时弃牌堆自动洗回；两者均空时跳过抽牌不报错 | 单元测试：空牌库边界场景 |
| AC7 | "使用后移除"卡战斗结束回卡组；消耗卡永久消耗不回 | 存档测试：战斗前后卡组内容对比 |
| AC8 | 多阶段战斗：当前阶段全灭后自动进入下阶段，资源全保留 | 集成测试：2阶段战斗，确认 HP/护盾/手牌/状态跨阶段一致 |
| AC9 | 兵种卡数量超过统帅上限时军营节点禁止加入 | 军营节点测试：统帅=3 时第4张兵种卡不可选 |
| AC10 | 诅咒卡"抽到触发型"在抽入手牌时立即触发，之后按正常流程处理 | 战斗测试：抽入诅咒卡后确认触发时机正确 |
| AC11 | 敌人行动在敌人回合开始前公示，玩家可在本回合出牌阶段据此决策 | UI 测试：敌人意图图标在玩家回合可见 |
| AC12 | 胜利后护盾/行动点清零，HP 保留，移除区卡回卡组 | 战斗结算测试：胜利后状态检查 |
