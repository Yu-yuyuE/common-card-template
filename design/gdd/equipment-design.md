## Overview

装备系统定义《三国称雄》中武将在战役过程中**携带、获取、替换装备**的全部规则，以及装备效果的触发结构。装备分为5个部位（武器/防具/坐骑/兵符/奇物），每个武将最多携带**5件**（袁绍特殊：6件），**部位可重复**（可携带多件同部位装备）。

装备通过两个渠道获取：**商店节点购买**（M2 定义价格）和**战斗胜利后的装备奖励事件**（小概率触发，3选1）。装备效果为被动型，分为常驻被动和条件触发两类，整局游戏内持续有效。

**系统职责边界：**

- 负责：部位定义、携带规则（上限5/6）、替换逻辑、效果触发结构、装备数据（60件）
- 不负责：装备在商店中的价格（M2 定义）、商店货架生成（M2 定义）、装备奖励事件的触发概率（M1 地图节点定义）

## Player Fantasy

**”我的武将不只靠一把刀——他靠的是整套战争机器。”**

**① 身份强化感（构筑层）**
装备是武将身份的延伸。带两件兵符的曹操强化了统御感；带赤兔马+朱雀符的吕布是冲锋型杀手；带多件防具的刘备体现坚守待援的风格。装备组合让每位武将在同一局内都有独特的战术轮廓。

**② 叠加惊喜感（发现层）**
部位可重复意味着玩家可以探索极端构筑：三件兵符全押行动点流、双武器追求爆发伤害。这种自由度带来”这个组合我自己想出来的”成就感。

**③ 积累成长感（进度层）**
装备跨地图保留，战役越到后期装备越多。前期一件初级装备，中期三四件形成核心，后期五件满配进入Boss战——这条成长曲线让玩家在每次获得装备时都感到进步。

**不应有的感受：**
- 装备与当前武将/卡组完全无关（每件装备都应有潜在适用情境）
- 随便5件什么都行，装备选择没有意义
- 带满5件后再也不需要关注装备（满配后的替换决策也应有张力）

## Detailed Rules

### 1. 部位定义（5类）

| 部位 | 代表装备风格 |
|------|------------|
| **武器** | 伤害强化、攻击类卡牌增益 |
| **防具** | 护甲、减伤、生存类增益 |
| **坐骑** | 机动、地形适应、行动顺序相关 |
| **兵符** | 行动点、兵种卡增益、统帅相关 |
| **奇物** | 特殊效果、抽牌、战术类增益 |

---

### 2. 携带规则

| 规则项 | 说明 |
|--------|------|
| **携带上限** | 默认5件；袁绍特殊被动：上限6件 |
| **部位限制** | **部位可重复**——可携带多件同部位装备（如2件武器、3件兵符） |
| **持久性** | 装备跨小地图保留，整局游戏有效；不跨局保留 |

---

### 3. 获取渠道

#### 3a. 商店购买

- 进入商店节点时，装备区随机展示2件装备
- 用金币购买，立即装备（价格由 M2 定义）
- 详见 M2 商店系统

#### 3b. 战斗奖励事件

- 战斗胜利后，以一定概率触发”装备奖励事件”
- 触发时展示3件装备供选择（3选1），可跳过
- 触发概率见 Tuning Knobs
- 奖励装备免费获取，无需消耗金币

---

### 4. 替换规则

- 已携带5件（袁绍6件）时，获得新装备触发替换流程：
  1. 展示当前所有已装备的装备列表
  2. 玩家选择移除其中一件，换入新装备
  3. 被移除的装备**永久失去**（本局内不可找回）
- 在奖励事件中也可选择”跳过”，放弃该件装备而不触发替换

---

### 5. 效果触发结构

每件装备包含：

| 效果类型 | 说明 | 示例 |
|---------|------|------|
| **常驻被动**（必有） | 始终生效，无条件 | 玄甲：初始护甲+10 |
| **条件触发**（可选） | 满足特定条件时触发一次性或周期性效果 | 朱雀符：攻击命中时30%概率附加1层灼烧 |

**触发时机规则（统一裁定）：**

| 触发时机关键词 | 含义 |
|--------------|------|
| 回合开始 | 玩家回合开始时，在抽牌之前结算 |
| 回合结束 | 玩家结束回合时，在敌方行动之前结算 |
| 攻击命中时 | 攻击卡造成伤害的瞬间结算 |
| 打出[类型]卡时 | 该类型卡牌从手牌打出并支付费用后立即结算 |
| 首次受击 | 每场战斗中第一次受到伤害时触发，之后本战不再触发 |
| 低血时 | HP ≤ 30% 时持续生效（常驻判定，每回合检查） |

---

### 6. 装备效果叠加规则

- **同名装备**：效果**不叠加**——携带两件同名装备时，仅第一件生效，第二件占槽但无效果（UI 显示”已有同名装备，效果不叠加”）
- **不同装备的百分比加成**：多件装备的百分比加成**相加后统一计算**，而非连乘
  - 示例：两件各+15%伤害的不同装备 = 总伤害+30%，而非 ×1.15×1.15
- **不同装备的触发类效果**：各自独立判定，互不影响

---

### 7. 袁绍特殊规则

- 袁绍被动：装备上限+1（即6件）
- M3 装备系统提供携带上限接口，由 D3 武将系统中袁绍被动写入
- 除上限外，袁绍的装备获取/替换/效果规则与其他武将完全相同

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **装备槽已满（5件）时从商店购买新装备** | 触发替换流程：展示当前5件装备，玩家选择移除一件换入新装备；取消则不扣金币 |
| E2 | **装备槽已满时触发战斗奖励事件** | 同样触发替换流程；可跳过放弃该奖励装备 |
| E3 | **玩家携带两件同名装备** | 允许携带（占两个槽），但仅第一件生效；第二件在装备栏显示"效果不叠加"灰色提示 |
| E4 | **袁绍携带满6件后触发装备奖励** | 同样触发替换流程（上限6件），选择移除一件换入新装备或跳过 |
| E5 | **装备效果与武将被动冲突（如两者都修改统帅值）** | 加法叠加：统帅符+1、武将被动+1 → 总统帅+2；不覆盖，不取最大值 |
| E6 | **低血时触发装备的"低血"判定时机** | 每回合开始时检查 HP ≤ 30%；若本回合内受伤导致低血，效果在下一回合开始时生效 |
| E7 | **"首次受击"装备在一场战斗中被多个敌人同时攻击** | 按伤害结算顺序，第一次结算时触发，后续同回合不再触发 |
| E8 | **战斗奖励3选1时，3件装备全部与当前已有装备同名** | 允许选择（玩家可选择占槽但不叠加的同名装备，或跳过）；不强制重新生成 |
| E9 | **装备被移除后，该装备的触发效果是否需要清除** | 立即清除：移除装备后，其所有常驻被动和触发效果在当前回合内立即失效 |
| E10 | **祭酒令（装备区+1件）与袁绍（上限6件）同时生效** | 效果独立叠加：商店装备区展示3件；袁绍携带上限仍为6件；两个效果互不影响 |

## Formulas

### F1. 装备总加成计算

```
FinalStat = BaseStat × (1 + ΣPercentBonus) + ΣFlatBonus

变量：
  BaseStat      = 基础数值（HP、伤害、护甲等，由 F2/C2 定义）
  ΣPercentBonus = 所有生效装备的百分比加成之和（同名装备只计一次）
  ΣFlatBonus    = 所有生效装备的固定值加成之和

示例：
  BaseStat=10，携带”倚天剑+20%”+”方天戟+15%”（不同名）：
  FinalStat = 10 × (1 + 0.20 + 0.15) + 0 = 10 × 1.35 = 13.5 → 取整 = 14
```

### F2. 条件触发期望值

```
TriggerEV = TriggerChance × TriggerImpact

变量：
  TriggerChance  = 触发概率（0.0~1.0）
  TriggerImpact  = 单次触发效果量（伤害值、护甲值等）

示例：
  朱雀符（攻击命中时30%概率+1层灼烧，灼烧期望伤害=3）：
  TriggerEV = 0.30 × 3 = 0.9（每次攻击命中平均贡献0.9点灼烧期望伤害）

用途：比较”稳定被动”与”高波动触发”装备的平均价值
```

### F3. 装备奖励触发概率

```
RewardChance = BaseChance × DifficultyMult

变量：
  BaseChance     = 基础触发概率（见 Tuning Knobs）
  DifficultyMult = 难度系数
    普通战斗：1.0
    精英战斗：1.5
    Boss战斗：0（Boss战不触发装备奖励，另有固定奖励机制）

示例：
  基础概率10%，精英战斗：10% × 1.5 = 15% 触发装备奖励
```

## Dependencies

### 系统依赖

| 系统 | 方向 | 接口内容 |
|------|------|---------|
| 商店系统（M2） | 读 | M2 装备区从本系统装备池读取可售装备；购买后 M2 调用本系统的装备接口执行装备逻辑 |
| 地图节点系统（M1） | 读 | M1 战斗胜利后触发装备奖励事件；触发概率由 M1 配置，本系统提供奖励装备候选列表 |
| 卡牌战斗系统（C2） | 读/写 | 装备的触发时机（攻击命中、回合开始等）由 C2 战斗引擎发出事件；本系统监听并执行效果 |
| 资源管理系统（F2） | 读/写 | 装备效果修改 HP、护甲等资源时，通过 F2 接口执行；F2 管理装备修改后的资源持久化 |
| 武将系统（D3） | 读 | 袁绍被动写入本系统的携带上限接口（默认5，袁绍覆盖为6）；装备效果可引用武将属性 |
| 存档持久化系统（F1） | 写 | 当前携带装备列表由 F1 持久化，跨小地图保留 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| 装备携带上限（默认） | 5件 | 4~6件 | 装备构筑的深度；过少则选择空间不足，过多则装备过于强势 |
| 袁绍额外装备槽 | +1（共6件） | 固定+1 | 袁绍的差异化强度 |
| 战斗奖励基础触发概率 | 10% | 5%~20% | 装备获取频率；过高则金币竞争意义降低 |
| 精英战斗奖励概率系数 | 1.5× | 1.2×~2.0× | 精英战相对普通战的装备奖励激励强度 |
| 奖励事件展示数量 | 3选1 | 2~4选1 | 装备奖励的选择宽度 |
| 百分比加成叠加上限 | 无上限 | 建议单属性≤+100% | 防止极端叠加导致数值崩溃 |
| 触发类效果概率范围 | 20%~40% | 15%~50% | 触发装备的稳定性与惊喜感平衡 |
| 同名装备效果叠加 | 不叠加 | 固定不叠加 | 防止重复携带同名装备形成过强叠加 |

## Acceptance Criteria

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| AC1 | 携带上限默认5件，满5件后获得装备触发替换流程 | 功能测试：携带5件后购买第6件，确认出现替换界面而非直接装备 |
| AC2 | 袁绍携带上限为6件 | 功能测试：选择袁绍后携带5件，确认第6件直接装备而不触发替换；第7件触发替换 |
| AC3 | 部位可重复（可携带两件武器等） | 功能测试：携带2件武器，确认两件均在装备栏显示，无报错 |
| AC4 | 同名装备只有第一件生效，第二件显示"效果不叠加"提示 | UI测试：携带2件玄甲，初始护甲应为+10（非+20），第二件显示灰色提示 |
| AC5 | 不同装备的百分比加成相加计算 | 数值测试：携带+20%和+15%伤害的两件不同装备，BaseDamage=10，最终=13.5→14 |
| AC6 | 战斗奖励事件基础触发概率约10%，精英战约15% | 统计测试：记录500场普通战斗的奖励触发次数，应在40~60次之间（8%~12%） |
| AC7 | 装备被移除后效果立即失效 | 功能测试：移除"虎符（每回合+1行动点）"后，下一回合行动点恢复正常值 |
| AC8 | "低血时"效果在回合开始检查，当回合受伤导致低血下一回合才生效 | 功能测试：HP从35%受伤降至25%，当回合低血装备不生效，下一回合开始时生效 |
