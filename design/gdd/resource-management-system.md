# 资源管理系统（Resource Management System）

> **状态**：设计中
> **作者**：设计团队
> **最后更新**：2026-04-06
> **实现支柱**：战斗循环基础层

## 概述（Overview）

资源管理系统定义并追踪战斗与地图层中所有数值化资源的状态。它管理四类原始资源：**HP**（决定武将存亡）、**护盾**（消耗型伤害缓冲）、**行动点**（战斗中出牌的费用货币）、**粮草**（地图移动消耗）。玩家通过打出卡牌、敌人行动、地图事件等途径消耗或恢复这些资源；这些资源的当前值直接决定玩家的可用决策空间与生存状态。资源管理系统本身不包含游戏逻辑——它是状态容器和接口，供所有其他系统读写。

## 玩家幻想（Player Fantasy）

玩家感受到的不是"资源管理"本身，而是"紧迫感"与"掌控感"：

- **HP 压力**：血量下降时，玩家感受到死亡威胁，驱动防御或爆发决策
- **行动点张力**：每回合行动点有限，强迫玩家在多张可打卡之间做取舍——"这张不够费，下回合打"的痛苦抉择
- **护盾缓冲**：护盾归零前的一回合是"安全窗口"，玩家在此窗口内敢于激进出牌
- **粮草焦虑**：粮草见底时，玩家必须权衡"是否绕路走旅馆/商店"，形成地图层决策张力

资源数值本身不应让玩家"盯着数字看"，而是让玩家通过数字变化直觉感知局势危急程度。

## 详细规则（Detailed Rules）

### 核心资源定义

本系统管理四类原始资源，每类资源有各自的上限、下限和结算规则：

#### 1. HP（生命值）

| 字段     | 值                                                                   |
| -------- | -------------------------------------------------------------------- |
| 范围     | 0 – 武将 HP 上限（武将个体差异：40–60）                              |
| 归零     | 战斗失败，本局结束                                                   |
| 跨战斗   | **持久化**——战斗结束后 HP 保留当前值进入下一节点，每次击败BOSS加满HP |
| 恢复来源 | 旅馆节点、卡牌效果、部分奇遇事件                                     |

#### 2. 护盾（Armor / Shield）

| 字段     | 值                                                            |
| -------- | ------------------------------------------------------------- |
| 范围     | 0 – 武将 HP 上限（消耗型）                                    |
| 工作机制 | 持续到被打破；受到伤害时先扣护盾，护盾归零后超出部分溢出扣 HP |
| 跨回合   | **持续**，不自动重置（护盾残留可跨回合保留）                  |
| 跨战斗   | 战斗结束后护盾值**清零**，不带入下一战                        |
| 恢复来源 | 防御类卡牌、盾兵兵种卡、部分武将被动                          |

#### 3. 行动点（费用 / Action Points）

| 字段       | 值                                                                                 |
| ---------- | ---------------------------------------------------------------------------------- |
| 初始上限   | 武将个体差异（3–4），由武将属性定义                                                |
| 战斗开始   | 加满至上限                                                                         |
| 每回合开始 | 不自动重置——**可累积**到下回合（累积上限 = 武将上限，防止无限囤积）                |
| 恢复来源   | 特定卡牌效果、武将被动（不自动回满）                                               |
| X 费卡     | X = 打出时剩余全部行动点，打出后行动点归零                                         |
| 上限成长   | 卡牌效果增加上限，**不累计**到下一场战斗；装备效果增加上限，**可累计**到下一场战斗 |

#### 4. 粮草（Food）

| 字段           | 值                                                                        |
| -------------- | ------------------------------------------------------------------------- |
| 范围           | 0–150                                                                     |
| 每张大地图开局 | 加满至 150                                                                |
| 移动消耗       | 移动到下一节点消耗 2–8 粮草（每一幕早期2-6，后期4-8，绘制地图时随机赋值） |
| 恢复来源       | 旅馆节点、商店购买、奇遇事件奖励、击败 Boss 后奖励50                      |
| 归零后果       | 扣减等量 HP 才能至下一节点，HP 扣减完战斗失败，本局结束                   |

---

### 资源结算时序

**战斗内每回合结算顺序（严格）：**

1. **回合开始**：行动点累积（保留上回合剩余行动点）
2. **玩家出牌阶段**：玩家打出卡牌消耗行动点；武将被动/特殊卡可在此阶段恢复行动点
3. **回合结束**：状态层数 -1（按状态触发时序结算）；持续伤害结算（中毒/灼烧/重伤等——不经过护盾）；持续回复结算
4. **敌人回合**：敌人行动执行；造成的伤害先扣护盾，超出部分溢出扣 HP
5. **HP 检查**：若 HP ≤ 0，战斗失败

**战斗结束结算（无论胜负）：**

| 资源   | 战斗结束行为                   |
| ------ | ------------------------------ |
| HP     | 保留当前值（败则视为游戏结束） |
| 护盾   | **清零**                       |
| 行动点 | **清零**                       |
| 粮草   | 不参与战斗，战斗结束无变化     |

---

### 跨战斗持久化规则

| 资源   | 战斗内           | 跨战斗（节点间） | 跨地图（大地图切换）     |
| ------ | ---------------- | ---------------- | ------------------------ |
| HP     | 动态变化         | **持久化**       | **持久化**（不自动恢复） |
| 护盾   | 动态变化         | **清零**         | **清零**                 |
| 行动点 | 动态变化，可累积 | **清零**         | **清零**                 |
| 粮草   | 不参与战斗       | 移动时消耗       | **加满至 150**           |

## 公式（Formulas）

### F1. 伤害结算

```
DamageToHP = max(0, IncomingDamage - CurrentArmor)
NewArmor   = max(0, CurrentArmor - IncomingDamage)
NewHP      = CurrentHP - DamageToHP
```

### F2. 行动点累积

```
MaxAP       = HeroBaseAP（武将属性定义，3–4）+ 装备增量（跨战斗累积）
APNextRound = min(MaxAP, PreviousRemainingAP)
```

注：每回合不自动补充行动点；卡牌效果临时提升的上限仅本战有效，不累积到下一场战斗。

### F3. X 费卡

```
X           = CurrentAP（打出时当前全部行动点）
RemainingAP = 0
```

### F4. 粮草移动消耗

```
FoodAfterMove = CurrentFood - NodeMoveCost
NodeMoveCost  ∈ [2, 8]（早期节点 2–6，后期节点 4–8，绘制地图时随机赋值）
```

### F5. 粮草归零惩罚

```
HPCost = NodeMoveCost - CurrentFood   （粮草不足的差额）
NewHP  = CurrentHP - HPCost
```

若 `NewHP ≤ 0`，触发游戏结束。

### F6. 持续伤害（DoT，常规经护盾，除非特殊说明）

```
DoTDamage = StatusLayers × DamagePerLayer
DamageToHP = max(0, DoTDamage - CurrentArmor)
NewArmor   = max(0, CurrentArmor - DoTDamage)
NewHP      = CurrentHP - DamageToHP
```

参见 `status-design.md` 各状态的 `DamagePerLayer` 定义值。

## 边界情况（Edge Cases）

1. **HP 归零时护盾仍有值**：HP 归零判断优先，护盾剩余无意义，游戏结束
2. **行动点为 0 时打出 X 费卡**：X = 0，卡牌效果以 X = 0 结算（不阻止打出）
3. **护盾值超出武将 HP 上限**：护盾无硬上限（只要有卡牌来源），允许超过武将 HP 上限
4. **粮草移动后恰好归零（= 0）**：可正常移动，不触发 HP 扣减惩罚
5. **粮草归零且 HP 不足以支付移动代价**：HP 扣至 0，触发游戏结束（不允许负 HP 移动）
6. **Boss 奖励粮草超出上限 150**：超出部分丢弃，粮草最高保持 150
7. **行动点累积恰好到上限**：保留 = MaxAP，不溢出
8. **同一帧多个 DoT 状态同时触发**：按状态编号顺序逐一结算，不并行（确保日志可读）

| 旋钮                 | 安全区间             | 影响                         |
| -------------------- | -------------------- | ---------------------------- |
| 武将 HP 上限         | 40–60                | 战斗耐久度，影响压力节奏     |
| 武将基础行动点       | 3–4                  | 每回合出牌数量，影响节奏快慢 |
| 粮草上限             | 100–200              | 地图焦虑强度                 |
| 节点粮草消耗（早期） | 2–6                  | 前期压力                     |
| 节点粮草消耗（后期） | 4–8                  | 后期压力                     |
| Boss 粮草奖励        | 30–70                | 阶段性续航补偿               |
| 粮草归零 HP 惩罚     | 1:1（差额直接扣 HP） | 粮草断供的死亡压力           |

## 可视化/音频需求（Visual/Audio Requirements）

- HP 条需有明确的低血量视觉警示（如红色/闪烁），阈值建议 < 30%
- 护盾与 HP 在 UI 上需有明确区分（颜色/层叠方式）
- 行动点宜用离散图标显示（而非进度条），方便玩家感知剩余量
- 粮草建议在地图界面显示专用图标 + 数字，战斗界面可隐藏
- 伤害结算时建议播放护盾破碎音效与护盾剩余音效两种不同反馈

## 验收标准（Acceptance Criteria）

1. HP 归零时立即触发战斗失败，不允许负值 HP 继续战斗
2. 护盾正确先于 HP 吸收伤害；超出部分溢出扣 HP
3. 护盾跨回合保留；战斗结束后清零
4. 行动点在本回合剩余量保留至下回合；累积值不超过武将上限
5. X 费卡以打出时实际行动点为 X，打出后行动点归零
6. 粮草移动消耗在地图绘制时确定并固定，与战斗结果无关
7. 粮草归零时，HP 替代粮草支付移动代价；HP 耗尽则游戏结束
8. Boss 战胜后恢复 50 粮草，上限 150
9. 大地图切换时粮草重置为 150
10. 持续伤害（DoT）除特殊说明外，也是先用护盾抵扣
11. HP 恢复来源（旅馆/卡牌/事件）不超过武将 HP 上限
