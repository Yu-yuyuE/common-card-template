## Overview

本文件定义《三国称雄》中所有敌人单位的设计规范：

- 敌人职业分5种：**步兵 / 骑兵 / 弓兵 / 谋士 / 盾兵**（决定小模型外观与AI行为倾向）
- 敌人分三级：**普通敌人 / 精英敌人 / 强力敌人**
- 敌人无卡组，每回合直接执行一种行动
- 行动在敌人回合开始前**公开显示**，玩家可提前应对
- 行动按规律轮换，每回合只执行一种行动
- 普通敌人只使用普通行动，精英敌人使用普通+精英行动，强力敌人可使用全部三级行动
- 普通敌人：行动种类≤2种，循环3步；精英敌人：行动种类2~3种，循环3~4步；强力敌人：行动种类3种，循环4~5步
- 行动循环中动作可重复，示例：普通敌人A：A01→A01→A03，精英敌人A：A01→A04→B04

## Player Fantasy

- **预判应对**：看到敌人下一步行动，提前规划出牌顺序
- **优先级博弈**：多名敌人同时公开意图，决定先打谁
- **历史沉浸感**：遭遇有历史背景的小人物，感受战场多样性

## Detailed Rules

### 1) 敌人分级规则

| 级别     | 可用行动                    | 典型出现场景        | 血量区间 | 护甲区间 |
| -------- | --------------------------- | ------------------- | -------- | -------- |
| 普通敌人 | 普通行动（18种）            | 普通战斗节点、1-8层 | 20~45    | 0~8      |
| 精英敌人 | 普通行动 + 精英行动（18种） | 精英节点、8-11层    | 35~80    | 5~15     |
| 强力敌人 | 普通+精英+强力行动（18种）  | Boss节点、关键战役  | 60~200   | 10~25    |

### 2) 行动顺序公开机制

- 每名敌人在**自己回合开始时**（玩家回合结束后）公示其本回合行动意图
- 公示内容：行动名称 + 行动效果简述 + 目标（若固定）
- 玩家可据此调整策略：优先清除威胁、提前堆护甲、预备反击
- 行动执行顺序：敌人按位置从左到右执行

**蓄力型行动（特殊公示）**：部分强力敌人的行动序列中含有「蓄力回合」——该回合本身不造成伤害，但公示图标/文本需明确标注"正在蓄力，下回合将释放 [行动名]"。蓄力型行动的核心设计价值在于：玩家获得一回合的确定性准备窗口，应对可预期但高威胁的爆发行动（如大额伤害、全场 Debuff）。蓄力后的行动威力应明显高于常规行动，否则蓄力提示失去战略意义。

### 3) 行动轮换规律

- 每名敌人有固定的**行动序列**（2~4种行动循环）
- 序列按顺序执行，循环播放（不随机，保证可预判性）
- 特殊行动（如召唤、复制）有**冷却回合**限制，触发后至少间隔2回合

### 4) 职业字段规范

每名敌人拥有一个职业标签，决定小模型外观、AI行为倾向和地形联动基准：

| 职业 | 外观特征    | AI倾向                      | 典型地形加成               |
| ---- | ----------- | --------------------------- | -------------------------- |
| 步兵 | 持刀/矛步行 | 近战输出为主，循环攻击/防御 | 关隘：防御+；平原：无加成  |
| 骑兵 | 骑马持矛    | 高速冲锋，攻击为主          | 平原：攻击+；山地：攻击-   |
| 弓兵 | 持弓/弩远程 | 远程输出，偶尔蓄力          | 关隘：射程优势；水域：减弱 |
| 谋士 | 持羽扇/书卷 | Debuff/诅咒/增援为主        | 无地形限制                 |
| 盾兵 | 持大盾重甲  | 防御/护甲/坚守为主          | 关隘：护甲++；平原：无加成 |

> 职业仅为外观与倾向标签，具体行动序列由行动编号决定，不受职业约束（设计自由度保留）。

### 5) 行动等级与状态强度约束

行动库中的施加状态行动受**状态强度等级**限制：

| 行动等级        | 可施加负面状态强度上限                | 说明                                             |
| --------------- | ------------------------------------- | ------------------------------------------------ |
| 普通行动（A类） | 仅限普通强度（D12重伤）或中等强度状态 | 不得施加强力负面状态（滑倒/眩晕/混乱/剧毒/瘟疫） |
| 精英行动（B类） | 中等 + 强力强度均可                   | 强力状态如眩晕/混乱/滑倒可在B类行动中出现        |
| 强力行动（C类） | 无限制                                | 可叠加、复合施加各类状态                         |

> **强力负面状态（不可出现于A类行动）**：滑倒（D6）、眩晕（D11）、混乱（D4）、剧毒（D2）、瘟疫（D10）

### 6) 单次行动只施加一种状态

- 每个行动每次只对目标施加**一种**状态（同类可叠加层数）
- 不同类型状态不可共存，后加的状态顶替当前状态
- 因此，**所有行动描述中不允许出现同时施加两种不同状态的写法**
- 攻击行动附带状态（如A07踢踹=伤害+盲目）属于"攻击动作结算后施加状态"，是合法的复合行动，不违反此规则（伤害不是状态）

### 7) 诅咒投递位置规范

敌人投放诅咒卡的目标位置决定了对玩家的压力类型，设计时需有意区分：

| 投递位置               | 感知时机                     | 压力特征                       | 推荐使用场景                   |
| ---------------------- | ---------------------------- | ------------------------------ | ------------------------------ |
| **加入手牌**           | 立即感知，当回合可能被迫使用 | 即时干扰、占用手牌资源         | 常驻手牌型诅咒、高优先级干扰   |
| **加入抽牌堆顶**       | 下次抽牌时触发               | 延迟定时炸弹，玩家已知何时遇到 | 中等威胁、可被管理             |
| **加入抽牌堆随机位置** | 触发时机不定                 | 焦虑感最强，难以规划应对       | 抽到触发型诅咒、高压节奏       |
| **加入弃牌堆**         | 循环时必定遭遇               | 持续污染、常驻牌库型效果最强   | 常驻牌库型诅咒（不丢弃不消除） |

- 同一场战斗中，诅咒敌人（如E029、E032、E040）应尽量使用不同投递位置，避免单一压力模式
- B12（阴险诅咒）默认投入抽牌堆；C11（诅咒风暴）同时投手牌+抽牌堆，体现等级差异

**地形联动格式规范：**

- 正面：`平原:攻击+2`、`关隘:护甲+5`、`水域:行动序列切换为远程优先`
- 负面：`山地:攻击-2`、`雨天:灼烧效果×0`
- 多条联动用 `|` 分隔：`平原:攻击+2|山地:攻击-2`

### 设计原则

1. **小人物历史风味**：角色源自历史上的基层人物、地方武装、特殊职业
2. **地域局域性**：每个敌人关联特定州/地形，在对应地图出现频率更高
3. **职业多样化**：涵盖武将、文士、商人、道士、江湖人等
4. **行动序列**：每个敌人有固定2~4步行动循环（见行动编号）
5. **身份即能力**：敌人的机制必须与其职业身份直接呼应，无需额外说明。粮草官应能为友军回血、斥候应有高速、毒士类应施 Debuff——玩家一眼看到名字就能预判行为风格。同名行动在不同敌人身上可以有数值差异，但机制方向应与身份一致。
6. **组合协同压力**：1v3 场面下，三名敌人的行动序列应产生战术交叉压力，而非三个独立单体的叠加。设计时考虑：护甲敌人 + 破甲输出的反向配合、诅咒文士 + 硬攻武将迫使玩家分心、召唤型 + 增益型的持续威胁链。单独设计每个敌人后，需核验该敌人与同地图常见搭档的组合效果是否有趣。
7. **相变机制（强力敌人优先）**：强力敌人（E041~E055）应至少有一个 HP 阈值触发的行为模式转变，而非单纯叠加数值。相变前应有「预告行动」（如先清空玩家护甲、先施加 Debuff），让玩家能读出"下回合要变"。相变后的行动序列与相变前应有明显节奏差异（如从防守型转为爆发型）。
8. **阵营隐性主题**：同一历史阵营的敌人应共享某种隐性机制倾向，形成区域风格差异：
   - **黄巾阵营**：数量压制、召唤援军、狂热蓄力
   - **西凉阵营**：高伤冲锋、破甲、平原骑兵加成
   - **江东阵营**：水域联动、毒攻、远程协同
   - **北方胡族**：狂暴增伤、天气恶化、召唤野兽/同族
   - **谋士系（各阵营）**：诅咒投放、Debuff叠加、干扰卡组

## Formulas

### F1. 敌人行动伤害（物理）

`FinalDamage = BaseDamage × RageMod × WeaknessDebuffMod × TerrainMod`

- RageMod：每层怒气 +50% 伤害倍率
- WeaknessDebuffMod：玩家有虚弱时 ×0.5（此为对敌公式参考，反向可用于敌人虚弱后的伤害折减）

### F2. 行动序列推进

`CurrentAction = ActionSequence[TurnCount % SequenceLength]`

- TurnCount：该敌人存活的回合数（从1开始）
- SequenceLength：行动序列长度（2~4）

### F3. 召唤上限

`MaxSummonCount = 3 - CurrentEnemyCount`

- 战场最多同时存在3名敌将（含召唤物）
- 若已满3人，召唤类行动自动跳过，替换为序列中下一个非召唤行动

### F4. 条件触发行动

`TriggerCondition = CurrentHP / MaxHP < TriggerThreshold`

- C12（绝地反击）：TriggerThreshold = 0.30
- C18（末路求生）：TriggerThreshold = 0.20
- 每场战斗仅触发1次

## Edge Cases

1. **召唤行动满场**：战场已有3名敌将时，召唤行动跳过，执行序列中下一个非召唤行动
2. **条件触发行动未满足**：HP未低于阈值时执行条件行动节点，替换为序列中上一个普通行动
3. **偷取空手牌（B15）**：玩家手牌为空时，B15 跳过，执行序列下一步；**偷取空抽牌堆（C15）**：抽牌堆少于2张时，偷取实际剩余张数
4. **镜像分身（C08）满场**：战场已满时自动跳过，等待下一个循环
5. **眩晕敌人**：敌人陷入眩晕（D11）时跳过本回合行动，但行动序列计数器照常推进
6. **复仇将领特殊判断**：E044血量低于50%时行动序列额外插入1次C01（不推进常规计数器）
7. **天气改变行动重复触发**：C05/C06 冷却回合=2，不可连续触发
8. **金币偷取超出玩家持有量**：偷取数量上限为玩家当前持有金币，不可超出
9. **C16破甲叠加与状态顶替**：C16三次攻击各自施加1层破甲，每次施加会刷新已有破甲（取较高层数），不会叠加累积；最终效果为玩家保有1层破甲

| ------------------ | -------- | ------------------ |

| 精英敌人HP         | 35~80    | 威胁感与成就感     |

| 召唤行动冷却       | 2~3回合  | 群战频率控制       |
| 条件触发阈值       | 20%~35%  | 绝地反扑惊险度     |
| 金币偷取量（普通） | 3~8      | 地图资源压力       |

## Acceptance Criteria

1. 所有55名敌人有完整属性：HP/护甲/行动序列/速度
2. 行动序列在每回合开始时正确公示给玩家（名称+效果简述+目标）
3. 普通敌人不得使用B类或C类行动
4. 精英敌人不得使用C类行动
5. A类行动不得施加强力负面状态（滑倒/眩晕/混乱/剧毒/瘟疫）
6. 所有行动每次只施加一种状态，不存在同时施加两种不同状态的行动
7. 召唤行动满场（3名敌将）时自动跳过
8. 条件触发行动（C12/C18）每场战斗仅触发一次
9. 偷取类行动在玩家资源为空时正确跳过
10. 行动序列轮换计数器在敌人眩晕时照常推进
11. 地域标签影响对应地图的敌人出现权重
12. 所有数据与 enemies.csv / enemy_actions.csv 保持一致
