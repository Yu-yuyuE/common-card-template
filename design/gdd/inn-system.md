## Overview

酒馆系统定义《三国称雄》中**酒馆节点**的全部服务行为与规则约束。酒馆以三国时代的驿站军营酒肆为背景，是战役地图中唯一的**免费恢复节点**，为玩家提供HP回复与粮草补给服务。

> **注**：诅咒卡的移除（净化）通过**军营节点**的"移出卡牌"功能实现，酒馆不提供净化服务。

**系统职责：**

1. **歇息恢复** — 进入节点自动触发，免费恢复固定 HP，无需玩家操作
2. **购买粮草** — 消耗金币补充行动力（粮草），可多次购买
3. **强化休整** — 消耗金币获得额外 HP 恢复，每次访问限 1 次

**与相邻系统的差异化定位：**

| 节点 | 核心价值 | 货币 | 净化 | 免费收益 |
|------|---------|------|------|---------|
| **酒馆** | 喘息·续航·HP恢复 | 金币 | ✗（军营负责） | ✓（歇息HP） |
| **商店** | 构筑·强化·装备 | 金币 | ✗ | ✗ |
| **军营** | 编排·兵种·卡组管理 | 粮草 | ✓（移出卡牌） | ✗ |

**系统边界（不负责）：**

- 不负责诅咒卡的净化（由军营系统"移出卡牌"功能提供）
- 不负责金币的来源与战斗奖励数值（由战斗结算系统定义）
- 不负责粮草的全局管理与跨节点持久化（由资源管理系统定义）
- 不负责武将最大 HP 的计算（由武将系统定义）
- 不负责地图节点的生成与权重分配（由地图节点系统定义）

## Player Fantasy

**"终于可以歇一口气了。"**

### ① 安全感（生存层）

战场上连续征战，HP 磨损、诅咒累积，酒馆是路途中唯一的避风港。进门就自动恢复体力——不是奖励，是应得的喘息。玩家应感受到从紧绷到松弛的节奏切换，短暂的安全感让后续战斗的压力更有意义。

### ② 资源续航（策略层）

粮草是行军的命脉。酒馆提供可多次购买的粮草补给，让玩家在路线决策上有更多余地。花多少金币换多少粮草，是一次小而具体的资源权衡——不沉重，但值得思考。

### ③ 补给续航（策略层）

粮草是行军的命脉。酒馆提供可多次购买的粮草补给，让玩家在路线决策上有更多余地。花多少金币换多少粮草，是一次小而具体的资源权衡——不沉重，但值得思考。

**不应有的感受：**
- HP 已满时歇息感觉"浪费了"（系统应明确提示，不产生损失感）
- 粮草已满时购买按钮仍可点击（应置灰并提示上限）
- 金币不足以做任何事时，酒馆感觉是无意义的停顿（歇息永远免费，保证最低收益）

## Detailed Rules

### 1. 进入流程

```
玩家移动至酒馆节点
  → 扣除移动粮草消耗（由地图节点系统执行，与其他节点相同）
  → 自动触发【歇息恢复】：立即恢复固定 HP（无需玩家确认）
  → 显示酒馆服务菜单，玩家自由选择消费项目
  → 玩家点击"离开酒馆" → 本次访问结束
```

### 2. 服务菜单

#### 服务 A：歇息恢复（自动，免费）

| 项目 | 说明 |
|------|------|
| **触发时机** | 进入节点时自动执行，不占用玩家操作 |
| **费用** | 免费 |
| **恢复量** | `InnHealAmount`（见 Formulas F1） |
| **上限** | 恢复后不超过武将当前最大 HP |
| **次数** | 每次访问 1 次（重访不重置） |

#### 服务 B：购买粮草（主动，消耗金币）

| 项目 | 说明 |
|------|------|
| **触发时机** | 玩家主动点击"购买粮草"按钮 |
| **费用** | `CargoPrice`（见 Formulas F2） |
| **购入量** | `CargoPurchaseAmount`（见 Formulas F2） |
| **上限** | 当前粮草 + 购入量 不超过粮草上限（`MaxCargo = 150`） |
| **次数** | 单次访问可多次购买，直至粮草上限或金币耗尽 |
| **条件** | 金币 ≥ `CargoPrice` 且当前粮草 < `MaxCargo` |

#### 服务 C：强化休整（主动，消耗金币）

| 项目 | 说明 |
|------|------|
| **触发时机** | 玩家主动点击"强化休整"按钮 |
| **费用** | `FortifyPrice`（见 Formulas F3） |
| **效果** | 额外恢复 HP（`FortifyHealAmount`，见 Formulas F3），超过最大 HP 的部分无效 |
| **次数** | 每次访问限 1 次 |
| **条件** | 金币 ≥ `FortifyPrice` 且当前 HP < 最大 HP |
| **可见性** | HP 已满时置灰，注明"当前体力已满" |

### 3. 重访规则

- 酒馆节点标记为**不可重复生效**：再次到达同一酒馆节点时，**歇息恢复不再触发**，**强化次数不重置**
- 购买粮草按钮仍可使用（粮草补给是资源交易，不受重访限制）
- 节点 UI 显示"已休整"标签，明确提示玩家当前状态
- 重访时服务菜单仍正常展示，但受限服务项保持已消耗状态

## Formulas

### F1. 歇息恢复量

```
InnHealAmount = BaseHeal

变量：
  BaseHeal = 15（HP，固定值）

约束：
  实际恢复后 HP ≤ HeroMaxHP
  若 CurrentHP + BaseHeal > HeroMaxHP，则实际恢复 = HeroMaxHP - CurrentHP

示例：
  武将最大 HP = 80，当前 HP = 70 → 恢复 10（到上限 80，非 15）
  武将最大 HP = 80，当前 HP = 50 → 恢复 15（到 65）
```

### F2. 粮草购买

```
CargoPurchaseAmount = BaseCargoBuy
CargoPrice          = BaseCargoPrice

变量：
  BaseCargoBuy   = 50（粮草）
  BaseCargoPrice = 40（金币）

约束：
  购买后粮草 ≤ MaxCargo（150）
  若 CurrentCargo + BaseCargoBuy > MaxCargo，实际购入 = MaxCargo - CurrentCargo
  （金币仍按全额扣除；UI 在结算前提示实际购入量）

示例：
  当前粮草 120，MaxCargo 150 → 可购入 30（非 50），花费 40 金
  当前粮草 150（已满）→ 购买按钮置灰，不可操作
```

### F3. 强化休整

```
FortifyHealAmount = BaseFortifyHeal
FortifyPrice      = BaseFortifyPrice

变量：
  BaseFortifyHeal  = 20（HP）
  BaseFortifyPrice = 60（金币）

约束：
  实际恢复后 HP ≤ HeroMaxHP（同 F1 约束）

设计意图：
  歇息（免费）+ 强化（60金）合计最多恢复 35 HP。
  对比一次普通战斗奖励约 30 金，强化休整需要约 2 场战斗的金币收入，属于中等消耗决策。
```

## Edge Cases

| # | 边界情况 | 裁定规则 |
|---|---------|---------|
| E1 | **HP 已满时进入酒馆** | 歇息恢复仍自动触发，实际恢复量为 0；UI 显示"体力充沛，已原地休整"，不产生"浪费"的负面提示；强化休整按钮置灰 |
| E2 | **粮草已达上限（150）时点击购买粮草** | 购买粮草按钮置灰，显示"粮草充足（上限）"；不允许超限购买 |
| E3 | **粮草接近上限，购买后将超限** | 允许购买，实际购入量 = MaxCargo - CurrentCargo（不足一整包）；UI 在确认前明确显示"实际购入 X 粮草（非 50），花费 40 金"；玩家确认后方扣金币 |
| E4 | **金币不足以支付强化费用（< 60金）** | 强化休整按钮置灰，显示所需金币与当前持有金币；不允许执行 |
| E5 | **强化休整后再次访问同一酒馆** | 强化按钮保持已使用状态（不重置）；购买粮草功能仍正常可用 |
| E6 | **同时满足 HP 已满且粮草已满且金币不足** | 酒馆显示为"所有付费服务项均不可用"状态，提示"当前状态良好，无需额外补给"；歇息恢复仍静默触发（恢复量为 0）；玩家可直接离开 |
| E7 | **歇息恢复触发时武将 HP 为 0（理论上不应到达酒馆）** | 防御性处理：HP = 0 视为当前 HP，恢复后 HP = BaseHeal（15）；记录警告日志；游戏继续而非崩溃 |
| E8 | **强化休整后 HP 超出最大 HP（公式约束已处理但需验证）** | 实际设置为 HeroMaxHP，不超上限；不产生"过量 HP"的隐藏加值 |
| E9 | **存档读取时酒馆已访问标记丢失** | 默认视为"未访问"（对玩家有利）；最多让玩家多获得一次免费 HP，属于可接受的容错 |

## Dependencies

### 系统依赖（双向）

| 系统 | 方向 | 接口内容 |
|------|------|---------|
| **地图节点系统（M1）** | 读（被触发） | M1 检测玩家进入酒馆节点后触发酒馆系统；M1 提供节点已访问标记（重访检查）；本系统向 M1 注册"酒馆节点已访问"状态 |
| **资源管理系统（F2）** | 读/写 | 粮草当前值与上限由 F2 管理；购买粮草后通过 F2 增加粮草；金币余额由 F2 管理；强化/购买粮草均通过 F2 扣除金币 |
| **武将系统（D3）** | 读 | 读取武将当前 HP、最大 HP；歇息与强化休整通过武将系统的接口更新 HP |
| **存档持久化系统（F1）** | 读/写 | 酒馆节点已访问标记、强化已使用标记均需持久化；跨会话恢复时读取上述状态 |
| **卡牌战斗系统（C2）** | 间接 | 本系统不直接调用 C2；卡组变更（如在军营净化诅咒后）由 C2 在下一场战斗时读取 |

## Tuning Knobs

| 调节项 | 当前值 | 安全区间 | 影响维度 |
|--------|--------|---------|---------|
| `BaseHeal`（歇息恢复量） | 15 HP | 10~25 HP | 酒馆的基础安全感；过低则喘息感不足，过高则削减战斗 HP 压迫 |
| `BaseFortifyHeal`（强化额外恢复量） | 20 HP | 15~30 HP | 强化的吸引力；与歇息叠加不应超过总 HP 的约 50% |
| `BaseFortifyPrice`（强化费用） | 60 金 | 40~80 金 | 强化的使用频率；低于 40 会变成必买项，高于 80 则无人问津 |
| `BaseCargoBuy`（单次粮草购入量） | 50 粮草 | 40~60 粮草 | 每次购买对行程的续航贡献；建议与 Boss 战后粮草奖励（50）保持一致 |
| `BaseCargoPrice`（粮草购买费用） | 40 金 | 30~60 金 | 粮草的金币价值；过低则金币失去取舍张力，过高则粮草购买形同虚设 |
| `MaxCargo`（粮草上限） | 150 粮草 | 由地图系统定义，此处只读 | — |

## Acceptance Criteria

| # | 验收标准 | 测试类型 |
|---|---------|---------|
| AC1 | 玩家进入酒馆节点后，自动触发歇息恢复，HP 增加 `BaseHeal`（15），且不超过最大 HP | 自动化（逻辑） |
| AC2 | HP 已满时进入酒馆，歇息恢复不报错，恢复量为 0，UI 显示"体力充沛"提示 | 自动化（逻辑） |

| AC5 | 强化休整执行后，HP 增加 `BaseFortifyHeal`（20），金币减少 `BaseFortifyPrice`（60），且 HP 不超过最大 HP；本次访问内强化按钮变为不可用 | 自动化（逻辑） |
| AC6 | HP 已满时强化休整按钮置灰，显示"当前体力已满" | 自动化（逻辑） |
