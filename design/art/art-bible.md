> **Art Director Sign-Off (AD-ART-BIBLE)**: APPROVED [2026-04-09]

## 1. Visual Identity Statement

**核心视觉准则 (Core Visual Rule)**
"笔墨运筹，战术明晰。" (Ink-brushed strategy, crystal-clear tactics.)
每一个视觉元素都应如绘于丝绢之上——而后以"一眼可辨"为标准，毫不妥协地加以审视。

**视觉设计原则 (Supporting Principles)**

1. **战略与战术——战场先于呼吸，先于阅读**
视觉层级以战斗信息为先，以氛围营造为辅。卡牌属性、地形修正值与兵力容量，须在无需任何前置认知的情况下，于一秒之内完成识别。
*设计验证*：当须在纹理丰富的地形格与简洁版本之间作出取舍时，选择能让初次接触的玩家在一秒内辨明地形类型及其属性加成的方案。

2. **历史主题——以克制彰显权威**
三国美学通过图形语言、字体设计与色彩体系加以传达，而非诉诸视觉噪音。书法笔触纹样、陈旧丝绢质感的暖调中性色，以及武将肖像的线稿风格，共同构建历史语境。装饰性元素须以将领军衔与卡牌稀有度为依据，方可获得使用资格——绝非填充画面的点缀。
*设计验证*：当某一界面元素显得"三国感不足"时，正确的应对方式是强化墨描边框、印章红点缀或汉字古典排印处理——而非在现有基础上叠加更多装饰细节。

3. **Roguelike清晰度——每一次状态变化，皆须自我宣告**
永久死亡机制与分支决策对视觉表达提出了严苛要求：抽牌、兵力消耗、天气变化、统御上限预警等游戏状态变化，必须以即时、明确的视觉信号呈现。动画的职能在于确认状态，而非表演状态。
*设计验证*：当新的状态效果、兵种卡牌或地形条件进入场上时，须追问："玩家在小尺寸移动设备屏幕上是否可能错过这一变化？"若答案为是，则当前视觉信号——图标、色彩脉冲或音效配合的闪光效果——尚未达标。

## 2. 情绪与氛围（Mood & Atmosphere）

本章节定义《三国称雄》四种核心游戏状态的情绪目标与视觉氛围指向。所有状态均以"笔墨运筹，战术明晰"为视觉准则，在同一套丝绢-水墨-印章美学体系内保持风格统一。

### 2.1 节点导航（Map Exploration）
**核心情绪目标**：运筹帷幄之感——"我站于高处，俯瞰战局，每一步皆是决断。"
- **光照性格**：暖调、漫射。以陈旧羊皮纸或经年丝绢的琥珀黄为底色基调。
- **氛围形容词**：沉静、深远、端重、密谋感、蓄势待发。
- **能量等级**：沉思型（Contemplative）。节奏舒缓，鼓励玩家停下来权衡资源。
- **视觉调度**：地图以枯润结合的行书体勾勒州郡轮廓，节点以印章红圆点点缀。

### 2.2 卡牌战斗（Combat · 1v3）
**核心情绪目标**："临阵决断，一息之差，生死立判。" 
- **光照性格**：冷暖对峙。己方区域以暖铜金侧光为主，敌方区域偏冷、灰蓝。整体对比度高。
- **氛围形容词**：肃杀、紧绷、锋锐、目张力强、临界感。
- **能量等级**：克制型紧张（Measured Tension）。每一张牌出手都有分量，动画有力但不冗长。
- **视觉调度**：关键数值区以最高亮度层保障可读性。敌将意图以冷青色线框脉冲高亮。

### 2.3 军营与酒馆（Barracks / Inn）
**核心情绪目标**："片刻安顿，厉兵秣马，再整行装。" 
- **光照性格**：
  - **军营**：白昼冷光透营帐布料，带轻微靛蓝调，整洁而有条理。
  - **酒馆**：夜间或黄昏，暖橙烛光晕散，桌案与武将面孔沐于温暖光晕中。
- **氛围形容词**：
  - **军营**：肃整、精练、专注、有序。
  - **酒馆**：温煦、松弛、人情味、踏实。
- **能量等级**：沉稳型（军营）与舒缓型（酒馆），为玩家提供感官缓冲。
- **视觉调度**：功能分区以毛笔绘制的竹简分隔线区分，维持二维丝绢绘卷风格。

### 2.4 战役结算（Victory / Defeat）
**核心情绪目标**：胜利带来的历史厚重感，与失败带来的悲壮叙事感。
- **光照性格**：
  - **胜利**：正午或晨光，金黄色斜射光，印章红与金箔质感为主色。
  - **失败**：暮色或夜晚，冷靛蓝调，仅保留一缕暖光。
- **氛围形容词**：
  - **胜利**：庄严、辉煌、厚重、史册感。
  - **失败**：肃穆、悲壮、沉郁、萧索。
- **能量等级**：仪典型（胜利，文字如史官执笔显现）与沉思型（失败，留存空白卷轴提供反思空间）。
- **视觉调度**：胜利展开绢帛长卷，失败以笔墨溶散方式淡出武将肖像。

## 3. 形状语言（Shape Language）

设计原则总纲：一切形状的取舍，皆服务于"笔墨运筹，战术明晰"的视觉身份。

### 3.1 角色剪影哲学
**核心原则：一眼辨将，千人千面。**
每一位武将的剪影需在缩略图尺寸（64×64px）下即可被唯一识别。
- **七分史实，三分提炼**：保留具有高度辨识度的历史器物轮廓（汉代冠冕形制、特定兵器造型），将关键辨识部件放大10–20%，舍弃视觉混淆的琐碎细节。
- **书法笔触感**：轮廓边缘模拟毛笔行笔时的粗细变化，令角色如独立的墨迹符号。

### 3.2 环境几何语言
**核心原则：流动的战略地图，棱角的战术场域。**
- **宏观战略地图（有机曲线）**：地形模拟泼墨晕染，河流呈S形，边界模糊，传递"运筹帷幄"的流动感。
- **微观战术场域（几何硬边）**：阵型格位采用正方形与菱形，建筑轮廓采用45°/90°强硬折角，传递战术决策的确定性。
两种语言通过"毛笔的双重性"统一（草书的流动与楷书的方块）。

### 3.3 UI 形状语法
**核心原则：界面即竹简，信息即印章。**
UI不是现代HUD，而是世界观的延伸物。
- **卡牌框体**：模拟竹简，纵向矩形，上下收束为竹节弧形。
- **功能按钮**：模拟官印，正方形微倒角，按下如印泥压痕。
- **信息面板**：模拟绢帛卷轴。
坚持"形似而不拟物"：功能可读性永远高于拟物细节。

### 3.4 形状锋利度层级（英雄 vs 配角）
**核心原则：主角尖锐抢眼，配角柔化退场。**
视觉层级通过形状锋利度建立：
- **英雄/主动卡**：3个以上显著锐角，高对比度，轮廓清晰如刀切。
- **配角/兵种**：简化为基础几何体，大圆角柔化，视觉重量仅为英雄的1/3。
- **地形/背景**：全有机曲线，无明显锐角，主动消融于背景。

### 4.5 卡牌来源与专属分类视觉语言
玩家在卡牌奖励界面与卡组管理界面中，须能在0.5秒内辨认出卡牌的来源分类。三套视觉维度叠加：**边框主色调 × 纹样底纹 × 来源印记**。

**三大卡牌类型的视觉基调：**
- **标准卡**：旧绢黄/冷墨灰 + 素麻纸纹 + 汉制铜印（无字）。
- **武将专属卡**：阵营主色 + 绢帛丝纹与专属纹章底印 + 武将阵营兽纹印。拥有唯一的**金箔压边**标识。
- **十三州专属卡**：州域特征色 + 地域风土纹样 + 州名篆刻朱文印章。

**十三州地域色系与风土纹样细则：**
- **荆州**：苍绿水色 + 莲荷水波纹（楚地水泽）
- **扬州**：碧波青蓝 + 江潮海浪纹（吴越海疆）
- **益州**：深山苍翠 + 蜀锦菱格纹（蜀道险峻）
- **司州**：黄土赭红 + 砖瓦叠纹（中原王畿）
- **冀州**：麦田金褐 + 麦穗连弧纹（华北平原）
- **兖州**：烟尘赭黄 + 焦土龟裂纹（中原兵燹）
- **凉州**：黄沙赤铜 + 胡纹回字连环（西域羌风）
- **幽州**：寒冰白蓝 + 雪纹与狼牙纹（北疆草原）
- **徐州**：苔泥沼绿 + 芦苇沼泽纹（淮泗水泽）
- **并州**：石灰冷灰 + 山岩层叠纹（太行山脉）
- **豫州**：土陶暖橙 + 细陶纹描线（中原礼器）
- **青州**：海天浅碧 + 鱼鳞浪涌纹（渤海海滨）
- **交州**：南疆深翠 + 南蛮兽面纹（百越丛林）

## 5. 角色设计方向（Character Design Direction）

### 5.1 视觉原型
**七分史实躯体，三分策略符号。**
- **比例**：头部放大至约1/5身高，标志性兵器放大10~20%，确保缩略图识别。
- **甲胄**：以汉代两裆铠、筒袖铠等为原型。魏（铁灰硬边）、蜀（赭红流畅）、吴（碧蓝鱼鳞）、群雄（杂陈拼凑）。

### 5.2 敌我快速辨识规则
战场视角下必须在0.5秒内完成敌我识别。
- **色温**：玩家与友方处于暖色区间（金/铜/赭），敌方处于冷色区间（冷灰蓝）。
- **形状**：玩家武将多锐角（向上生长），友方兵种呈规整矩形，普通敌人大圆角，精英/Boss呈横向张开的锐角。

### 5.3 姿态与表情（绢画舞台感）
- 动作不追求写实摄影，定位于"戏曲武生亮相"与"绢画静态张力"。
- 动作幅度克制，受击不倒地，死亡以墨迹溶解淡出。禁止现代漫画式夸张表情。

### 5.4 细节层级（LOD Philosophy）
建立从 P0 到 P3 的细节分级。游戏镜头下必须优先保障 P0（轮廓、阵营色、关键表情），允许舍弃 P3（面部微表情、金属高光）。角色必须通过64px的"缩略图测试"。

## 6. 环境设计语言（Environment Design Language）

### 6.1 建筑风格与历史文化
**汉制为骨，军事为皮，战时为魂。**
所有建筑以东汉末年物质文化为原型，半写意呈现。
- 军营：折线硬边帐篷，辕门立旗。
- 关城：重墨粗线勾勒，保留马道箭垛。
- 酒馆：悬山顶木构，透出烛光。
- 阵营通过旗帜主色与徽纹（魏玄鸟、蜀龙、吴鱼鳞）区分，建筑本体保持材质中性。

### 6.2 质感哲学
**如绘于绢上——有笔触痕迹，无照片写实。**
拒绝PBR，采用"程序化绢画风格"的三层模型：
1. 基础型色层（固有色）
2. 绢帛噪声层（低频纤维纹理）
3. 笔触描边层（毛笔粗细变化的轮廓线）
严禁使用法线贴图与过曝高光。

### 6.3 道具密度规则
**中心清场，边缘叙事。**
- **前景层**（屏幕下35%战术交互区）：严格清空，禁止放置任何装饰性道具！
- **中景层**：叙事道具集中于此（普通战斗5件，Boss战12件）。
- **远景层**：极稀疏，最大3个元素。

### 6.4 环境叙事指南
**道具状态即战报。** 不依赖文字标签讲述节点故事。
- 普通战斗：散乱的粮车、插地的箭矢（仓促遭遇）。
- 精英战斗：整齐的矛架、挺立的将旗（有备而来）。
- 关隘节点：城门与旗帜的破损程度直接反映守军士气。

### 6.5 十三州大地图专属特征
宏观大地图须具备"主色调+地标轮廓+大气效果"的视觉锚点（与卡牌系统的州专属色严格对应）：
- **凉州**：黄沙赤铜 + 玉门烽燧 + 沙尘粒子
- **幽州**：寒冰白蓝 + 燕山锯齿 + 细碎雪花
- **并州**：太行冷灰 + 折线山体 + 谷底山岚
- **冀州**：麦田金褐 + 极低地平线 + 金色麦浪
- **青州**：海天浅碧 + 泰山独峰与渤海岸线 + 海风薄雾
- **兖州**：烟尘赭黄 + 焦土废墟 + 烟尘灰粒子
- **徐州**：苔泥沼绿 + 淮河多叉水道 + 潮湿低雾
- **豫州**：土陶暖橙 + 许昌方城 + 正午直射暖光
- **司州**：黄土赭红 + 洛阳双阙剪影 + 印章红光晕
- **荆州**：苍绿水色 + 长江大弧横道 + 水气蒸腾
- **扬州**：碧波青蓝 + 三角洲入海口 + 江潮动效
- **益州**：深山苍翠 + 四面环山对峙 + 山间云雾
- **交州**：南疆深翠 + 丛林树冠起伏 + 热带绿雾

## 7. UI/HUD 视觉方向（UI/HUD Visual Direction）

**视觉总原则**："笔墨运筹，战术明晰。" 界面是战略卷轴的延伸，而非现代数字仪表盘。

### 7.1 沉浸式与屏幕空间 HUD 划分（Diegetic vs Screen-Space）
- **拟世界层（优先）**：天气卷轴、HP竹简刻痕、AP令牌、竹简牌束。
- **屏幕空间层（绝对清晰）**：手牌区、敌将意图、状态图标栏、数值字体、伤害跳字（FCT）。
> 当拟世界表现与信息可读性冲突时，**可读性强制优先**。

### 7.2 字体排印方向
采用双轨制，层级功能严格隔离：
- **L0 / L1（氛围）**：篆刻仿古体 / 毛笔行楷体（用于标题、卡名）。
- **L2 / L3（功能）**：现代宋体 / 几何无衬线（用于效果文本、战术数值）。
> 严禁将毛笔体用于任何动态变化的数值显示。移动端L2文本最小绝对下限为 15px。

### 7.3 图标设计风格
**单色平面 + 毛笔轮廓边。**
- 不使用渐变、高光或立体斜切。
- 移动端最小触控图标强制要求：**44×44 pt**。

### 7.4 UI 动效手感
遵循"厚重落印、轻盈展卷"双节奏：
- **厚重型（80–120ms）**：超临界阻尼，带有印泥下压的闷响（按钮、状态施加）。
- **轻盈型（140–200ms）**：弹性收束，竹节清脆声（手牌展开、卷轴弹出）。
- **呼吸型（2.5–4s）**：玩家思考期间，除呼吸动效外所有动画强制暂停。

### 7.5 无障碍设计强制规范（BLOCKING）
- **色盲双重编码**：所有核心色彩信号（Buff/HP/阵营）必须配合独立的形状信号。
- **触控目标尺寸**：物理屏幕上不低于 9mm × 9mm（碰撞框与视觉解耦）。
- **减少动效模式**：必须支持系统级无障碍设置，替换为 ≤ 50ms 淡入淡出。

## 8. 资产规范（Asset Standards）

**核心矛盾裁决**：水墨美感通过着色算法与Shader层实现，不依赖超高分辨率位图。图集是强制约束，不是事后优化。

### 8.1 文件格式与命名
- **格式**：游戏包内资产统一导出为 `.webp`，特效帧序列使用 `.png` 打图集。严禁使用 `.bmp`、`.tga`。
- **命名**：`[type]_[system]_[name]_[variant].[ext]`（例如：`card_troop_spear_mob.webp`）。PC高分辨率层加 `_hd`，移动端基准层加 `_mob`。

### 8.2 纹理分辨率分级（基于 512MB RAM）
- **T1 旗舰级（1024×1024）**：武将全身肖像。
- **T2 标准级（512×512）**：卡牌插画主图。
- **T3 功能级（256×256）**：普通兵种卡牌、中景元素。
- **T4 图标级（128×128）**：UI图标。
- **T6 平铺级（256×256）**：底纹、Shader参数图（单张常驻）。
> PC端 `_hd` 版本上限可提升一档（如 T1 可达 2048×2048），不计入移动端内存预算。

### 8.3 图集（Atlas）与 Mipmap
- **强制图集**：所有UI元素、地图图标、状态图标必须按场景共现性打入图集（如 `atlas_ui_combat_hud`）。
- **Mipmap**：频繁缩放的节点图标和状态图标**开启**；固定尺寸展示的插画和武将图**关闭**；平铺纹理**开启**。

### 8.4 导出设置方案
- **压缩格式**：iOS使用ASTC 6×6，Android使用ETC2，PC使用S3TC/BC3或WebP。
- **WebP质量**：武将全图 92，卡牌插画 88，小图标 78。

### 8.5 技术美术硬性约束（BLOCKING）
- **Draw Calls**：战斗场景 ≤ 2000，战略地图 ≤ 800。
- **内存峰值**：战斗场景纹理内存 ≤ 120MB，全局峰值 ≤ 280MB（给引擎留出安全空间）。
- **视觉折衷方案**：禁止用高切图实现高稀有度卡牌效果或全屏泼墨特效，必须改用Shader时间函数扫光或噪声纹理驱动。

## 9. 参考方向（Reference Direction）

本节列出视觉开发的参考来源，每条均明确指定**借鉴元素**与**回避方向**。

### 9.1 游戏参考
- **《杀戮尖塔》**：借鉴卡牌信息层级的视觉权重分配（如费用和属性的固定几何锚点）；回避其西方奇幻插图与阴沉配色。

- **明代版画《三才图会》**：借鉴线描阵型示意图的俯视构图与线条逻辑；回避粗糙的刻版质感与黑白限制。
