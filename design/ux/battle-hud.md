# 战斗界面交互设计说明 (Battle HUD UX Spec)

_Created: 2026-04-09_
_Status: Draft_
_Accessibility Tier: Standard (WCAG 2.1 AA contrast, focus order documented, colorblind support via shape coding)_

---

## 1. Overview (概述)

本设计规范定义《三国称雄》中核心 1v3 卡牌战斗界面的信息架构、视觉层级与交互逻辑。战斗界面需要同时承载我方武将状态、最多3名敌方将领的状态（血量、护盾、意图、Buff/Debuff）、手牌管理、行动点管理、以及全局的地形天气和武将专属计数器。其核心挑战在于解决高密度信息在单屏内的组织问题，避免认知过载，确保玩家能顺畅执行"预判-构筑-破局"的战术循环。

## 2. Player Fantasy (玩家体验目标)

- **清晰掌控感**：我能一眼看穿战局的核心威胁（谁要打我，谁有破绽，当前什么地形）。
- **聚焦而非搜寻**：当我想打出一张卡时，界面会自然突显相关目标或连锁反应，不需要我逐一去比对图标。
- **信息分层折叠**：常态下我只看到必要的关键信息（血量、护盾、下回合大招），当我想深究某个状态（如"3层隐忍"到底什么效果）时，通过悬停能获得详尽但无干扰的说明。

不应有的感受：

- “屏幕上全都是小图标，我完全不知道该看哪里。”
- “我没注意到天气变了，结果我的引燃战术失败了。”
- “我选不中躲在后面的那个敌人，点错目标导致战斗崩盘。”

---## 3. Information Architecture (信息架构)

HUD 划分为 **5 大核心区域**，按认知优先级排序：

### 3.1 我方武将状态区 (Player Hero Zone)

- **位置**：左侧中下方 / 屏幕左1/3区域
- **信息层级**：
  - **一级 (最高)**：HP 进度条及数值 (如 45/58)；护盾数值 (如有，覆盖/叠加在HP上方，如 🛡️12)
  - **二级**：武将专属计数器 (如：隐忍、嗜血，采用独立于状态栏的显著图标+数字显示，如隐忍x3)
  - **三级**：常驻Buff / Debuff 图标栏 (按剩余回合排序，普通强度图标较小，强力强度图标略大/有特效边框)
  - **四级**：武器/装备微缩图标 (鼠标悬停显示词条)

### 3.2 敌将矩阵区 (Enemy Array Zone)

- **位置**：右侧 / 屏幕右1/2区域，呈前中后三排或扇形阵列分布（支持最多3将）
- **信息层级 (每名敌将)**：
  - **一级 (最高)**：下回合行动意图 (如 ⚔️12, 🛡️+8, ☠️诅咒)，在单位头顶显著悬浮
  - **一级 (最高)**：HP 进度条及数值；护盾数值
  - **二级**：Buff / Debuff 图标栏 (悬停看详情)
  - **三级**：蓄力中行动的进度预警 (如 ⏳下回合施放大招)

### 3.3 手牌与手牌操作区 (Hand & Interaction Zone)

- **位置**：屏幕正下方，水平扇形排列
- **信息层级**：
  - **一级**：卡牌可视范围 (包含兵种卡、攻击/技能卡、诅咒卡，诅咒卡采用明显的紫/黑色边框以示警)
  - **二级**：卡牌费用 (左上角数值，费用不足时暗化卡面)
  - **三级**：卡牌类型标识及效果短文本
  - **交互态**：当鼠标悬停或拖拽卡牌时，相关卡牌放大置前，其余卡牌沉降。

### 3.4 费用与战局资源区 (Action Points & Deck Zone)

- **位置**：屏幕左下角 (紧贴手牌左侧)
- **信息层级**：
  - **一级**：当前行动点 (AP) (如 🟢🟢⚪⚪，离散图标显示，当前费用高亮，空槽暗化)
  - **二级**：抽牌堆数量 (如 🎴15) 与 弃牌堆数量 (如 🗑️4)
  - **交互**：点击可查看具体堆中卡牌列表 (不含手牌)

### 3.5 全局环境与提示区 (Environment & Global Log Zone)

- **位置**：屏幕顶部居中及左上角
- **信息层级**：
  - **一级**：当前地形与天气 (如 ⛰️山地 / ☀️晴天)，天气变化时必须有全屏边缘转场特效和顶部显著横幅提示。
  - **二级**：战斗轮次计数器 (如 回合 4)
  - **三级**：战斗事件短播报 (如 "敌方释放了破甲"，位于屏幕左上或右下角的短暂飘字，不阻挡主视野)

## 4. Key Interaction Behaviors (关键交互逻辑)

### 4.1 "状态重叠" 与 "信息爆炸" 的降噪设计

- **悬停高亮关联项**：
  当玩家悬停某张手牌（如：含有「引燃」或「中毒」字眼的卡牌），界面上带有相应负面状态层数的敌军会通过光圈、震动或连线**高亮提示**，其他无关单位轻微变暗，降低筛选成本。
- **状态图标折叠 (Status Stacking)**：
  - 如果一名敌将身上挂了3层以上的**同一种**状态（如5层灼烧），图标只显示一个带火苗的图标，右下角标明**数字5**（如：🔥x5）。
  - 若状态种类超过3种（如同时有灼烧、破甲、虚弱、减速），优先展示**强力**（如滑倒、眩晕）和**即将失效**的状态，其余通过"..."收起，鼠标悬浮时展开完整列表。
- **武将专属计数器的特殊化处理 (Hero Specific Counters)**：
  由于司马懿"隐忍"、贾诩"诡谋"等并非C1通用状态，因此它们不应与通用Buff/Debuff混排。应在武将HP血条下方（或者状态栏的起始特殊位置）**独立展示**。
  - _表现形式_：**图标 + 数字** 的形式。例如曹仁的"铁壁"，显示为一个特殊的圆形金属边框图标，右下角标明层数（如 🛡️铁壁x2），与常规方/菱形状态图标位置区分，避免使用占空间的插槽设计。

### 4.2 敌方意图与阵列 (1v3 Panel Legibility)

- **1v3 空间排布**：
  屏幕右侧采用"**品字形**"（前排一人、后排两人）或"**斜向阶梯**"排列敌方单位，切忌一字排开导致模型相互遮挡。
- **"意图" (Intents) 的预判反馈**：
  敌方的下回合动作会在各自头顶悬浮图标（如 ⚔️ 剑代表攻击，带数字12表示伤害；🛡️ 盾代表上Buff）。
  当玩家拖拽一张防御卡到场上准备打出时，如果该防御卡能提供8点护盾，那么头顶有 ⚔️12 的敌人，其意图图标下方会动态显示出 **"12 -> 4"** 的预测计算，直观反馈"这张牌能挡住多少伤害"。

### 4.3 兵种卡的选定反馈

- **兵种入场动画**：
  因为兵种卡是核心概念，当打出兵种卡（如骑兵、弓兵）时，不应该是简单的光效，而应在场上快速闪现该兵种的**剪影或模型残影**，执行对应动作（如冲锋击退），然后消失，强化"召唤军队助阵"的统帅感。
- **地形/天气的联动预演 (Terrain/Weather Affordance)**：
  当玩家拖拽一张兵种卡（如"沙漠游骑"）且当前地形恰好是"沙漠"时，该卡牌的边框应**闪烁特有的环境色**（如沙金流光），且卡面上被增强的文本（如 +8伤害）会用高亮大字显示，形成强烈的正向确认感。

### 4.4 诅咒卡的警示 (Curse Visibility)

- **手牌中的诅咒卡**：
  紫色/黑色毒雾萦绕整个卡身。若该卡为"在手中每回合扣血"的类型，回合结束按钮会呈现呼吸状红色警示，并在诅咒卡和玩家HP之间绘制一条微弱的虚线，提示危险即将发生。
- **净化操作**：
  净化诅咒（军营移除等非战斗节点）时，诅咒卡应以碎裂、燃烧或消散的解脱音效/特效展现，与常规弃牌区分。

## 5. Visual Hierarchy & Legibility (视觉层级与可读性)

### 5.1 色彩与形状编码 (Color & Shape Coding)

- **血量 (HP)**：🟩 绿色 (我方)、🟥 红色 (敌方)
- **护盾 (Armor/Shield)**：🟦 蓝色 / 银色 (附着在 HP 条上/外侧)
- **行动点 (AP)**：🟡 黄色 / 橙色离散能量球
- **状态 (Status) 形状**：
  - **Buff**：🟢 正方形或向上箭头形边框 (如坚守、反击)
  - **Debuff**：🔴 菱形或向下箭头形边框 (如中毒、灼烧、破甲)
  - **专属被动**：🟣 圆形特殊金属质感边框 (如隐忍、铁壁)
- **环境 (Environment) 色彩**：
  - 各类地形（平原/森林/水域等）应在屏幕最顶部横幅有对应的**主题渐变色**（如水域为深蓝渐变，沙漠为黄沙渐变），且带有天气微小动效（如下雨、落雪）。

### 5.2 字体排印 (Typography)

- **伤害/回复跳字 (Floating Combat Text, FCT)**：
  - **白字/黑边**：普通伤害
  - **黄字/大字号/抖动**：暴击或地形增强的伤害
  - **绿字**：回复 HP
  - **蓝字**：回复护盾
  - **紫字**：持续伤害 (DoT) (如中毒、灼烧扣血)
  - 减免 (如格挡/坚守) 发生时，需在原伤害旁飘起带盾牌的 **"减免!"** 或 **"-25%"** 的短字，同时护盾破碎伴随清脆音效。

### 5.3 无障碍设计 (Accessibility)

- **色盲支持**：除了颜色区分 Buff/Debuff 外，还必须依靠**边框形状**（正方形 vs 菱形）来区分。
- **动态数字提示**：意图上的 ⚔️12 伤害，除了红色外，还应有一把明显带缺口的剑来表示威胁。
- **可配置暂停**：在玩家思考时，战斗应处于**完全静止状态**，所有动画仅为微小呼吸，不干扰文本阅读。

## 6. Acceptance Criteria (验收标准)

1. **信息无遮挡**：在 1v3 满场情况下（我方1人，敌方3人，各带满 Buff/Debuff），任何 UI 元素互不遮挡，FCT 不覆盖意图图标。
2. **专属计数器独立性**：所有 23 名武将的专属被动（如隐忍、诡谋）以独立图标+数字形式，在位置和形状上（圆形）与常规方/菱形 Buff/Debuff 栏明确区分。
3. **预测计算准确性**：当玩家选中防御卡（如8点护盾）悬停时，能准确在敌方意图下方反映出扣减后的受击数值（如原12伤，现在显示4）。
4. **地形天气强感知**：切换地图地形/天气时，顶部 UI 栏有显著颜色和动效变化，且对应的兵种卡面立即闪烁环境色高亮。
5. **状态折叠逻辑正常**：当敌将挂载的异常状态超过 3 个时，正确折叠显示不重要或低强度的状态，悬浮能查看所有。
6. **诅咒卡警示清晰**：诅咒卡处于手牌中时，有显著的视觉差异（紫黑边框或毒雾特效），且回合结束前如果诅咒卡会造成伤害，有清晰的预警提示。
