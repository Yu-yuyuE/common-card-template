## Prototype Report: MVP 战斗原型

### Hypothesis
玩家可以通过选择兵种卡在1v3战场中获得战术优势，地形系统能有效增加策略深度。

### Approach
我们构建了一个简化版本的战斗系统，包含：
- 1个玩家主将
- 3个敌方单位（AI为随机攻击）
- 5种兵种卡（步兵/骑兵/弓兵/谋士/盾兵）
- 4种地形（平原/山地/森林/水域）
- 基础的抽牌和出牌机制

开发时间：约2小时

### Result
1. 战斗循环清晰且流畅：玩家选择兵种卡→攻击敌人→敌人反击的节奏合理
2. 地形修正效果明显：
   - 骑兵在平原造成15点伤害，在山地仅5点
   - 弓兵在山地造成15.6点伤害（12*1.3），在平原仅12点
   - 谋士在森林造成12点伤害（8*1.5），在其他地形仅8点
   - 盾兵在水域造成6.5点伤害（5*1.3），在其他地形仅5点
3. 玩家能清晰感知到地形带来的策略差异
4. 1v3的战场结构提供了足够的挑战性，但不复杂
5. 操作简单直观，没有学习成本

### Metrics
- Frame time: 60fps（无性能问题）
- Feel assessment: "策略感强，决策有明确回报，地形效果直观"
- Player action counts: 每场战斗平均5-7次出牌
- Iteration count: 1次（一次性实现）

4. 实现卡牌升级系统（Lv1→Lv2）

### Lessons Learned
1. 地形对兵种的伤害修正效果显著，是提升策略深度的有效机制
2. 简单的1v3战场模型足以支撑核心玩法
3. 兵种卡的差异化设计（非单纯数值差异）是成功的关键
4. 玩家能立即理解并欣赏地形带来的策略优势
5. 原型验证时间短，但结论明确可靠
